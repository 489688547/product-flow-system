# 静态版到 React 版功能缺口审计

基准版本：`/Users/roger/Documents/product-flow-system/index.html`

目标版本：`/Users/roger/Documents/product-flow-system-react`

## 总体结论

React 版目前是“骨架迁移”，不是功能等价迁移。

旧版单文件已经包含登录、组织架构、权限、需求池、产品档案、产品进度、评审纪要、资料包、产品复盘、问题反馈、系统配置、钉钉日历/待办/会议/文档导入等完整业务闭环。React 版目前只迁了页面入口、基础状态、少量 CRUD、富文本和基础资料上传。

粗略规模对比：

- 旧版：8 个主页面、317 个业务函数、146 个 DOM 状态节点、68 个按钮/动作入口。
- React 版：7 个主页面、48 个函数、11 个测试入口，缺少大量业务流和权限/钉钉集成。

## 页面级缺口

### 1. 登录与账号权限

旧版已有：

- 钉钉免登登录页。
- 检测环境、重新登录、本地测试登录。
- 当前用户、部门、职位、最高优先级岗位判断。
- 组织架构缓存、过期判断、强制同步。
- 页面可见范围、操作权限、设置页/BUG 优化页仅总经办可见。
- 退出账号并清理本地登录状态。

React 缺口：

- 现在只有顶部固定“本地测试账号 · 总经办 / 总经理”。
- 没有钉钉免登流程。
- 没有组织架构缓存和岗位匹配。
- 没有按岗位控制页面、按钮、编辑权限。
- 没有真实退出/重新登录。

### 2. 总览

旧版已有：

- 需求池、在流程中产品、待评审会议三项指标，且可点击跳转。
- 本周待办事项：当前阶段未完成任务 + 所有逾期任务。
- 点击待办可定位到产品进度对应产品、阶段、任务。
- 风险提醒：逾期任务、当前阶段未完成、关键会议/纪要缺失。
- 风险点击可跳到对应产品进度。
- 受权限控制显示可进入页面。

React 缺口：

- 指标有，但待评审会议跳转现在指向已移除的 `reviews` 逻辑，不够稳定。
- 待办只展示基础任务，没有逾期/当前阶段排序策略完全迁移。
- 点击待办没有定位到具体任务。
- 风险提醒逻辑太粗，只按阶段未完成数提示。
- 没有按权限过滤入口。
- 没有从产品图片/身份组件复用完整产品信息。

### 3. 需求池

旧版已有：

- 需求推进规则。
- 需求池列表：名称、状态、等级、讨论摘要、来源、负责人、创建时间、操作。
- 状态筛选、等级表头筛选。
- 状态可切换并自动写入默认讨论摘要。
- 立项按钮常显示，但讨论完成前禁用。
- 已转开发需求不在池内，但可从产品退回后恢复。
- 编辑需求支持富文本机会描述、讨论摘要、图片。
- 负责人为组织架构人员搜索下拉。
- 来源部门为组织架构部门下拉。
- 必填校验：名称、负责人、等级、来源、描述。
- 删除需求有确认。

React 缺口：

- 富文本已迁一部分，但人员/部门仍是静态数组。
- 没有组织架构搜索下拉。
- 状态切换没有自动生成对应讨论摘要。
- 删除无确认弹窗。
- 缺少需求图片/描述图片的预览和复用。
- 立项后没有写入完整决策记录。
- 没有“已转开发”历史关联查看。

### 4. 产品档案

旧版已有：

- 产品列表支持状态、等级、阶段表头筛选。
- 产品身份组件复用图片、名称、描述、等级、负责人、阶段。
- 每个产品可进入进度、资料包、评审、复盘。
- 编辑产品时展示产品完整档案：图片、元信息、讨论记录。
- 图片上传、按产品名生成封面、图片兜底。
- 修改等级会影响任务模板、阶段适用性和正式定级状态。
- 删除产品会清理任务、资料包、评审、决策，并把来源需求退回可重新评估。

React 缺口：

- 产品列表只有基础按钮和基础描述。
- 状态/等级/阶段筛选缺失。
- 产品完整档案弹窗信息层级不足，缺少讨论记录、来源需求、生命周期/复盘数据。
- 封面图生成/上传逻辑不完整。
- 修改等级后未完整同步默认任务、阶段适用和正式定级。
- 删除产品功能缺失。
- 产品身份组件未抽成全站复用组件。

### 5. 产品进度

旧版已有：

- 产品选择融合在标题内，带图片、等级、阶段、负责人。
- 7 阶段按产品等级显示不同模式和是否涉及。
- 当前阶段、选中阶段、已完成阶段区分。
- 阶段任务：类别、任务内容、责任部门、交付物、截止、完成、同步待办、删除。
- 每个任务可同步钉钉待办，选择同事来自组织架构。
- 设为当前阶段有确认；如果往回调，会清除后续任务完成状态和评审通过状态。
- 立项阶段有“产品正式定级”：等级、定级依据、确认时间、决策记录。
- 不同等级自动生成不同默认任务。
- 会议准备项、会议节点、部门动作统一到产品进度。
- 退回需求池会清除任务、资料、评审、决策、钉钉会议记录。

React 缺口：

- 基础阶段和任务表已迁，但只是薄版。
- “同步待办”只是本地标记，没有同事选择和钉钉 API。
- 设为当前阶段无确认风险提示，也未清除后续状态。
- 正式定级流程缺失：等级依据、确认、决策记录、影响任务模板。
- 默认任务模板比旧版少，等级差异不完整。
- 会议节点/部门准备动作没有完整迁移。
- 任务删除无确认。
- 退回需求池缺少决策清理、钉钉会议清理、跳转需求池和最少保留产品保护。

### 6. 评审会议/会议纪要

旧版已有：

- 评审会议与产品进度同源。
- 根据产品等级决定需要哪些会议。
- 会议状态：待开会、待补纪要、已通过、未到阶段。
- 可预约钉钉日程。
- 可立即发起钉钉会议。
- 可同步/导入会议纪要。
- 支持从钉钉日历选择会议、匹配 AI 听记、导入钉钉文档、粘贴正文。
- 自动生成会议纪要文档和重点纪要图。
- 纪要图写入资料包，可下载。
- 查看纪要进入文档式页面，可编辑标题、负责人、原文。
- 可新增/删除评审和决策记录。

React 缺口：

- 独立 `ReviewDecisionPage` 还残留，但导航已去掉，产品进度里只有简化会议表。
- 没有根据等级完整过滤会议。
- 没有钉钉日程/会议/AI 听记/文档导入。
- 没有纪要图生成。
- 没有纪要进入资料包。
- 没有文档式查看/编辑纪要页面。
- 没有新增/删除评审、决策记录。

### 7. 资料包

旧版已有：

- 返回产品档案按钮。
- 当前产品资料包。
- 拖拽文件，支持图片、视频、PDF、PPT、表格、文档、压缩包、链接。
- 文件类型识别、预览标签、大小、来源、更新时间。
- 异常文件校验：空文件、超 12MB、本地不可读。
- 图片/视频/PDF/文本内容可打开预览。
- 打开时生成独立 viewer 页面。
- 删除文件有确认。
- 会议纪要图/纪要文档可自然沉淀到资料包。
- 清理打不开的默认坏文件。

React 缺口：

- 文件卡片有，但预览/打开逻辑太简单。
- 没有文件大小限制和异常报错策略。
- 没有 PPT/PDF/文档/视频等分类预览。
- 没有独立 viewer 页面。
- 删除无确认。
- 没有返回键。
- 没有会议纪要产物自动写入资料包。
- 没有坏文件清理/迁移逻辑。

### 8. 产品复盘

旧版已有：

- 产品复盘标准和决策路径。
- 产品复盘列表：产品、复盘节点、决策路径、核心数据、下一步动作、交付物。
- 可编辑销量、评分、退货率、利润率。
- 可调整产品状态。
- 根据数据推导升级、维持、清仓、淘汰路径。
- 第六/第七阶段对应上市复盘和产品复盘。

React 缺口：

- 复盘 tab 已按最新要求从左侧移除，但档案页里只放了一段占位文字。
- 没有复盘标准。
- 没有复盘数据编辑。
- 没有决策路径。
- 没有把状态更新同步影响其他页面。

### 9. BUG 优化 / 问题反馈

旧版已有：

- 右侧悬浮“提问题”按钮。
- 提交问题弹窗，问题描述和截图必填。
- 自动记录提交人、部门/职位、提交时间。
- 总经办 BUG 优化页汇总：待处理、处理中、已解决。
- 可调整问题状态、删除问题。
- 权限限制：只有总经办看到管理页。

React 缺口：

- 有问题反馈页，但没有全局悬浮按钮。
- 提交人只是“本地测试账号”，没有组织架构部门/职位。
- 没有总经办权限控制。
- 没有状态统计。
- 不能更新状态或删除。

### 10. 设置

旧版已有：

- 流程阶段配置：简称、阶段目标。
- 评审会议规则配置：责任部门、适用等级。
- 产品复盘路径配置：执行动作、交付物。
- 恢复默认配置。
- 系统设置：登录方式、组织架构缓存状态、流程阶段、资料与会议。
- 设置可见范围。
- 人员权限：每个岗位可见页面和操作权限。

React 缺口：

- 没有阶段配置。
- 没有会议规则配置。
- 没有复盘路径配置。
- 没有恢复默认。
- 没有组织架构缓存状态。
- 没有岗位级页面/操作权限矩阵。

### 11. 全局能力

旧版已有：

- 权限系统 `can()` / `denyIfCannot()`。
- 全局确认弹窗。
- Toast。
- 全屏 loading。
- 钉钉 API 环境检测。
- 本地缓存 + Cloudflare D1 共享状态。

- 产品详情深链。
- 图片兜底。
- Rich text 图片粘贴/上传。

React 缺口：

- 没有权限中间层。
- 没有统一确认弹窗。
- 没有 Toast。
- 没有全屏 loading。
- 没有钉钉 API 真实集成。

- 没有深链。
- 图片兜底和生成逻辑弱。

## 建议迁移顺序

1. 先迁数据模型：把旧版 `demands/products/tasks/deliverables/reviews/decisions/dingMeetings/feedbackIssues/config/orgCache/currentUser` 完整拆进 React domain/state。
2. 再迁全局组件：确认弹窗、Toast、全屏 loading、产品身份组件、人员选择器、文件 viewer、权限 wrapper。
3. 再补四个高价值页面：需求池、产品档案、产品进度、资料包。
4. 然后补评审纪要和钉钉集成：日历、待办、会议、文档导入、AI 听记、纪要图。
5. 最后补设置和权限矩阵。

## 当前 React 版最低优先级修复清单

- 产品进度：正式定级、阶段切换确认、后续任务状态清理、完整等级任务模板。
- 需求池：组织架构负责人/来源部门下拉、图片富文本、状态默认讨论摘要、删除确认。
- 产品档案：筛选、完整产品详情、删除产品、产品身份组件复用。
- 资料包：文件校验、viewer、删除确认、返回、各种文件预览。
- 总览：待办点击定位、风险点击定位、逾期逻辑和评审会议计数。
