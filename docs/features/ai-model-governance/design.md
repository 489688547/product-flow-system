# AI 大模型治理功能设计书

## 用户任务

老板或数据中心读者进入页面后，要在一分钟内看明白：公司哪些 App 和功能在用 AI、最近一段时间用了多少 Token、哪些 Skill 被调用，以及统一模型服务当前是否安全可用。维护 Provider 是低频任务，不应抢占使用统计的首屏位置。

## 信息层级

1. 主信息：已确认日期、模型调用、Token、成功率、Skill 调用、规则降级，以及按 App/功能/模型拆分的用量。
2. 辅助信息：Skill 使用明细、最近使用时间、无调用或历史缺失说明。
3. 低频信息：Provider 状态、模型选择、连接测试、数据外发策略，默认收起。
4. 不展示信息：员工排行榜、个人明细、提示词、回答、Skill 参数、费用估算和原始错误。

## 页面结构

### 导航与标题

- 保留屏幕 ID `data-services` 和内部 section `services`，只将可见名称改为「AI 大模型」，图标改为 `Bot`。
- 页面标题为「AI 大模型」，描述为「查看各业务 App 的模型与 Skill 使用情况，统一管理公司 AI 服务。」
- 页面身份说明为「服务端统一调用 · 不展示个人排行 · 内容不入审计」。

### 首屏

1. 日期工具栏：最近 7 天、30 天、90 天、自定义开始日、自定义截止日、查询、刷新。
2. 汇总带：模型调用、总 Token、成功率、Skill 调用、规则降级。输入与输出 Token 作为总 Token 下的辅助文字，不再增加一排装饰卡片。
3. 「App 与功能用量」表：
   - App / 功能
   - Provider / 模型
   - 模型调用
   - 输入 Token
   - 输出 Token
   - 总 Token
   - 成功率
   - 规则降级
   - 最近使用

### 次级信息

「Skill 用量」表按以下列展示：

- 调用方：调用 AI 的 App 与功能。
- Skill：中文名称与稳定 `skillId`。
- 数据来源：Skill 读取的业务 App。
- 调用次数。
- 成功 / 失败。
- 返回记录数。
- 最近使用。

### 页面底部

使用原生 `details/summary` 语义包装现有 `AiProviderSettings`，标题为「模型与安全设置」。默认关闭；摘要区仍显示「可用 / 未就绪 / 只读」状态。展开后复用现有 Provider 状态、合成连接测试、模型配置和数据外发策略。凭证不在此处录入或回显；总经办看到「前往数据接入配置连接」入口，目标为「数据接入 > 公司数据 > 灵算 AI 网关」。

## 交互流程

### 日期查询

1. 页面初始化时形成最近 30 天完整区间并自动请求一次。
2. 快速按钮会生成完整区间、立即应用并请求；当前选项使用 `aria-pressed=true`。
3. 自定义日期使用 `DateRangeControls` 作为基础输入，但写入独立 `draftRange`；仅点击「查询」后复制到 `appliedRange` 并请求。
4. 用户只选开始日、清空一端或输入倒序日期时，现有结果保持不变；查询按钮禁用，并在输入区给出原因。
5. 「刷新」只重新请求当前 `appliedRange`，不读取未确认草稿。

### 表格阅读

- 默认按总 Token 降序；Token 相同时按模型调用次数、App 名称排序。
- 已登记但无记录的功能显示灰色「暂无调用」，Provider/模型和成功率显示「—」。
- 同一功能在区间内使用多个模型时分多行，避免合并不同模型。
- 规则降级使用中性或警示状态，不用红色表达正常降级；只有统计读取失败使用错误状态。
- 页面不提供按员工下钻入口。

### 设置展开

- 所有页面读者可展开查看脱敏状态。
- 非总经办控件保持禁用并显示「只读」。
- 展开、收起不触发保存。只有点击现有「保存设置」才写入。
- 保存或连接测试成功后保持展开并显示反馈；失败时焦点移动到反馈区域。

## 组件复用

- `PageHeader`：页面标题、描述和身份说明。
- `DateRangeControls`：两个日期输入；由页面持有草稿与确认区间，避免输入即请求。
- `Button`：快速区间、查询、刷新与 Provider 操作。
- `DataTable`：两张高密度表，复用横向滚动、空状态和表头语义。
- `AiProviderSettings`：保留 Provider 状态、配置和数据外发策略，只调整外层展示和统一 AI 文案。
- `status-badge`：成功、警示、只读和暂无调用状态。

不新增通用图表组件。当前只有一个用量页面，且表格比折线图更直接回答 App、功能、模型与 Skill 的归属问题。

## 新增组件

### `AiModelWorkspace`

- 职责：管理日期草稿、已应用日期、统计加载和 Provider 设置折叠区。
- 输入：无；权限由统计 API 和 `AiProviderSettings` 各自在服务端确认。
- 状态：`draftRange`、`appliedRange`、`preset`、`loading`、`error`、`data`。
- 键盘：日期输入按文档顺序聚焦；Enter 在完整有效区间时等同「查询」；刷新与快速按钮均可由 Space/Enter 触发。
- 复用边界：该组件属于数据中心业务页面，不进入 `src/ui`。

### `AiUsageSummary`

- 职责：用紧凑语义列表显示五项汇总。
- 输入：`summary`。
- 状态：加载占位、0 值、成功率不可计算。
- 复用边界：只在 AI 治理页面使用，保留在同一 feature 目录，不提前抽为共享组件。

### `AiUsageTables`

- 职责：定义 App/功能表和 Skill 表列、数值格式及空状态。
- 输入：`features`、`skills`。
- 状态：已登记无调用、Skill 全空、部分历史缺失。
- 复用边界：只处理 AI 用量展示，不承担请求或权限逻辑。

## 页面状态

- 加载：日期工具栏保持可用；汇总和两张表显示固定高度骨架，避免布局跳动。
- 空数据：汇总显示 0，已登记功能显示「暂无调用」，Skill 区显示「所选时间内没有 Skill 调用记录」。
- 错误：页面顶部显示「AI 使用统计加载失败」和安全错误说明、Request ID、重新加载；保留当前日期与上一次成功结果但标明数据未刷新。
- 无权限：显示「当前账号没有数据中心查看权限」，不显示统计数量或 Provider 状态。
- 禁用：自定义区间不完整、倒序或超过 366 天时禁用查询，并在日期区说明原因。
- 成功：查询完成后在结果标题显示实际区间；不使用弹窗或全局提示打断阅读。
- 部分历史：电商运营行在首次统一接入前显示说明「统一接入前暂无统计」，不显示推测值。

## 响应式与钉钉 WebView

- 1280–1440px 笔记本：汇总单行，表格允许容器内横向滚动，不扩张整页宽度。
- 768–1279px：日期工具栏换成两行，快速按钮保持一组；汇总可换成 3+2。
- 小于 768px 与钉钉 WebView：日期输入纵向排列；表格保留第一列可读性并横向滚动；设置摘要占满一行。
- 使用 `100dvh` 所在页面现有布局，不新增固定底部操作区，避免与钉钉安全区和虚拟键盘冲突。
- Provider 设置中的宽表继续使用已有 `DataTable` 横向滚动。

## 无障碍

- 页面标题使用唯一 `h1`；两个数据区和设置区使用有名称的 `section`。
- 快速区间是一组可聚焦按钮，当前选项使用 `aria-pressed`，不只依赖颜色。
- 日期错误与输入通过 `aria-describedby` 关联；请求错误使用 `role=alert`，刷新成功状态使用 `role=status`。
- 表格中的成功率和趋势均提供文字，不使用只有颜色的上升下降图标。
- `details/summary` 保留浏览器键盘与读屏行为；摘要中图标设置 `aria-hidden=true`。
- 焦点顺序为日期工具栏、汇总、App 表、Skill 表、设置摘要、设置控件。
- 文本和状态色满足 WCAG AA；尊重现有 `prefers-reduced-motion`，不新增非必要动画。

## 视觉验收

需要在真实数据、空数据、加载、API 错误、只读设置和总经办设置六种状态下检查：

- 1440 × 900 笔记本宽度。
- 1024 × 768 窄笔记本宽度。
- 390 × 844 手机与钉钉 WebView 宽度。
- 键盘 Tab 完整走查日期、表格前后和设置摘要。
- 明暗状态下检查警示、只读、暂无调用和错误文本对比度。
