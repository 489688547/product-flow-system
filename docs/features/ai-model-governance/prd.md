# AI 大模型治理 PRD

## 文档状态

- 状态：已评审，开发中
- 负责人：产品负责人
- 最近更新：2026-07-22
- 功能范围：数据中心「AI 大模型」与全系统统一 AI 调用边界

## 背景与问题

数据中心当前左侧导航仍叫「数据服务」，页面同时放置 AI Provider、销售数据服务和应用订阅，无法回答老板真正关心的三个问题：哪些 App 和功能正在使用 AI、消耗了多少 Token、哪些 Skill 被调用了多少次。

系统已有公司 AI 总助及其灵算 Responses 服务端网关，也已经记录 Token 和 Skill 审计；但 `ai_usage_audit` 缺少稳定的 App 与功能归属。电商店铺运营的方案点评仍在自己的路由中直连 OpenAI，使用单独环境变量，失败时再使用规则检查，因此这部分调用既不经过统一配置，也没有进入统一审计。后续再增加 AI 功能时，若继续由各 App 自行接 Provider，会重复权限、安全、错误、审计和配置逻辑。

## 目标

1. 将数据中心左侧「数据服务」改名为「AI 大模型」，使用 AI 语义图标，并保留 `#data-services` 旧链接兼容。
2. 默认展示最近 30 天的公司 AI 汇总，支持最近 7 天、30 天、90 天和最长 366 天的自定义日期。
3. 清楚展示 App、功能、Provider、模型、模型调用次数、输入 Token、输出 Token、总 Token、成功率、规则降级次数和最近使用时间。
4. 清楚展示调用方 App/功能使用了哪些只读 Skill、读取哪个业务 App、调用次数、成功/失败次数、返回记录数和最近使用时间。
5. 所有业务 App 的模型调用统一经过服务端 AI 能力；首期迁移公司 AI 总助和电商店铺运营方案点评两个真实消费者。
6. 让未来 AI 功能必须先在服务端功能注册表登记，并通过自动化边界检查，避免再次出现 App 直连 Provider、无法归属或无法统计。
7. 保留 AI Provider 与数据外发策略配置，但移到页面底部并默认收起。
8. 将灵算 AI 网关纳入「数据接入 > 公司数据」的共享平台连接保险箱，页面只展示脱敏连接状态；现有环境 Secret 仅作为兼容回退。

## 非目标

- 不提供按员工排名、绩效比较或公开个人 AI 使用量。
- 不保存或展示提示词、回答正文、Skill 参数值、公司上下文或 Provider 原始响应。
- 不计算人民币成本或维护随时可能变化的 Provider 价格表。
- 不新增 AI 自动写业务数据、发送钉钉消息、创建待办或替代主管审批的能力。
- 不改变公司 AI 总助现有只读数据域和财务外发阻止规则。
- 不删除 `/api/platform/v1/data-services/sales` 销售数据 API；本次只移除「销售数据服务」和「应用订阅」页面内容。
- 不恢复电商运营上线前缺失的 Token 历史，也不根据调用次数猜测历史 Token。

## 用户与权限

- 具有数据中心查看权限的已登录员工可以查看 AI 汇总统计。响应只包含 App、功能、模型和 Skill 聚合，不返回员工身份或部门明细。
- 只有非只读总经办身份可以打开并修改 Provider 启停、模型和推理强度，也只有该身份可以执行连接与 Skill 合成测试。
- 运营部的数据中心编辑权限不能修改 AI Provider 或数据外发策略。
- `ai_usage_audit.user_id` 和 `department` 继续仅用于受控安全审计、故障定位和滥用调查，不进入本页面 API 响应。
- 所有权限由服务端基于真实钉钉会话再次校验；隐藏按钮不是授权边界。

## 当前流程

1. 用户进入数据中心「数据服务」。
2. 页面先展示 AI 模型服务，再展示销售数据查询和应用订阅。
3. 用户看不到 App、功能、Token 或 Skill 的使用情况。
4. 公司 AI 总助写入 Token 与 Skill 审计，但 Token 审计不能稳定区分 App 和功能。
5. 电商运营方案点评读取 `OPENAI_API_KEY` 和 `OPENAI_MODEL`，直接调用 OpenAI；未配置或失败时使用本地规则，统一 AI 页面无法统计。

## 目标流程

1. 用户从数据中心左侧进入「AI 大模型」。
2. 页面以最近 30 天为默认区间，完整日期选择只有点击「查询」后才生效；快速区间按钮直接应用已确定的完整区间。
3. 用户先看到公司级汇总，再查看「App 与功能用量」和「Skill 用量」。
4. 没有使用记录的已登记功能仍显示为「暂无调用」，避免把“没接入”和“没使用”混为一谈。
5. 需要维护模型服务时，总经办展开页面底部「模型与安全设置」；其他用户只能展开查看安全状态。
6. 需要维护灵算凭证时，总经办前往「数据接入 > 公司数据 > 灵算 AI 网关」，候选凭证通过合成只读请求后才替换当前版本；AI 页面不接收或回显凭证。
7. 业务 App 在服务端登记 `appId + featureId`，通过统一调用器使用模型。调用器统一解析 Provider、固定 `store:false`、限制超时、映射安全错误并记录一次无内容审计。
8. 电商运营方案点评继续使用原路由和原响应格式；模型可用时使用统一灵算能力，模型不可用时返回原有规则检查结果并记录规则降级。

## 业务规则

### 统计区间

- 默认区间为上海时区下包含今天在内的最近 30 个自然日。
- 快速区间提供最近 7、30、90 天；开始日和截止日均计入统计。
- 自定义区间必须同时包含开始日和截止日，开始日不得晚于截止日，最长 366 天。
- 日期输入先保存在页面草稿中，只有点击「查询」才替换当前结果；只选择开始日不得立即刷新。
- API 以 `Asia/Shanghai` 的自然日边界换算为 UTC 查询 `created_at`。

### 调用与 Token 口径

- 一次业务功能执行只产生一条 `ai_usage_audit`。最终由模型完成时 `execution_mode=model`；最终返回规则结果时 `execution_mode=rule_fallback`。
- `provider_called=1` 表示请求实际到达模型 Provider；模型调用次数等于区间内 `provider_called=1` 的审计条数。
- 模型成功次数等于 `provider_called=1`、`execution_mode=model` 且 `completed=1` 的条数。
- 成功率等于模型成功次数除以模型调用次数；模型调用次数为 0 时显示「—」，不显示 0%。
- 总 Token 等于输入 Token 与输出 Token 之和。规则降级固定为 0 Token，不计作模型成功，但单独计入规则降级次数。
- Provider 已调用但失败后回退规则时，记录一条 `execution_mode=rule_fallback, provider_called=1`；它计入模型调用和失败，也计入规则降级，不重复写第二条审计。
- Provider 未配置而直接使用规则时记录 `execution_mode=rule_fallback, provider_called=0`；它只计入规则降级。

### App、功能与 Skill 归属

- 调用归属只能由服务端已登记的固定 `appId` 和 `featureId` 决定，浏览器提交的同名字段一律不作为可信归属。
- 首期功能注册表包括：`company-ai-assistant / assistant-chat`（公司 AI 总助 / 对话分析）与 `ecommerce-operations / plan-review`（电商店铺运营 / 方案点评）。
- 未登记的 App 或功能不得调用统一 Provider，返回稳定错误并且不得发送任何公司数据。
- App 与功能表按 `appId + featureId + providerId + model` 分组；同一功能切换模型后分行展示，避免把不同模型的 Token 混在一起。
- Skill 表按调用方 App、调用方功能、数据源 App 和 `skillId` 分组。Skill 名称来自服务端注册表，未知历史 ID 使用安全原 ID 展示。
- Skill 调用次数等于 `ai_skill_audit` 条数；成功、失败、返回记录数和最近使用时间均由服务端聚合。

### 数据安全

- 统计 API 不返回 `user_id`、`department`、消息、回答、参数值、上下文内容或原始 Provider 错误。
- Provider 凭证只在服务端解析。「数据接入 > 公司数据」提供灵算 AI 网关的加密配置入口，候选凭证使用固定合成内容验证后才能激活；统一 AI 调用优先使用共享平台连接保险箱，`LINGSUAN_API_KEY` 仅作为兼容回退。浏览器、响应和日志不得出现已有明文或尾号。
- 当前财务数据域对灵算 Provider 继续固定阻止，迁移电商运营点评不得扩大其输入字段。
- AI 网关不可用时，业务流程继续工作；只允许使用已定义的本地规则降级，不允许浏览器改为直连 Provider。

## 数据定义

### `ai_usage_audit` 新字段

| 字段 | 含义 | 默认与历史处理 |
|---|---|---|
| `app_id` | 调用方业务 App 稳定 ID | 历史记录回填 `company-ai-assistant` |
| `feature_id` | App 内 AI 功能稳定 ID | 历史记录回填 `assistant-chat` |
| `execution_mode` | 最终结果模式：`model` 或 `rule_fallback` | 历史记录回填 `model` |
| `provider_called` | 是否实际调用 Provider，0 或 1 | 历史记录回填 1 |

原有 `input_tokens`、`output_tokens`、`result_code`、`completed`、`provider_id`、`model`、`created_at` 继续使用。增加 `created_at + app_id + feature_id` 索引以支持日期聚合。

### 统计 API

`GET /api/platform/v1/ai/usage?from=YYYY-MM-DD&to=YYYY-MM-DD` 返回：

- `range`：确认后的日期区间与 `Asia/Shanghai`。
- `summary`：模型调用、成功、成功率、输入/输出/总 Token、Skill 调用和规则降级。
- `features[]`：App、功能、Provider、模型及聚合用量；已登记但无记录的功能也返回。
- `skills[]`：调用方 App/功能、数据源 App、Skill、调用/成功/失败/记录数和最近使用时间。

响应不包含员工维度。SQL 聚合在 D1 内完成，不把原始审计行发送给浏览器。

### 历史数据

- 现有公司 AI 总助审计可确定归属，迁移时使用列默认值兼容并按公司总助展示。
- 电商运营旧 OpenAI 直连没有统一审计，无法可靠恢复调用次数或 Token；页面注明「统一接入前暂无统计」，不补造历史。
- Skill 历史通过 `request_id` 关联迁移后的公司总助调用归属；原有 `ai_skill_audit.app_id` 继续表示 Skill 读取的数据源 App。

## 异常与边界

- 无数据：显示已登记功能和「暂无调用」，汇总为 0，成功率为「—」。
- D1 不可用：统计 API 返回可重试的稳定错误，页面保留已选日期并显示「重新加载」。
- 日期非法：返回 `AI_USAGE_RANGE_INVALID`，不执行查询。
- 无权限：返回 `AI_USAGE_ACCESS_DENIED`，不泄露是否存在审计记录。
- Provider 未就绪或超时：电商运营返回规则降级；公司 AI 总助沿用现有安全错误，不恢复客户端直连。
- Token 缺失：按 0 聚合，并保留结果码；不得估算。
- 未登记功能：统一调用器返回 `AI_FEATURE_NOT_REGISTERED`，Provider 不被调用。
- 旧客户端：`/api/platform/v1/ai/chat` SSE 和 `/api/ecommerce-operations/ai-review` JSON 响应结构保持兼容。

## 验收标准

1. 打开 `#data-services` 时，左侧显示「AI 大模型」和 AI 图标，页面标题为「AI 大模型」，旧链接仍可用。
2. 页面不再出现「销售数据服务」或「应用订阅」，但销售数据 API 合同测试继续通过。
3. 首次进入自动查询最近 30 天；只修改开始日期不请求，补齐截止日期并点击「查询」后才更新。
4. 点击最近 7/30/90 天可得到完整区间，并显示对应聚合结果。
5. 有 AI 审计时，汇总 Token 等于输入与输出之和；App/功能/模型分组正确，成功率与规则降级口径符合本 PRD。
6. 没有审计时，两个已登记功能均显示「暂无调用」，页面没有虚构历史数据。
7. Skill 表能区分调用方功能和被读取的数据源 App，不展示参数值或返回内容。
8. 任意统计响应中均不出现员工 ID、部门、提示词、回答、凭据或 Provider 原始错误。
9. 电商运营方案点评不再读取 `OPENAI_API_KEY` 或请求 `api.openai.com`；成功时记录统一 Token，失败时保持原规则降级响应。
10. 静态边界测试能阻止新的业务路由直接出现 Provider 域名、Provider Secret 名或绕过统一调用器。
11. 非总经办用户无法修改 Provider；总经办可在默认收起的设置区展开、测试和保存，现有外发策略不变。
12. 新迁移在已有数据上可重复执行，旧公司总助审计仍可读取；回滚前端或 API 时不删除审计表和历史记录。
13. 「数据接入 > 公司数据」能配置并验证灵算 AI 网关；统一调用优先读取保险箱，保险箱停用时才回退现有环境 Secret，任何接口都不回显凭证。

## 上线与回滚

- 先部署兼容迁移与统计 API，再部署统一业务调用器和页面；迁移只加列与索引，不删除或改写现有审计内容。
- 发布前更新环境能力清单和集成注册表，生成平台模块，并验证 Cloudflare D1、Pages 与灵算三个受影响平台。
- 上线后分别验证本地 `npm start`、Preview 和 Production；本地成功不能替代生产验证。
- 若统计页面异常，可回滚前端并保留新增列。若电商点评统一接入异常，可回滚该路由到规则检查模式，但不得恢复 OpenAI 直连。
- 若统一 Provider 出现安全或稳定性问题，关闭现有 AI 功能开关或 Provider 启用状态；公司 AI 总助停止生成，电商点评继续规则降级，业务数据与审计记录保留。
