# 产品规划时间轴交互设计

## 用户任务

用户要在年度视图中快速读懂每个产品的开始和上线日期，并用低成本操作调整整个周期或单个端点；需要精确输入时仍使用现有日期弹窗。

## 信息层级

1. 主信息：产品名称、等级、日期范围。
2. 辅助信息：月份网格、时间段颜色图例、负责人标识。
3. 低频信息：来源需求已删除、完整日期和编辑提示。

时间条内部只保留日期范围。“开发至上线”作为颜色语义留在区块图例和可访问名称中。

## 页面结构

- 保留现有区块标题、产品固定列、12 个月网格和横向滚动结构。
- 时间轴行统一为舒适的数据表密度，产品信息拥有上下留白。
- 最后一行保留明确底部分隔，滚动内容在面板圆角内裁切，纵向网格与底边在同一基线结束。

## 交互流程

### 点击

- 时间条仍是单一键盘焦点。
- 点击且移动不足 4px时打开 `ProductPlanModal`。
- Enter 或 Space 同样打开弹窗。

### 中部拖动

- 中部使用 `grab` / `grabbing` 光标。
- 按下后记录原日期、指针位置和轨道宽度。
- 横向位移按当前年份实际天数换算为整数天，预览中同步移动两个端点。
- 松开时提交一次；结果被限制为至少与当前年份相交。

### 边缘缩放

- 左右边缘各提供视觉克制的命中区，悬停或拖动时显示手柄。
- 使用 `ew-resize` 光标。
- 左侧只更新 `developmentStart`，右侧只更新 `launchDate`。
- 开始、结束落在同一个日历日时即达到最短周期，停止继续收缩。
- 只有真实端点位于当前年份内时才显示对应手柄。

### 取消与回退

- `Esc`、`pointercancel` 或异常终止恢复原日期。
- 拖动期间不触发共享保存。
- 精确输入、键盘操作和异常恢复继续由现有弹窗承担。

## 组件复用

- 复用 `AnnualPlanningTimeline` 的月份轨道和 `timelineSegment` 定位。
- 复用 `ProductPlanModal`、`DatePickerField` 和 `updateProductPlan` 完成精确编辑与持久化。
- 复用现有焦点环、颜色变量、间距变量和权限判断。
- 不引入第三方拖拽库：现有依赖中没有可复用的时间范围拖动组件，本场景用 Pointer Events 即可覆盖鼠标和触控。

## 新增组件

### `PlanningRangeBar`

- 职责：渲染日期范围、区分移动与缩放命中区、维护拖动预览并在完成时提交日期。
- 输入：计划、当前年份、可编辑状态、编辑回调、日期变更回调。
- 内部状态：拖动模式、指针起点、原始日期、预览日期、是否超过拖动阈值。
- 输出：一次 `{ developmentStart, launchDate }` 变更，或普通点击编辑事件。
- 复用边界：保持在产品规划功能内；出现第二个真实时间范围消费者后再评估共享化。

### 纯日期计算

- 在 `src/domain` 提供无 React 依赖的日期格式化、整段移动和端点缩放函数。
- 所有计算按本地日历日进行，避免毫秒和夏令时造成偏移。
- 纯函数覆盖同月、跨月、闰年、跨年、同日最短周期和边界裁切测试。

## 页面状态

- 加载、空数据、错误和无权限沿用现有页面。
- 拖动预览：日期文字和条形位置同步变化，不显示额外弹窗。
- 拖动完成：无额外成功 Toast，时间条稳定停在新位置即可。
- 拖动取消：无错误提示，立即恢复原位置。

## 响应式与钉钉 WebView

- 笔记本宽度继续使用固定产品列和横向月份滚动。
- 窄屏不压缩月份宽度，保留横向滚动。
- 时间条使用 Pointer Events，不依赖 HTML5 Drag and Drop；钉钉 WebView 和触控设备可复用同一路径。
- 边缘命中区通过不可见扩展达到可触达高度，视觉手柄仍保持克制。
- `touch-action: pan-y` 保留纵向页面滚动，只有明确横向动作才调整日期。

## 交互文案

- 同月：`7月8日—28日`
- 跨月：`7月22日—8月25日`
- 时间条完整说明：`开发至上线：2026年7月22日至2026年8月25日。拖动中部移动时间段，拖动两端调整日期；点击精确编辑。`
- 左边缘提示：`调整开发开始日期`
- 右边缘提示：`调整预计上线日期`

## 无障碍

- 时间条保持原有 button 语义和清晰焦点样式。
- 无权限时不宣告拖动能力。
- 指针拖动的等价键盘路径是 Enter 或 Space 打开包含两个日期字段的弹窗。
- 颜色图例不作为唯一信息来源，完整日期和周期语义通过文本与可访问名称提供。
- 拖动引起的位置变化不使用持续动画；遵循减少动态效果设置。

## 视觉验收

- 1440px 和实际笔记本宽度：检查三条不同长度、异常来源提示、最后一行底边和日期可读性。
- 900px：检查固定产品列与月份横向滚动的边界。
- 390px 钉钉 WebView：检查纵向滚动、横向拖动、边缘命中区和弹窗回退。
- 悬停、拖动、焦点、只读和跨年端点隐藏状态分别截图核对。
