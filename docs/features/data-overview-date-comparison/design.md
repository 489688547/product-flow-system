# 数据总览日期范围与环比功能设计书

## 用户任务

用户进入数据总览后，需要快速选择一段完整经营周期，并在同一组 KPI 中判断相较紧邻上一期是上升、下降还是持平。日期编辑过程不应干扰当前正在阅读的数据。

## 信息层级

1. 主信息：已应用日期范围、五项本期 KPI、环比方向和变化值。
2. 辅助信息：上期范围、口径版本、数据截止时间和覆盖率。
3. 低频信息：范围校验原因、上期不可比原因、重新计算入口。

环比放在 KPI 主值下方，不新增一排装饰性卡片，也不恢复“数据分析”页面。

## 页面结构

- 顶部经营口径条保留“订单创建时间”和时区说明。
- 右侧原有两个原生日期输入合并为一个 `DateRangePickerField` 触发按钮，按钮显示已应用范围。
- 浮层顶部横向展示“近 7 天、近 15 天、近 30 天”快捷项；中部使用双月或单月响应式日历选择范围；底部固定显示取消、确认和校验说明。
- KPI 连续状态带保持五列结构，每项在主值与口径元数据之间增加一行环比。
- 趋势、平台贡献和数据健康区继续展示本期数据，不增加双周期图表。

## 交互流程

1. 点击日期按钮后打开浮层，草稿复制当前已应用范围。
2. 点击快捷项只更新草稿并高亮对应范围；点击日历第一天设置开始，第二天设置结束。
3. 草稿不完整或非法时确认按钮禁用，并以文字说明原因。
4. 点击确认后调用一次 `onConfirm(range)`；若范围未变化则只关闭浮层。
5. 点击取消、浮层外部或 Esc 时不调用 `onConfirm`，焦点返回日期按钮。
6. 已应用范围变化后，页面同时调度本期和上期共享口径结果；本期销售事实继续按已应用范围读取。
7. 环比结果到达前显示“环比计算中”；上期失败只影响环比行。

## 组件复用

- 复用 `react-day-picker`：项目已用于 `DatePickerField`，不新增依赖。
- 复用 `FloatingMenu`：保证浮层脱离裁切容器，并沿用关闭、定位和层级行为。
- 复用 `Button`：确认和取消使用统一按钮状态。
- 复用现有设计 Token、焦点样式和数据总览状态色。
- 不直接复用 `ProductCatalogWorkspace` 的日期逻辑：它把原生双输入与业务查询耦合，仍存在半选范围立即提交的问题。

## 新增组件

### `DateRangePickerField`

位置：`src/ui/DateRangePickerField.jsx`。

职责：提供业务无关的、确认式日期范围选择基础控件。

主要属性：

- `value: { from, to }`：已应用范围。
- `onConfirm(range)`：仅在用户确认完整且变化后的范围时调用。
- `presets: [{ id, label, resolve }]`：可选快捷范围，不包含业务文案默认值。
- `maxDate`：允许的最晚日期。
- `maxDays`：最大包含天数。
- `disabled`、`disabledReason`、`ariaLabel`：统一禁用与无障碍语义。

状态：关闭、打开未修改、选择开始、范围完整、非法范围、禁用。组件内部只保存草稿，不发网络请求、不写全局状态。

键盘行为：

- Enter/Space 打开；日历沿用方向键导航。
- Tab 顺序为快捷项、日历、取消、确认。
- Esc 取消并返回触发按钮。
- 确认后关闭并返回触发按钮。

复用边界：这是基础日期控件，可以进入 `src/ui`；本次只迁移数据总览。业务的“截止昨天”“近 7/15/30 天”和环比范围由数据中心传入，不写死在共享组件内。

### 数据总览环比行

保持在 `src/features/data-center/DataOverview.jsx`，因为指标方向、单位、文案和不可比原因属于数据总览业务组合，不进入共享 UI。

## 数据与状态流

1. `DataCenterProvider.range` 继续代表已应用范围，只有 `DateRangePickerField.onConfirm` 才调用 `setRange`。
2. `src/domain/dataCenter.js` 增加纯函数：
   - 生成截止昨天的 7/15/30 天范围；
   - 校验范围并计算包含天数；
   - 生成紧邻等长上期范围；
   - 计算普通数值百分比变化和比率百分点变化。
3. `DataStandardsProvider` 扩展为一次调度本期与上期结果，保留两组独立结果和状态；继续通过现有 `dataStandardsApi` 调用内部 API。
4. `DataCenterAppPage` 根据已应用范围计算上期范围，并把两组结果传给 `DataOverview`。
5. `DataOverview` 只负责展示，不直接请求 API，也不计算经营指标原值。

## 页面状态

- 加载：本期旧结果继续显示；环比行显示“环比计算中”。
- 空数据：本期无结果沿用“暂无结果”；上期无结果显示“上期暂无可比结果”。
- 错误：本期错误沿用顶部错误条；上期错误只显示在环比行并保留本期值。
- 无权限：沿用数据中心页面权限边界，不新增操作入口。
- 禁用：日期按钮禁用时显示原因；确认按钮在范围不完整或非法时禁用并显示原因。
- 成功：确认后触发按钮立即展示新范围，数据区域按现有加载机制更新。
- 兼容回滚：显示“环比暂不可用”，不把旧摘要与共享口径混算。

## 响应式与钉钉 WebView

- 1440/900px：浮层允许双月日历，快捷项横向排列。
- 640/390px：浮层限制在可视宽度内并改为单月日历；快捷项可换行，底部操作保持可见。
- 浮层使用现有固定定位与安全层级，不被状态带或页面滚动容器裁切。
- 不使用大幅动效；打开/关闭遵守现有浮层过渡和 `prefers-reduced-motion`。

## 交互文案

- 触发按钮：`2026/07/01 – 2026/07/15`
- 快捷项：`近 7 天`、`近 15 天`、`近 30 天`
- 操作：`取消`、`确认`
- 未完成：`请选择结束日期`
- 超限：`日期范围最多查询 370 天`
- 未来日期：`数据总览最多查询到昨天`
- 普通指标：`环比上升 20%`、`环比下降 8.5%`、`环比持平`
- 比率指标：`环比下降 2 个百分点`
- 不可比：`上期为 0，暂无可比比例`、`上期暂无正式结果`

## 无障碍

- 触发按钮包含完整已应用范围和“选择数据日期范围”的可访问名称。
- 浮层具有 dialog 语义和明确标题；日历范围起止不只靠背景色表达。
- 环比使用箭头、文字和数值，不只依赖红绿。
- 所有按钮保留可见焦点；禁用原因与控件建立可读取关系。
- 日期与数字使用稳定顺序，避免屏幕阅读器只读图标。

## 视觉验收

- 1440px：五项 KPI 单行，日期选择器与口径说明对齐。
- 900px：日期浮层不遮挡关键导航，KPI 自适应保持可读。
- 640px：顶部口径条换行自然，日期按钮和快捷项无横向溢出。
- 390px 钉钉 WebView：单月日历、底部按钮、焦点与页面返回行为正常。
- 状态：初始、选择开始未结束、范围完整、非法、加载、上期缺失、上期为零、上升、下降、持平、禁用。
