# 数据总览日期范围与环比 PRD

## 文档状态

- 状态：已评审
- 负责人：产品负责人
- 最近更新：2026-07-22

## 背景与问题

数据总览当前把两个原生日期输入框直接绑定到全局查询范围。用户修改开始日期时，旧的结束日期仍然存在，组件立即提交新的范围；销售事实请求与共享口径计算都监听该范围，因此用户尚未选完一段时间，页面就开始刷新。

当前页面也缺少常用日期快捷入口和上期对照。用户只能读取本期绝对值，无法快速判断净销售额、销售数量、毛利、退款率和毛利率相较紧邻上一期的变化。

## 目标

1. 用户在一个标准时间段选择器中完成开始、结束日期并确认后，页面才提交查询。
2. 提供近 7 天、近 15 天、近 30 天快捷范围，统一截止到昨天。
3. 五项数据总览 KPI 同时展示相同天数、紧邻上一期的环比变化。
4. 环比继续读取共享数据口径结果，不在前端恢复另一套经营指标公式。
5. 日期选择器成为业务无关的共享基础控件，覆盖键盘、焦点、禁用、错误和窄屏状态。
6. 精简总览层级，删除重复身份标语、正常结果版本元数据和底部数据健康面板。
7. 经营趋势和平台分布统一使用已确认范围的 GMV；趋势日柱展示完整经营明细。
8. 页头以一个可点击状态说明最新事实日期和待处理异常，并进入同步记录。

## 非目标

- 不恢复已经删除的“数据分析”导航或独立工作区。
- 不给经营趋势和平台分布增加双周期叠加或指标切换器。
- 不修改快麦、钉钉、电商平台等外部数据采集方式。
- 不修改 D1 表结构、数据口径公式、结果字段或 API 版本。
- 不在本次顺手迁移商品主数据、口径重算等其他日期输入界面。

## 用户与权限

- 所有已有数据中心查看权限的用户都可以选择日期并查看环比。
- 日期查询和 `ensure_current` 结果补齐沿用现有数据口径读取权限。
- 本功能不增加口径维护、数据写入或外部平台操作权限。

## 当前流程

1. 用户修改开始日期。
2. 页面立即修改全局 `range`。
3. 销售事实请求和口径结果补齐立即重新执行。
4. 用户再修改结束日期，页面再次执行整套查询。
5. KPI 只展示本期值，没有紧邻上一期的比较。

## 目标流程

1. 用户打开日期范围选择器，选择器以当前已应用范围创建本地草稿。
2. 用户选择快捷范围，或依次选择开始和结束日期。
3. 范围不完整、倒序、超过 370 天或晚于昨天时，“确认”保持禁用并给出原因。
4. 用户点击“确认”后，页面只提交一次全局范围并关闭浮层。
5. 系统读取本期销售事实，并为本期和紧邻上一期补齐共享口径结果。
6. KPI 展示本期值、上期范围和环比方向；上期不可比较时展示明确原因。
7. 经营趋势和平台分布使用同一个已确认范围；聚焦或悬停日柱时展示 GMV、销售数量、毛利率及各平台 GMV。
8. 页头显示最新事实日期；存在未解决质量问题、待确认映射或最新同步失败时显示警告，点击进入同步记录。

## 业务规则

### 日期范围

- 所有日期按 `Asia/Shanghai` 自然日解释，开始和结束日期均包含。
- 默认范围继续为当月 1 日至昨天。
- 近 7、15、30 天都包含昨天，分别向前覆盖 7、15、30 个完整自然日。
- 快捷范围只更新选择器草稿；与手工选择一致，点击“确认”后才应用。
- 打开后点击取消、按 Esc 或点浮层外部，均丢弃草稿并保留已应用范围。
- 单次范围最多 370 天，结束日期不能晚于昨天。

### 环比范围

- 环比使用紧邻本期之前、与本期包含天数完全相同的时间段。
- `上期截止日 = 本期开始日前一天`。
- `上期开始日 = 上期截止日 -（本期天数 - 1）`。
- 示例：本期 2026-07-01 至 2026-07-15，共 15 天；上期为 2026-06-16 至 2026-06-30。
- 上期计算显式使用本期五项结果实际采用的相同口径版本；历史日期早于该版本生效日时也保持同公式比较，不回退到另一版本或前端公式。

### 环比值

- 金额与数量指标：`(本期值 - 上期值) / abs(上期值) × 100%`。
- 退款率与毛利率：展示 `本期值 - 上期值`，单位为“个百分点”。
- 本期和上期都为 0 时显示“环比持平 0%”或“持平 0 个百分点”。
- 上期为 0、本期非 0 时不生成无穷百分比，显示“上期为 0，暂无可比比例”。
- 任一期值为 `null`、结果缺失、覆盖不完整或计算失败时不补 0，显示对应不可比原因。
- 上升、下降和持平必须同时使用图标/文字表达，不只依赖颜色。
- 净销售额、销售数量、毛利和毛利率上升为有利方向；退款率下降为有利方向。颜色表达经营方向，文案仍如实写“上升”或“下降”。

### 趋势、分布与数据状态

- 趋势和平台分布使用销售事实 `sales` 作为 GMV，不能混用净销售额；平台占比与排序也按 GMV。
- 每日毛利率使用当日 `grossProfit / netSales`，净销售额为零时显示“暂无结果”，不补零。
- 每个趋势日柱可悬停和键盘聚焦，明细包含日期、GMV、销售数量、毛利率和当日各平台 GMV。
- 最新事实日期取全部已读取销售事实中最大的合法业务日期，不随用户查看历史范围而回退。
- 当前销售事实、指标或上期指标读取失败时，页头必须显示警告而不是继续显示健康状态；点击仍进入同步记录定位问题。
- 页头警告数由未解决质量问题、待确认商品映射及每个来源最新一次非成功同步组成；历史失败后已有更新成功时不再计数。
- 没有最新事实日期时显示“暂无可用数据”，不能显示健康通过。

## 数据定义

| 字段 | 含义 | 来源 |
| --- | --- | --- |
| `currentRange` | 用户已确认、当前应用的开始和结束日期 | 数据中心全局查询范围 |
| `draftRange` | 浮层内尚未确认的日期草稿 | `DateRangePickerField` 本地状态 |
| `comparisonRange` | 与本期等长、紧邻本期之前的范围 | 纯领域日期函数计算 |
| `currentResults` | 本期五项 KPI 结果 | `/api/platform/v1/data-standards/results` |
| `comparisonResults` | 上期五项 KPI 结果 | 同一共享数据口径结果 API |
| `delta` | 环比数值、方向、单位和不可比原因 | 纯领域比较函数计算 |
| `day.platforms` | 每个业务日按 GMV 排序的平台销售事实 | 当前已确认范围的销售事实分组 |
| `latestDataDate` | 当前读取到的最大合法销售业务日期 | 全部销售事实行 |
| `syncAttentionCount` | 每个来源最新一次非成功同步数量 | 数据中心同步记录 |

API 返回的 `version`、`coverageRate`、`cutoffAt`、`status` 和 `reasonCode` 保持原义。环比不持久化，不产生新的数据库字段。

## 异常与边界

- 范围未选完整：不能确认，不触发数据请求。
- 日期倒序或超过 370 天：显示内联错误，不改变已应用范围。
- 结束日期晚于昨天：禁止选择或确认。
- 用户取消选择：恢复已应用范围，页面数据不变化。
- 本期读取成功、上期缺失：本期 KPI 正常展示，环比显示“上期暂无结果”，不把整个总览标成失败。
- 本期读取失败：沿用现有总览错误状态和重新计算入口。
- 上期计算失败：保留本期结果，环比单独显示不可比原因。
- 重复确认相同范围：不修改全局范围，不重复请求。
- 兼容回滚口径开启时：本期继续读取旧摘要；环比标记不可用，不在组件中计算第二套旧口径。

## 验收标准

1. 修改草稿开始日期后，销售事实和口径请求次数不增加；确认完整范围后只增加一次。
2. 近 7、15、30 天在上海时区均准确截止到昨天，确认后才刷新。
3. 2026-07-01 至 2026-07-15 自动比较 2026-06-16 至 2026-06-30。
4. 本期净销售额 120、上期 100 时显示“环比上升 20%”。
5. 本期退款率 8%、上期 10% 时显示“环比下降 2 个百分点”，并表达为有利变化。
6. 上期为 0 或没有正式结果时不显示 `Infinity`、`NaN` 或伪造的 0。
7. 选择器可用键盘打开、选取、确认、取消；关闭后焦点返回触发按钮。
8. 1440、900、640 和 390px 下无横向溢出，钉钉 WebView 中浮层不被父容器裁切。
9. 全量治理、集成、环境、测试和构建门禁通过。
10. 正常 KPI 不显示“版本 v…”和“数据截止…”，覆盖不足仍显示覆盖率原因。
11. 趋势柱鼠标悬停与键盘聚焦均显示 GMV、销售数量、毛利率和各平台 GMV。
12. “经营趋势”和“平台分布”均跟随已确认范围并按 GMV 计算，页面不再出现“销售事实视图”“平台贡献”或底部“数据健康”。
13. 页头健康状态可跳转 `#data-sync`；健康、异常和无数据三种状态文案准确。

## 上线与回滚

- 不需要数据库迁移、环境变量或外部平台配置。
- 上线后观察日期确认后的销售事实请求与口径计算次数，确认没有半选范围请求。
- 如出现阻断，可回滚共享范围选择器和环比编排提交，恢复原有日期输入与本期结果读取；数据库和历史口径结果不需要回滚。
