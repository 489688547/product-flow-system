# 供应链审批匹配 PRD

## 文档状态

- 状态：开发中
- 负责人：供应链部、财务部
- 最近更新：2026-07-18

## 背景与问题

钉钉付款审批同时承载采购付款、报销和工资等业务。现有同步把没有采购关联的付款也写入供应链，造成“待匹配付款”虚高。采购申请模板没有独立供应商和产品字段，相关信息主要写在事由中，导致所有采购记录都进入待映射。

## 目标

- 供应链只保留真正关联采购申请的付款审批。
- 事由包含供应商完整名称或已确认别名时自动关联供应商。
- 从标准采购事由中提取供应商别名和采购品名，供人工确认和后续复用。
- 没有可靠证据时不自动关联产品。

## 非目标

- 不修改钉钉审批模板。
- 不读取或保存收款账号、身份证、详细地址及完整审批原文。
- 不根据模糊相似度自动猜测供应商或产品。
- 不为仓库汇总费用自动生成产品分摊。

## 用户与权限

沿用供应链审批页面权限：总经办、供应链、采购和财务可同步；映射编辑权限保持不变。

## 当前流程

1. 同步最近 30 天采购申请和付款审批。
2. 空采购关联的付款仍被保存为未映射。
3. 采购模板没有供应商和产品字段，记录无法进入映射学习流程。

## 目标流程

1. 同步采购申请并识别标准事由中的供应商片段和采购品名。
2. 使用供应商完整名称或已确认别名关联供应商。
3. 仅同步钉钉中存在采购申请关联实例 ID 的付款。
4. 产品无已确认映射时继续显示待映射，由业务人员确认。

## 业务规则

- 付款审批没有 `采购申请单` 关联实例 ID 时不进入供应链付款集合。
- 付款关联的采购不在供应链采购范围内时一并跳过。
- 供应商完整名称命中事由时可自动关联；简称必须经过人工确认后写入别名映射。
- 标准事由形如“向供应商采购品名”，只提取两段候选文本，不将候选文本视为已映射。
- 产品只有命中已确认映射时才能参与实付和库存资金分摊。

## 数据定义

- `supplierId`：系统供应商 ID，来源为完整名称命中或已确认别名映射。
- `productIds`：系统产品 ID，仅来源于已确认产品值映射。
- `unmappedValues.supplier`：待确认供应商简称。
- `unmappedValues.product`：待确认采购品名。
- 历史数据：下一次同步时移除无采购关联的付款，并重新计算采购映射。

## 异常与边界

- 仓库汇总费用等非标准事由不提取候选值。
- 同一事由同时命中多个供应商时不自动关联。
- 钉钉限流、超时和权限错误沿用现有同步错误处理。
- 重复同步按审批实例 ID 幂等更新。

## 验收标准

- 报销、工资及无关联审批的付款不会出现在供应链付款集合。
- 完整供应商名称出现在事由中时自动关联唯一供应商。
- 简称和采购品名显示为待映射值，人工确认后后续同步复用。
- 未确认产品不会进入产品维度实付金额。
