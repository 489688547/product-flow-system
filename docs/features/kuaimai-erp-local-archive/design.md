# 快麦 ERP 本地原始归档与完整导入设计

> 快麦是公司网页数据采集器的首个完整 provider adapter。通用运行时见 [`../company-web-data-collector/design.md`](../company-web-data-collector/design.md)。

## 用户任务

数据负责人把快麦官方导出文件放入一个固定目录，系统安全保存原文件、识别数据类型、写入线上标准事实，并清楚显示成功、等待或失败原因。

## 架构

采用“本地原始归档 + D1 在线索引与事实”两层结构。公司 Mac 采集器拥有桌面目录写权限；Pages Functions 只接收脱敏标准记录和归档元数据。业务 App 通过平台 API 读取 D1，不直接访问桌面文件。

平台能力决策为 `抽取共享能力`：`ensureCompanyChrome`、`ChromeDevtoolsBrowser`、调度、任务状态、下载等待、通知、幂等和人工接管进入公司网页数据采集器；快麦只保留 provider 页面动作、文件识别、字段映射、ERP ingest、商品目录和货流投影，不新建第二套 ERP 集成。

## 组件边界

- `schedule.mjs`：纯函数计算上海时区计划范围、资源频率、补跑和稳定幂等键，不访问浏览器或网络。
- `browser-export.mjs`：只操作固定快麦域名和资源注册表中的报表地址，设置筛选、触发导出、识别登录/验证/结构变化并等待下载。
- `chrome.mjs` / `chrome-devtools.mjs`：复用独立 Profile 启动和 CDP 会话；扩展下载目录、导航、DOM 查询、点击与输入的受限原语。
- `orchestrator.mjs`：领取任务，串联浏览器导出、现有扫描器、归档、上传和任务状态，不解释快麦原始字段。
- `/api/platform/v1/erp-collection/jobs`：幂等创建/领取计划任务，记录心跳、游标、结果、错误和批次关联。
- 现有 `scanner.mjs`、`archive.mjs`、`ingest` 和 `archives` API：保持文件稳定性、归档、脱敏、分块和 D1 投影职责。

## 本地目录

```text
~/Desktop/公司数据中心/快麦ERP/
├── 待导入/
├── 原始归档/
│   └── <resourceType>/<YYYY-MM>/<sha256>__<fileName>
├── 已处理/
├── 失败文件/
├── 处理报告/
└── manifest.jsonl
```

根目录权限为 `0700`。manifest 只记录相对路径和脱敏错误，不复制数据行。归档使用临时文件、校验哈希后原子改名；目标已存在时验证哈希并复用。

## 数据流

1. 扫描器发现 `.xlsx`、`.xls`、`.csv` 或 `.zip` 文件。
2. 稳定性门检查两次 stat 结果；磁盘空间门要求剩余空间大于文件大小加 1 GiB。
3. 哈希器识别重复文件，资源识别器结合文件名和前 30 行表头确定类型。
4. 归档器保存原始字节并写本地 manifest。
5. 解析器在内存/流中剔除个人信息，产生标准记录和质量问题。
6. API 先写 D1 archive manifest，再幂等写批次、最小索引和对应投影。
7. 本地文件进入已处理或失败目录；重试复用 archive ID 和批次 ID。

## 资源投影

| 资源 | D1 结果 |
| --- | --- |
| 销售/订单明细 | `product_sales_daily` 聚合、订单最小索引、archive manifest |
| 订单 | ERP 订单最小索引与批次覆盖 |
| 商品/SKU | `product_catalog_items`、`product_catalog_skus` |
| 库存快照 | `goods_flow_inventory_daily` |
| 库存流水 | `goods_flow_events` |
| 供应商/采购 | 供应商最小索引、采购货流事件与异常 |
| 售后 | 售后货流事件与销售质量异常 |
| 店铺/仓库 | 维度映射与来源范围 |

每个投影适配器只消费标准记录，不读取 provider 原始字段；快麦表头映射留在本地解析器。

## 每日处理

首次安装由真实公司会话调用注册接口，生成固定范围的 `kuaimai_erp_ingest` 采集令牌；服务端只保存哈希，CLI 将明文直接写入 macOS 钥匙串。macOS LaunchAgent 每 15 分钟运行一次 `daily`，运行时从钥匙串读取令牌。

`daily` 首先上报设备心跳并从 D1 获取到期任务。上海时间每天 05:00，事实资源生成“昨天 00:00:00 至 23:59:59”的任务，快照和维度资源生成当天完整导出任务。任务按固定顺序串行执行，避免多个快麦导出争用同一下载队列。每个导出成功下载后立即进入现有 `scan` 流程；没有到期任务时只更新心跳，不制造零值或成功记录。

独立 Chrome Profile 保存平台自己的登录会话，但采集器不读取、导出或上传 Cookie。任务只通过 CDP 控制该 Profile 中固定白名单页面。登录或人工验证出现时，页面保持打开，任务进入 `waiting_human`；下一个 15 分钟周期检测到已恢复登录后继续同一任务。

## 资源计划

| 资源 | 频率 | 默认范围 |
| --- | --- | --- |
| 订单、订单明细、库存流水、采购、售后 | 每日 05:00 | 昨天一个完整自然日 |
| 当前库存、商品、SKU | 每日 05:00 | 当前完整快照 |
| 店铺、仓库、供应商 | 每日 05:00 | 当前完整快照 |

所有销售和订单时间均使用订单创建时间与 `Asia/Shanghai`。资源注册表同时保存报表 URL、筛选字段、导出按钮、下载文件特征、超时和选择器版本；未登记页面不能执行。

## D1 任务模型

- 任务：provider、resource、计划范围、幂等键、状态、attempt、claimed runner、租约、选择器版本和创建时间。
- 运行：任务 ID、开始/结束时间、阶段、下载文件哈希、archive ID、batch ID、记录数、错误码和安全摘要。
- 游标：resource 最近计划范围、最近完整成功时间和最近事实日期；只有完整成功后推进。
- 设备：runner 最近心跳、Chrome 状态、登录状态、当前任务和版本；不保存本机绝对路径、浏览器会话或令牌。

任务状态固定为 `queued`、`claimed`、`exporting`、`downloading`、`ingesting`、`success`、`failed`、`waiting_human`、`schema_changed`。领取使用短租约；进程退出后租约到期即可由同一设备恢复。任务幂等键阻止同一资源和范围重复创建。

## 失败通知

本地通知器在任务首次进入 `failed`、`waiting_human` 或 `schema_changed` 时，通过当前 macOS 用户会话发送一条系统通知。通知只显示资源名、业务日期、安全错误类型和“打开数据中心查看”，不包含页面正文、订单、店铺、客户数据、账号或凭据。通知去重键为 `businessDate + resourceType + errorCode + notificationStage`，同一错误当天不重复弹出。

每天 06:30 调度器检查当日全部资源任务；只要仍有未成功资源，就发送一次汇总通知并保持数据同步页面异常。若 macOS 通知发送失败，任务和 D1 异常仍正常保存，下一轮不把通知失败误判为采集失败。

## 页面状态

数据中心执行记录展示资源、范围、归档状态、入库状态、行数和错误摘要；不显示本地绝对路径。

- 加载：读取线上 archive manifest 和最近批次。
- 空数据：说明尚未归档任何文件，并给出桌面待导入目录。
- 等待导出：归档器可用但当天没有新文件。
- 错误：区分磁盘、表头、网络、权限和 D1。
- 无权限：只显示状态，不提供导入动作。
- 成功：显示文件哈希短码、覆盖范围、事实数量和处理时间。

## 错误与恢复

所有失败使用稳定错误码。归档成功是独立状态，API 失败不得回滚或删除本地文件。批次分块写入继续使用文件哈希与 chunk idempotency key；投影只有完整批次成功后才切换当前快照。

## 安全

- 原始文件仅保存在当前 Mac 的权限目录，不上传 Git、日志或前端。
- Pages API 拒绝密码、Cookie、Token、验证码、个人身份、电话、地址、物流单号和整行原始 payload。
- D1 与页面只保存相对归档路径；迁移 NAS/R2 时更新存储位置，不改变 archive ID。
- LaunchAgent 配置和日志不包含任何令牌或快麦凭据；采集器运行时只从 macOS 钥匙串读取固定范围令牌。

## 测试与验收

- 单元测试覆盖路径清理、哈希去重、原子归档、磁盘空间、文件稳定性和 manifest 恢复。
- 解析契约测试覆盖全部资源类型、敏感字段剔除和真实表头 fixture。
- API/迁移测试覆盖认证、权限、archive 幂等、批次关联、投影原子性、失败重试和回滚。
- LaunchAgent 契约测试验证无秘密、15 分钟间隔、单实例和重启恢复。
- 调度测试验证 05:00 创建昨天任务、全部资源每日执行、关机补跑和幂等键。
- 通知测试验证首次失败立即提醒、06:30 汇总、同日去重、无敏感内容和通知失败不改变采集状态。
- 浏览器适配器测试使用本地安全 HTML fixture，覆盖登录、验证、结构变化、导出成功、下载超时和 URL 白名单；测试不连接真实快麦或保存会话。
- API 测试覆盖任务幂等创建、租约、心跳、阶段转换、游标仅在完整成功后推进以及敏感结果拒绝。
- 真实验收分别记录本地目录、生产 D1、Pages 部署、Git main 和快麦导出证据。
