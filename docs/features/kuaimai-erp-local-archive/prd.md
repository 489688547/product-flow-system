# 快麦 ERP 本地原始归档与完整导入 PRD

## 文档状态

- 状态：已评审
- 负责人：数据中心
- 最近更新：2026-07-22

## 背景与问题

快麦销售历史聚合已覆盖 2024-02-13 至 2026-07-21，但生产 D1 中 ERP 原始批次、来源记录、库存和货流仍为空。真实销售主题明细 15 天约 340 MiB，全历史原始文件不适合进入 D1。Cloudflare R2 尚未启用，绿联 NAS 只有 Time Machine 服务发现，没有可用业务共享目录；本阶段经确认先使用公司 Mac 桌面固定目录保存原始文件。

## 目标

- 在 `~/Desktop/公司数据中心/快麦ERP/` 建立可恢复、可去重的原始文件归档。
- 归档现有快麦销售明细和交易订单导出，来源文件保持不变。
- 将订单、商品、SKU、库存、库存流水、供应商、采购、售后、店铺和仓库文件解析为标准事实。
- D1 只保存文件清单、批次、质量问题、必要索引和现有业务投影，不保存大体积原始行。
- 公司 Mac 每 15 分钟扫描待导入目录；只有完整稳定的新文件才触发归档和导入。
- 补齐代码、生产迁移、历史数据、Git main 和线上部署。

## 非目标

- 不自动绕过快麦登录、验证码、滑块、扫码或邮箱验证。
- 不把桌面目录描述为异地备份或高可用存储。
- 不保存快麦密码、Cookie、Token 或验证码。
- 不把缺失的快麦导出推断为零业务数据。

## 用户与权限

公司 Mac 的当前系统用户可写本地待导入目录和查看处理报告。本地归档目录权限固定为当前用户可读写。首次安装采集器时必须由真实公司会话注册固定范围的 `kuaimai_erp_ingest` 采集令牌；令牌明文只写入 macOS 钥匙串，D1 只保存 SHA-256 哈希，LaunchAgent 和日志不得保存令牌。手工导入仍使用真实公司会话；总经办、运营、数据中心、供应链和财务按现有路由权限读取或导入，普通业务 App 只读取标准事实。

## 当前流程

文件散落在桌面多个目录；销售聚合已写入 D1，但原始文件、批次清单和其他 ERP 资源没有统一归档、游标、投影或每日处理机制。

## 目标流程

1. 用户或浏览器将官方 XLSX/CSV 放入 `待导入`。
2. 本地守护任务等待文件大小和修改时间连续两次不变。
3. 采集器计算 SHA-256，识别资源类型和表头，先创建不可覆盖的归档副本，再执行预检。
4. 预检通过后分块调用现有受控 API；D1 保存归档清单、批次、必要标准索引和业务投影。
5. 成功文件移入 `已处理`，失败文件移入 `失败文件` 并生成不含个人信息的报告。
6. 重复哈希直接关联既有归档和批次，不重复保存或写入事实。

## 业务规则

- 销售和订单日期固定使用订单创建时间，时区为 `Asia/Shanghai`。
- 正常经营口径排除空平台、其它、其他、未知和未知平台；原始归档不修改来源内容。
- 归档路径使用 `原始归档/<resourceType>/<YYYY-MM>/<sha256>__<安全文件名>`，同一哈希只保存一次。
- 已存在的桌面来源文件采用 APFS clone/copy 创建归档副本，不移动、不删除、不覆盖来源文件。
- 原文件可能含个人信息，目录权限为 `0700`；文件内容不得上传到日志、D1 payload、Git 或前端。
- D1 `erp_source_records` 只保存稳定来源键、业务时间、店铺/仓库、内容哈希和允许的最小标准索引；完整原始字段只存在本地文件。
- 商品和库存快照使用最后成功文件覆盖对应来源快照；订单、库存流水、采购和售后为幂等历史事实。
- 文件成功归档但线上导入失败时保留归档并可重试；不得把“已归档”标记为“已入库”。
- 文件未出现时扫描结果为 `waiting_for_export`，不是成功，也不写零值。

## 数据定义

- 本地 manifest：哈希、原文件名、归档相对路径、字节数、资源类型、来源修改时间、归档时间、处理状态、批次 ID、错误码。
- D1 archive manifest：归档 ID、平台、资源类型、哈希、文件名、字节数、相对路径、存储类型 `local_desktop`、公司 Mac 标识、状态和最近处理时间；不包含绝对用户路径。
- 采集令牌：令牌 ID、SHA-256 哈希、固定范围 `kuaimai_erp_ingest`、状态、创建人、创建时间和最后使用时间；明文只在创建响应中出现一次并立即写入 macOS 钥匙串。
- 批次：沿用 `erp_collection_batches`，通过 archive ID 关联文件清单。
- 标准索引：订单号/明细键、时间、SKU/69 码、店铺、仓库及资源所需数量或状态；个人信息字段永不进入请求。
- 投影：商品进入共享商品目录；库存和库存流水进入货流；采购和售后进入货流事件及异常；销售继续进入 `product_sales_daily`。

## 异常与边界

- 文件仍在下载：保持待导入，不读取半文件。
- 重复文件：复用既有哈希，不重复保存和入库。
- 未知表头：归档后进入失败状态，报告列出缺少的字段名，不包含数据行。
- 本地磁盘空间不足：归档前停止，不移动来源，不创建线上批次。
- API、网络或 D1 失败：归档保留，状态为可重试；部分批次不能完成。
- 快麦无可导出的某类资源：记录 `waiting_for_export`，不伪造数据完整性。
- Mac 关机或睡眠：恢复后继续扫描，依赖哈希和幂等键补处理。

## 验收标准

- 现有桌面快麦原始文件均有唯一哈希归档，来源文件仍存在且字节一致。
- 同一文件重复放入待导入两次，只产生一个归档对象和一份事实。
- 含敏感字段的订单文件可以本地归档，但上传内容和 D1 中不存在这些字段或值。
- 订单、商品、库存、采购和售后样例分别通过预检并进入正确投影；无样例的资源明确显示等待导出。
- 每 15 分钟扫描任务可在重启后继续，处理成功、失败和等待状态可审计。
- 生产迁移、完整质量检查、数据回填、Git main 合并和 Pages 部署均有可复核证据。

## 上线与回滚

迁移只新增归档清单和投影关联字段，不删除既有数据。回滚时停用 LaunchAgent 和新路由，保留本地归档；按 archive ID 删除新增批次/投影并恢复部署。任何回滚都不删除用户原始来源文件。
