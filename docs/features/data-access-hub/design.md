# 统一数据接入中心设计书

## 设计方向

参考用户提供的应用市场截图，但保持经营执行平台“浅色、克制、高效”的产品气质。页面用横向分类和规整卡片建立浏览秩序，不复制截图中的营销评分、咨询按钮或夸张品牌色。

## 页面结构

1. 页面标题：数据接入。
2. 顶部说明条：说明“统一管理外部平台、ERP 与公司数据；敏感信息加密保存”。右侧保留刷新操作。
3. 横向分类：电商平台、ERP、公司数据。
4. 目录区：桌面三列卡片；各分类独立处理加载、错误、空状态和权限。
5. 详情区：点击可配置卡片后在同页进入详情，提供“返回数据接入”。连接器实例编辑继续使用现有弹窗。

不再显示“经营数据连接器”“内部系统保险箱”“平台连接”等需要理解系统实现的一级概念。“保险箱”只在安全说明和受控查看动作中出现。

## 分类导航

- 使用原生 `button` 和 `role=tab`，容器使用 `role=tablist`。
- 当前分类同时有蓝色文字、浅色背景和 `aria-selected=true`。
- 默认分类为 `ecommerce`。
- 支持 `initialCategory`，环境就绪阻断入口传入 `company`。
- 左右方向键移动焦点并激活相邻分类；窄屏允许横向滚动，但不截断文字。

## 统一卡片

卡片采用三段式结构：

1. 头部：本地平台标识、名称、用途、状态。
2. 主体：连接数量、支持方式、最近成功/验证时间或规划原因。
3. 底部：主操作“添加连接 / 管理 / 查看详情”，不可用时展示明确原因。

卡片使用白色背景、1px 中性边框、统一 12px 圆角和稳定高度。品牌色只用于 48px 标识区域；卡片本身不使用渐变、厚阴影或营销标签。整张卡片不嵌套另一张卡片，实例列表使用分隔行。

## 三类内容

### 电商平台

复用现有本地 Logo 与连接器状态。`ConnectorCatalog` 接收经过筛选的定义列表，不再自行固定遍历全部定义。快麦 ERP 不出现在此分类。

已配置实例最多展示三行摘要，更多实例在详情中查看。无编辑权限时卡片仍可读，操作按钮显示“只读”。

### ERP

快麦只出现一次。卡片汇总两个独立事实：

- 平台连接：公司级 App Key、App Secret、Token 是否验证并启用；
- 数据同步：订单、商品、库存和销售连接实例是否健康。

进入详情后按顺序显示：

1. 快麦平台连接专属表单；
2. 同步实例列表；
3. 添加或管理同步实例的入口。

两部分共享标题与返回路径，不共享敏感草稿。关闭或切换分类时清空未保存敏感输入，并遵循现有二次确认。

### 公司数据

- 钉钉：使用现有专属字段、验证、版本冲突和停用流程。
- 阿里云：展示用途和“准备接入”，操作禁用并说明尚无可用适配器。
- NAS、邮箱、财务系统、政务/SaaS、自定义内部系统：使用保险箱条目能力；卡片显示条目数和脱敏状态。

点击内部系统卡片后，详情只显示该类型的条目。NAS 详情突出地点、协议、地址、共享目录和只读策略；不得展示用户名、密码长度、Token 前缀或其他可推断凭据的信息。

## 组件与职责

### `dataAccessCatalog.js`

纯领域模块，只定义分类 ID、顺序和来源到分类的映射。它不包含 React、网络请求、凭据或重复的平台描述。

### `DataAccessTabs`

功能局部组件，负责三分类导航、键盘行为和选中状态，不读取业务数据。

### `DataAccessCard`

功能局部的统一视觉外壳。接收标识、标题、说明、状态、元数据和操作槽位；不决定权限、不发请求。

### `DataConnectionsWorkspace`

统一编排数据连接器状态、平台连接状态和内部保险箱条目。负责分类、详情选择、加载与错误边界；不保存敏感字段。

### `PlatformConnectionsWorkspace`

继续负责平台连接 API 和专属表单，但增加受控的 `platformIds`、`embedded` 和返回回调，使钉钉与快麦可以嵌入统一目录。服务端契约不变。

### `InternalVaultWorkspace`

继续负责保险箱条目和受控查看，增加 `initialType` 与 `visibleTypes`，使公司数据卡片进入对应类型而不重复二级分类。

## 共享能力判断

结论：扩展现有能力，不新增共享平台 API。

- 分类、卡片和详情组合仅服务数据中心，保留在 `src/features/data-center/connections`。
- 平台连接 API、加密凭证保险箱、权限、审计和 provider adapter 已是共享能力，保持原契约。
- `DataAccessCard` 虽有多个消费者，但包含数据接入专属状态与操作，不进入业务无关的 `src/ui`。
- 本次发现的“公司级连接与同步实例可同卡展示但不可混存”写回本设计书和 `docs/platform/integrations.md`。

## 加载、错误与权限

- 首次加载：保持分类和卡片骨架，不让布局跳动。
- 某一数据源失败：只在相应区域显示错误和重试，其他已加载卡片仍可使用。
- 平台连接 API 失败：钉钉和快麦显示“状态暂不可用”，不能显示“未连接”。
- 数据连接器 API 失败：电商平台和快麦同步显示错误，不能覆盖上次成功事实。
- 只读用户：卡片和脱敏状态可见，修改操作禁用并显示原因。
- 尚未接入：阿里云卡片可见但禁用，不提供虚假表单。

## 响应式与无障碍

- `>= 1180px`：三列卡片。
- `641–1179px`：两列卡片。
- `<= 640px`：单列卡片，分类横向滚动，按钮至少 44px 高。
- `390px`：详情单列，输入框不产生横向滚动，软键盘打开后主要操作仍可到达。
- 焦点清晰；状态有图标和文字；错误使用 `role=alert`，成功使用 `role=status`。
- 返回目录后恢复原卡片焦点；`prefers-reduced-motion` 下关闭骨架动画。

## 兼容与回滚

- 旧 `#/data-connections` 路由映射为 `#/data-sources/company`，不产生首页跳转。
- 环境阻断入口改为“前往数据接入”，直接打开公司数据分类。
- 回滚仅需恢复前端导航和组合组件；不涉及 D1、Secret、凭据密文或外部平台配置回滚。
