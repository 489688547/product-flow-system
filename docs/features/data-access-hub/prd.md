# 统一数据接入中心 PRD

## 背景

数据中心目前把“数据接入”和“平台连接”拆成两个左侧入口。快麦 ERP 同时出现在经营连接器目录和平台连接页，用户需要先判断自己是在配置业务数据源，还是在配置公司级 API 凭据。钉钉、阿里云、NAS 等公司数据入口也分散在平台连接和内部系统保险箱中。

老板需要看到的是“公司接了哪些数据、是否可用、下一步做什么”，不需要理解凭据保险箱、环境变量或适配器边界。

## 目标

1. 数据中心左侧只保留“数据接入”，移除独立“平台连接”入口。
2. 数据接入页面固定提供“电商平台、ERP、公司数据”三个分类。
3. 原“经营数据连接器”改名为“电商平台”，快麦 ERP 从该分类移入“ERP”。
4. 钉钉、阿里云、NAS、邮箱、财务系统和其他内部系统统一归入“公司数据”。
5. 同一平台在目录中只出现一次，但继续使用原有专属配置、验证、权限和加密存储能力。
6. 页面参考应用市场：横向分类、规整卡片网格、清晰状态和就近操作，适合老板快速浏览和处理。

## 非目标

- 不合并 `platform-connections` 与 `credential-vault` 两套服务端存储职责。
- 不迁移、复制或回显现有钉钉、快麦、连接器或内部系统凭据。
- 不把尚未完成适配器的阿里云或 NAS 能力伪装成已接通。
- 不新增外部平台 API 调用、环境变量、D1 表或 Cloudflare binding。
- 不在本功能中实现阿里云 OpenAPI、NAS 文件索引或新的 ERP 供应商。

## 用户与权限

- 所有已登录员工：查看三个分类、平台用途和脱敏状态。
- 具有数据中心编辑权限的用户：管理电商平台数据源实例及同步设置。
- 最高权限管理员：维护钉钉、快麦等公司级平台连接；管理被授权的公司内部系统条目。
- 服务端继续执行原有权限校验；隐藏或禁用按钮不是授权边界。

## 分类规则

| 分类 | 包含内容 | 主要任务 |
| --- | --- | --- |
| 电商平台 | 抖音电商、巨量引擎、快手生态、淘系、拼多多、小红书生态、京东/京麦 | 店铺平台等待原始文件样例；广告账户按真实能力配置 |
| ERP | 快麦 ERP；后续其他 ERP | 维护公司级 API 连接，并管理商品、订单、库存等同步设置 |
| 公司数据 | 钉钉、阿里云、NAS、邮箱、财务系统、政务/SaaS、自定义内部系统 | 维护组织协同平台、云资源和公司内部数据入口 |

分类是产品信息架构，不改变底层安全边界。快麦在目录中只有一张卡片：详情中分别展示“平台连接”和“数据同步”，前者使用公司级平台连接 API，后者使用数据连接器与凭证保险箱。

## 核心流程

### 浏览与切换

1. 用户进入“数据中心 → 数据接入”。
2. 默认打开“电商平台”。
3. 用户通过顶部横向分类切换“ERP”或“公司数据”。
4. 分类切换保留在当前会话中；从环境阻断入口进入时直接打开“公司数据”。

### 电商平台

1. 页面展示七个电商平台卡片。
2. 卡片显示平台标识、用途、支持方式、连接数量、汇总状态和操作。
3. “添加连接”或“管理连接”继续打开现有平台专属配置弹窗。

### ERP

1. 页面只显示一张“快麦 ERP”卡片，不再在其他分类重复出现。
2. 卡片汇总公司级 API 连接状态和数据源实例状态。
3. 进入详情后先显示平台连接配置，再显示同步实例和“添加同步”操作。
4. 两类保存仍调用各自现有 API，页面不在客户端复制凭据。

### 公司数据

1. 钉钉、阿里云和内部系统类型使用统一卡片外观。
2. 钉钉进入现有专属连接表单；阿里云显示“准备接入”且不可配置。
3. NAS、邮箱、财务系统、政务/SaaS 和自定义内部系统进入对应保险箱条目列表。
4. 公司数据列表只显示脱敏元数据；凭据查看继续要求授权、用途和审计。

## 状态与文案

- 已接通：平台和最近连接可正常使用。
- 需处理：凭据失效、配置不完整、同步失败或需要人工验证。
- 同步中：正在采集或同步。
- 尚未连接：存在真实适配器但未配置。
- 准备接入：注册表已有规划，但尚无可用适配器。
- 只读：当前账号可以查看但不能修改。

状态必须同时有文字和图标，不能只依赖颜色。读取失败时显示受影响分类和重试按钮，不把失败误报为“尚未连接”。

## 验收标准

1. 左侧数据中心导航不再显示“平台连接”。
2. “数据接入”默认显示“电商平台 / ERP / 公司数据”三个分类，键盘可切换。
3. 电商平台只显示七个平台，快麦 ERP 只出现在 ERP 分类。
4. 公司数据包含钉钉、阿里云和 NAS 等内部系统入口；钉钉归入公司数据。
5. 快麦目录只有一个卡片入口，详情可分别管理公司级连接和同步实例。
6. 原 `#/data-connections` 链接兼容跳转到“数据接入 → 公司数据”；环境阻断按钮使用新入口和新文案。
7. 已保存凭据不回显；服务端权限、验证、版本冲突、停用与审计行为不变。
8. 阿里云和未实现的 NAS 集成保持真实规划状态，不出现可保存的假配置。
9. 1440px 为三列卡片，900px 为两列，640px 以下为单列；390px 钉钉 WebView 无横向滚动。
10. 加载、空、错误、禁用、只读、连接成功和连接失败状态均可理解并可操作。

## 集成影响

- 直接涉及：钉钉开放平台、快麦开放平台、阿里云、ERP/文件导入、绿联 NAS。
- 一跳相关：Cloudflare D1、Cloudflare Pages、巨量引擎/巨量千川。
- 本阶段只改变信息架构和前端组合，不改变集成注册表的生命周期、API 路由、环境变量或数据表。
- PR 必须声明上述 `Integration-Impact`，并把本 PRD、设计书和相关平台文档列入 `Rule-Writeback`。
