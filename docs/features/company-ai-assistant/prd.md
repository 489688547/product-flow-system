# 公司 AI 总助 PRD

## 文档状态

- 状态：Skill 增量规格待书面复核
- 负责人：总经办 / 平台管理员
- 最近更新：2026-07-19

## 背景与问题

经营执行平台已经承载公司经营、供应链管理、产品全周期和数据中心等业务 App，但判断、风险识别和跨 App 分析仍依赖人员逐页查找和手工汇总。单页 AI 按钮无法回答“公司今天最需要关注什么”“产品延期是否会影响库存与经营目标”这类跨域问题，也容易让各业务 App 重复接入模型、重复处理权限和密钥。

公司需要一位面向所有员工、按组织权限提供不同视角的 AI 总助。总助读取可信服务端数据，跨 App 汇总事实并给出只读分析；首期不修改业务数据、不执行钉钉或外部平台动作。

## 目标

1. 在所有业务 App 中提供常驻的公司 AI 总助入口，并支持展开为完整工作台。
2. 建立供应商中立的公司 AI 网关，首期接入灵算的 Responses 兼容 API，后续可增加 Anthropic 或其他适配器。
3. 建立服务端公司 AI Skill 注册表，让战略、项目、产品、供应链、经营、数据中心及其他已接入业务 App 通过受控只读工具被总助按需查询。
4. 建立独立 AI 数据权限：总经理/总经办拥有全部内部数据权限，其他员工按数据域、部门和岗位授权。
5. 将用户查看权限与第三方外发权限分离；首期普通经营数据允许发送，财务数据对当前 Provider 统一阻止。
6. 在数据中心“数据服务”中管理 AI Provider 的安全元数据、连接状态、模型和外发策略，不保存或回显密钥。
7. 返回本次参考的数据域、数据更新时间和未纳入范围，让建议可核查。
8. 让长对话在面板和工作台中持续可滚动，并采用 `Enter` 发送、`Shift + Enter` 换行的常见输入行为。

## 非目标

- 首期不建立向量数据库、长期记忆、自动日报或主动弹窗。
- 不允许 AI 写入产品、任务、供应链、审批、财务或数据中心记录。
- 不执行钉钉待办、日历、消息、审批及任何第三方平台动作。
- 不把浏览器内完整应用状态或客户端提交的业务数据直接交给模型。
- 不保存对话内容、模型回答、财务原文、客户隐私或原始订单明细。
- 不承诺第三方 Provider 的内部留存行为；在供应商提供可审计的不留存承诺前，财务数据不得外发。
- 不接入用户已在聊天中暴露的旧 API Key；旧 Key 必须撤销并更换。
- 首期不实现 Anthropic 适配器，只保留稳定扩展接口。

## 用户与权限

- 所有已登录公司员工：可使用 AI 总助，但只能引用其已有权限和 AI 数据授权共同允许的数据。
- 总经理/总经办：拥有全部内部数据域的用户访问权限，可维护、测试、启停 Provider 和配置 AI 数据授权。
- 部门负责人和普通员工：按 `数据域 + 部门 + 岗位` 读取；无权在客户端扩大范围。
- 财务数据域：内部用户权限可授予总经办和财务部，但首期 Provider 外发策略为 `blocked`，因此任何人都不能通过总助把财务数据发送到灵算。
- 数据中心现有运营编辑权限不能修改 AI Provider；Provider 配置只允许总经办非只读身份维护。
- 服务端从登录会话确定身份并读取授权，不接受客户端传入角色、部门、数据域许可或公司业务上下文。

## 当前流程

1. 用户进入某个业务 App 查看局部数据。
2. 需要跨 App 判断时，手工切换页面并复制、整理事实。
3. 不同岗位依据各自看到的信息独立判断，结论难以复核数据范围和更新时间。
4. 外部模型配置散落在个人工具中，项目没有统一 Provider、权限、审计和失败边界。

## 目标流程

1. 用户从任意业务 App 顶部栏打开“AI 总助”，或展开为总助工作台。
2. 总助首屏展示当前用户可使用的数据域、数据新鲜度和受阻范围。
3. 用户输入问题；浏览器只提交问题、短会话历史和当前 App 提示。
4. 服务端验证登录并根据当前身份生成本次可见的只读 Skill 清单；客户端不能声明或扩大 Skill 权限。
5. AI 网关以 `store: false` 调用已启用 Provider；模型只能从服务端下发的 Skill 中选择查询，并提交符合固定 Schema 的参数。
6. 服务端重新校验 Skill、参数、数据域、用户权限和 Provider 外发策略，再从 D1 或现有共享存储执行只读查询。
7. 服务端把脱敏、限额后的查询结果交回模型，最多执行两轮、六次 Skill 调用，然后流式生成最终回答。
8. UI 展示正在调用的 Skill、查询结果状态、参考 App、数据日期、未纳入范围和 request ID。
9. 对话仅保留在当前浏览器会话；用户清空或关闭会话后删除。

## 业务规则

1. 总助是公司级能力，当前 App 仅作为问题路由提示，不限制可分析范围。
2. AI 数据域首期包括：公司战略、重点项目、部门承诺、产品需求与生命周期、供应链、经营检查、销售经营摘要、数据质量与同步状态、财务。
3. 财务数据包括成本、毛利、利润、预算、结算、奖金和可反推上述指标的细项。
4. 数据访问必须同时通过用户权限和 Provider 外发权限；任一道拒绝即不读取或不发送。
5. 用户权限中总经办隐式拥有全部数据域；其他身份使用显式部门和岗位范围。
6. Provider 外发策略优先级高于用户权限；首期 `finance=blocked` 对总经办同样生效。
7. 客户手机号、地址、Cookie、Token、密码、验证码、个人会话、原始供应商响应和原始订单隐私字段永不进入上下文。
8. 公司业务文本一律视为不可信引用资料，不能改变系统指令、权限或工具边界。
9. 单次最多 12 条会话消息，单条用户消息最多 4,000 字符，服务端上下文最多 24,000 字符，模型输出最多 2,000 token。
10. 每个用户同一时间只允许一个生成请求；流式失败不自动重试，避免重复计费。
11. Provider 必须使用注册表中的固定域名和协议路径，设置页不能配置任意外连 URL 或任意 HTTP Header。
12. `store: false` 必须写入每个 Provider 请求；若适配器无法表达该参数，连接测试失败且 Provider 不得启用。
13. Skill 必须由服务端注册，具有稳定 ID、所属 App、用途说明、JSON 参数 Schema、数据域、权限解析器、超时和结果限额；模型不能构造任意 URL、SQL、模块名或 HTTP Header。
14. 首期 Skill 全部只读，不得调用任何 POST/PUT/PATCH/DELETE 业务动作，不得触发同步、审批、消息、待办、日历、导出或第三方平台操作。
15. 单次回答最多执行两轮、合计六次 Skill 调用；同一 Skill 的同参数重复调用只执行一次，单次超时不超过 8 秒，结果按能力设置记录数和字符上限。
16. Skill 执行必须复用现有服务端存储和权限函数，不通过浏览器转发，也不通过内部 HTTP 绕过当前会话权限。
17. Skill 失败时保留其他已成功来源；若全部失败，返回可恢复错误，不允许模型把缺失数据推断成事实。
18. 对话输入使用 `Enter` 发送、`Shift + Enter` 换行；中文输入法组合状态、空白输入、禁用和生成中不得触发重复发送。
19. 消息区始终是面板或工作台的独立滚动区域；新消息默认跟随到底部，用户主动向上查看后停止强制跟随，并提供回到底部入口。

## 首期只读 Skill 范围

- 公司经营：查询战略、重点项目、部门承诺、经营风险和复盘摘要。
- 产品全周期：查询需求、产品、阶段、未完成任务和上市节点。
- 供应链管理：查询供应商、库存、质量问题、采购与付款审批状态；金额和成本字段继续遵循权限与财务外发阻止规则。
- 数据中心：查询销售经营聚合、数据质量问题、同步状态和数据新鲜度。
- 电商店铺运营：查询可见店铺、重点产品、经营方案、执行记录、协同与复盘事实。
- 品牌内容协同：查询可见内容任务、素材资产、发布记录和非财务表现指标。
- 绩效管理：仅查询当前身份原本可见的考核状态与经营证据，不读取薪资、奖金或隐藏评分。
- 尚未部署、没有稳定服务端读取边界或没有当前用户权限的 App 不向模型登记 Skill，也不以演示数据替代。

## 数据定义

### AI Provider 安全元数据

- `providerId`：稳定 ID，首期为 `lingsuan-responses`。
- `displayName`：展示名“灵算”。
- `wireApi`：固定为 `responses`。
- `baseUrl`：来自平台注册表的 HTTPS 白名单，UI 只读展示。
- `responsesPath`：Provider 适配器固定路径，不接受用户输入完整 URL。
- `model`：首期配置 `gpt-5.6-sol`。
- `reasoningEffort`：首期配置 `xhigh`。
- `enabled`：总经办控制的启停状态；缺少服务端密钥时不能启用。
- `storeResponses`：固定 `false`，不可改为 `true`。
- `lastCheckedAt`、`lastLatencyMs`、`lastStatusCode`：最近一次合成数据连接测试结果。
- `skillsSupported`、`lastSkillCheckedAt`：最近一次纯合成 Function Calling 能力测试结果，不包含公司数据。
- `secretConfigured`：由服务端环境变量派生，只返回布尔值，不写入 D1。

服务端密钥使用 `LINGSUAN_API_KEY`；如 Provider 必须使用专用 Actor Header，其值使用独立服务端 Secret，不进入数据中心状态。`AI_ASSISTANT_ENABLED` 控制总入口和 API 可用性。

### AI 数据策略

- `domainId`：稳定数据域 ID。
- `classification`：`internal`、`sensitive` 或 `restricted`。
- `viewDepartments`、`viewTitles`：非总经办的允许范围。
- `providerTransfer`：按 Provider 记录 `allowed` 或 `blocked`。
- `reason`、`reviewedAt`、`reviewedBy`：外发决策的依据和复核记录。

### AI 使用审计

- `requestId`、时间、用户稳定 ID、部门、Provider、模型。
- 使用的数据域 ID、每域记录数和更新时间。
- 被权限或外发策略阻止的数据域。
- 输入/输出 token、耗时、结果码和是否流中断。
- 不保存问题文本、回答文本、上下文值、手机号或原始外部响应。

### AI Skill 定义与调用审计

- `skillId`：稳定且业务中立的只读能力 ID；不使用页面路由或内部文件名。
- `appId`、`displayName`、`description`：所属 App 和供模型选择的用途说明。
- `domainIds`：执行前必须通过的 AI 数据域集合。
- `inputSchema`：固定 JSON Schema；拒绝多余字段、未知枚举、过长文本和越界日期。
- `execute`：服务端只读执行器，复用现有存储与权限投影。
- `timeoutMs`、`maxRows`、`maxChars`：每个 Skill 的执行与外发上限。
- 调用审计仅保存 request ID、Skill ID、参数键名或脱敏摘要、结果数量、耗时和结果码；不保存原始查询结果。

## 异常与边界

- 功能关闭：返回 `AI_DISABLED`，入口显示暂未启用。
- Provider 未配置或未启用：显示数据中心配置入口；普通员工只看到安全状态。
- 未登录：返回 401；会话失效时提示重新登录。
- 无数据权限：不读取该域；若没有任何可用域，返回可解释空状态。
- 财务被阻止：返回回答时明确列出“财务数据未纳入”，不得用其他字段推测财务结论。
- 数据缺失或过期：保留来源更新时间并提示限制，不把缺失值当 0。
- Provider 超时、429、5xx 或流错误：映射稳定 AI 错误码，保留已生成文本和手动重试入口。
- 客户端篡改当前 App、用户角色或数据域：仅作为无权限提示处理，不改变服务端选择结果。
- Provider 返回未知 SSE 事件：忽略未知类型并继续；无法恢复时安全结束流。
- 无 D1：聊天 API 返回配置错误；不得退回把前端完整状态发送到 Provider。
- Provider 不支持 Function Calling：连接测试将 Skill 能力标记为不可用；总助可以保留原有只读摘要回答，但 UI 必须明确显示“未启用按需查询”，不得伪装成已调用 Skill。
- Skill 参数无效或越权：服务端拒绝该调用并记录稳定错误；不向模型泄露被拒绝数据是否存在。
- 单个 Skill 超时或存储失败：将安全失败结果交回模型继续其他 Skill；最终回答标明该来源未读取成功。
- Skill 循环或重复调用：服务端按轮次、总调用数和参数指纹终止，返回已取得的有限事实。

## 验收标准

1. 任意业务 App 顶部栏都能打开总助，桌面为右侧面板，窄屏和钉钉 WebView 为全屏。
2. 总助可展开为完整工作台；当前页面只是提示，回答可引用多个业务 App。
3. 浏览器请求不包含公司业务状态、财务数据或权限声明。
4. 总经办请求普通经营分析时可引用其全部允许域；普通员工只能引用授权部门和岗位范围。
5. 任意用户询问利润、成本、预算、结算或奖金时，灵算请求体不含财务数据，UI 明确显示财务被阻止。
6. 每个 Provider 请求包含 `store: false`，使用白名单 URL，密钥仅来自服务端 Secret。
7. 数据中心“数据服务”展示 Provider、模型、推理强度、密钥配置布尔状态、连接测试和外发策略；非总经办不能修改或测试。
8. 合成连接测试不发送公司数据；旧 Key 未配置时 Provider 保持未启用。
9. 审计记录数据域、用量和结果，但不包含问题、回答或上下文值。
10. 加载、空、禁用、无权限、超时、429、Provider 失败、流中断和清空会话状态均可恢复。
11. 键盘焦点、关闭后焦点恢复、减少动效、390px 无横向溢出和 WCAG AA 对比通过验收。
12. `npm run lint`、`npm run check:governance`、`npm run check:integrations`、`npm test` 和 `npm run build` 全部通过。
13. 面板和工作台中的长回答可使用鼠标滚轮、触控板、触摸和键盘持续向下滚动；模型就绪与未就绪两种结构都不会把消息区放入自动高度行。
14. `Enter` 发送，`Shift + Enter` 换行；中文输入法确认候选词不发送，生成中连续回车不会创建第二个请求。
15. AI 能按问题调用数据中心及其他已登记 App 的只读 Skill；服务端权限不足的 Skill 不进入模型清单，客户端伪造 Skill ID 或参数不能越权。
16. UI 在生成期间展示 Skill 名称和所属 App；回答完成后可展开查看成功、失败、结果数量和数据日期，不展示原始敏感结果。
17. 单 Skill 失败不会丢失已生成对话或其他来源；全部失败、超时、循环和 Provider 不支持工具调用均有明确恢复状态。
18. Skill 审计不包含问题、回答或原始业务结果，现有 AI 功能关闭开关仍能整体回滚。

## 上线与回滚

- 新增 `AI_ASSISTANT_ENABLED` 开关，默认关闭；未配置新密钥时不得启用 Provider。
- 新增 Provider 安全元数据、AI 数据策略和 AI 使用审计表；现有业务数据无需迁移。
- 首次连接测试仅使用合成问题，不读取公司业务数据。
- 回滚时关闭功能开关并移除总助入口；现有业务 App、数据中心和 D1 事实不受影响。
- AI 审计记录可保留用于安全复核，不因回滚删除。
- 未经明确授权不部署、不写生产 Secret、不执行真实 Provider 验收。

## 集成影响

- `Integration-Impact`: `lingsuan-ai-gateway`, `cloudflare-pages`, `cloudflare-d1`
- `Integration-Impact-Reason`: 新增服务端 Responses Provider、Pages Functions AI API、D1 Provider/权限/审计元数据；不直接调用钉钉、快麦或其他业务平台。
