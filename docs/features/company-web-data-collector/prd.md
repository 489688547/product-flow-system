# 公司网页数据采集器 PRD

## 文档状态

- 状态：已评审
- 负责人：数据中心
- 最近更新：2026-07-23

## 背景与问题

公司需要从快麦、抖店、快手、天猫、广告平台和其他已登录后台持续获取经营数据。仓库已经存在数据连接浏览器代理、用户洞察采集器和快麦文件采集器，但它们分别管理 Chrome、任务、令牌、状态和错误，不能形成一个每天可审计、可恢复的后台采集能力。当前快麦 LaunchAgent 只处理人工导出的文件，电脑没有人工操作时不会主动产生数据。

2026-07-23 真实页面验收确认：快麦订单页点击“导出订单”或“导出订单明细”后不会直接触发浏览器下载，而是先创建异步导出任务并提示前往固定“下载中心”；文件生成完成后仍需点击对应任务行的“下载”。扩展若只监听订单页的下载事件，会把已经生成成功的文件误报为 `EXTENSION_DOWNLOAD_TIMEOUT`。

## 目标

- 提供公司 Mac 上唯一的通用网页采集运行时：MV3 插件运行在公司日常 Chrome Profile 中复用现有登录态，本机执行器管理每日调度、下载、任务、游标、通知和恢复。
- 平台差异收敛为 provider adapter；首个完整 adapter 为快麦 ERP，后续平台不再复制后台程序。
- 默认每天上海时间 05:00 执行，历史事实读取昨天完整自然日，快照与维度读取当前完整状态。
- 采集结果通过受控平台 API 写入 D1，业务 App 只读取数据库。
- 首次失败立即弹出 Mac 系统通知，06:30 仍未全部成功时再发一条汇总通知；数据同步页面同步显示真实状态。

## 非目标

- 不宣称任意已登录网页无需适配即可理解业务字段。
- 不绕过验证码、扫码、滑块、邮箱验证、手机验证或平台风控。
- 不保存或上传密码、Cookie、Token、验证码、完整页面正文、网络响应或浏览器会话。
- 不调用未登记域名、任意页面脚本或由远端任务下发的任意选择器代码。
- 本阶段不同时完成快麦以外平台的真实 adapter 验收。

## 用户与权限

总经办和数据中心管理员可登记采集设备、启停 adapter、查看所有任务与错误。运营、供应链和财务按现有数据权限查看对应来源状态。公司 Mac 当前登录用户完成首次平台登录和必须人工处理的验证。普通员工不能控制 Chrome、创建任务或读取采集令牌。

## 当前流程

各平台由人工登录、选择日期和导出；不同脚本扫描文件或读取页面，运行记录分散。失败通常在经营页面发现数据异常后才被追查。

## 目标流程

1. 管理员从仓库加载内置 MV3 插件，并在公司日常 Chrome 中保持目标平台登录；首期不要求发布 Chrome 应用商店。
2. 本机 LaunchAgent 持续运行 loopback 执行器；每天 05:00 根据 adapter 资源注册表生成幂等任务，插件最多每分钟领取一次本机任务。
3. 插件只访问代码中登记的 HTTPS 域名和页面，只执行打包在扩展内的固定动作；本机或 D1 任务不能下发任意 URL、选择器或 JavaScript。
4. 对快麦异步导出，adapter 记录本次导出开始时间和资源类型，进入代码登记的下载中心，等待并精确匹配本次任务行，生成完成后点击官方“下载”；下载文件再进入 provider 的待导入目录，由 adapter 做本地解析、敏感字段剔除和预检，并通过受控 API 写入 D1。
5. 完整成功后推进资源游标；失败、人工验证、结构变化和设备离线不推进游标。
6. 首次失败立即弹 Mac 通知；06:30 未全部成功再发一次汇总通知。数据同步页面显示设备、资源、业务日期、阶段、错误和恢复入口。

## 业务规则

- 调度、任务、运行、游标、设备心跳、通知和下载等待是共享能力；页面 URL、筛选方式、导出动作、文件识别和字段映射属于 provider adapter。
- 每个任务使用 `providerId + resourceType + businessDate + scheduleVersion` 生成幂等键。
- 默认日计划为 05:00；事实资源范围为昨天 00:00:00 至 23:59:59，时区 `Asia/Shanghai`。快照与维度资源使用执行时当前完整状态。
- 电脑关机、休眠或网络断开时，恢复后的下一轮补跑未完成任务；相同任务不得重复生成事实。
- 一个设备串行执行同一 provider 的导出任务。领取使用短租约，租约过期后允许恢复。
- 异步下载中心属于 provider adapter，不属于通用下载器。匹配必须同时满足申请导出时间不早于当前任务窗口、资源文件名前缀正确和任务状态为导出完成；`orders` 不得误选“订单明细”，不得仅凭“最新一行”下载。
- 下载中心入口、刷新控件、任务行和下载控件均固定在扩展包内；远端任务不得下发路径、选择器或文件名。
- 状态固定为 `queued`、`claimed`、`opening`、`waiting_human`、`exporting`、`downloading`、`validating`、`ingesting`、`success`、`failed`、`schema_changed`。
- 只有 `success` 推进游标。页面登录成功不等于采集成功，文件下载成功也不等于 D1 入库成功。
- 通知去重键为 `businessDate + providerId + resourceType + errorCode + stage`；同一错误当天只提醒一次。

## 数据定义

- Provider adapter：代码内固定 ID、允许域名、资源列表、页面入口、采集方式、选择器版本、超时、文件特征和解析器 ID。
- 设备：runner ID、名称、版本、最后心跳、Chrome 状态、当前任务和安全状态，不包含绝对路径或凭据。
- 任务：provider、resource、业务日期、范围、幂等键、状态、租约、attempt 和选择器版本。
- 运行：任务 ID、阶段、开始/结束、文件哈希、archive/batch ID、记录数、安全错误码和摘要。
- 游标：provider/resource 最近完整成功业务日期、任务和批次。
- 通知：任务、通知阶段、去重键、发送时间和结果；正文不持久化业务数据。

## 异常与边界

- 未登录时插件可打开固定登录页；验证码、扫码、滑块或风控出现后进入 `waiting_human`，不尝试绕过。
- 页面结构变化：进入 `schema_changed`，保存 adapter/选择器版本和安全错误码，不保存页面正文。
- 异步导出失败：返回安全的生成失败错误码，不点击失败任务行，不把平台追踪码、页面正文或原始备注写入 D1。
- 下载中心等待超时：临时文件不入库；任务保留导出阶段和安全错误码，人工导入仍是回退路径。
- 浏览器下载超时：下一轮先检查本次任务对应的既有下载再决定是否重试，不能改为下载其他任务的文件。
- 设备离线：心跳超过 30 分钟显示离线；恢复后补跑。
- 解析、D1 或网络失败：保留本地归档和任务游标，重试使用相同幂等键。
- Mac 通知失败：记录通知失败，但不改变采集任务结果；数据中心异常仍保留。
- 未安装 adapter：只显示“尚未适配”，不得执行通用抓取或宣称已接入。

## 验收标准

- 调度器在 05:00 为每个启用资源生成一个任务，重复执行不产生第二个任务。
- 电脑 05:00 关机、06:00 恢复后能够补跑昨天任务。
- 快麦 adapter 在已登录状态下先完成昨天订单与订单明细导出、归档和 D1 入库，再按已验证页面逐项扩展商品、库存等资源；未完成真实验收的资源必须显示未适配。
- 快麦订单与订单明细任务能从订单页进入固定下载中心，区分“订单”和“订单明细”，只下载当前任务窗口内生成完成的文件；异步生成失败或等待超时返回不同安全错误码。
- 登录失效时任务进入 `waiting_human`；完成登录后下一轮恢复相同任务。
- 首次失败立即弹一条 Mac 通知，06:30 未完成时弹一条汇总通知；同日相同错误不重复。
- D1 和数据同步页面能对应显示设备、任务、运行、游标和错误，不把计划时间或下载完成冒充成功。
- 任何任务、日志、D1 行和通知都不包含密码、Cookie、Token、验证码或客户个人信息。
- 增加第二个测试 adapter 时只新增 provider 目录和 adapter 注册，不复制调度器、Chrome 启动、任务 API、通知器或 D1 任务表。

## 上线与回滚

新能力置于 adapter 和调度开关之后。迁移只新增通用采集设备、任务、运行、游标和通知表；旧快麦归档、ERP ingest 和现有浏览器代理继续兼容。回滚时在 Chrome 扩展页停用插件并停用通用 LaunchAgent，恢复 provider 原有人工导出/`scan`，保留本地文件和 D1 运行元数据，不删除业务事实。
