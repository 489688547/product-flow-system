# 公司网页数据采集器设计

## 用户任务

数据负责人从仓库加载内置 MV3 插件，在日常 Chrome 中保持平台登录，并确认每天 05:00 自动采集是否成功；日常只处理登录失效、验证和页面改版等异常。

## 信息层级

主信息是今天哪些 provider/resource 成功、失败或等待人工，以及公司 Mac 是否在线。辅助信息是业务日期、当前阶段、耗时、记录数、批次和下次动作。历史运行、选择器版本和通知记录默认折叠。

## 页面结构

复用数据中心“数据接入”管理 adapter，复用“数据同步”展示设备、任务和运行。首期不新增左侧导航。快麦卡片是第一个启用的网页采集 adapter，未适配平台显示能力边界而不是虚假连接状态。

## 交互流程

管理员启用 adapter 后，插件可在当前 Chrome 打开固定登录页，复用用户已经建立的会话但不读取或上传凭据。每天 05:00 本机执行器生成任务，插件领取后自动运行；成功记录折叠，失败任务置顶并显示 Mac 通知。人工完成登录或验证后无需点击“重试”，下一轮自动恢复相同任务。

快麦订单类资源采用异步下载中心流程：adapter 在订单页完成创建时间筛选并点击官方导出，记录任务开始时间，随后导航到扩展内登记的 `#/index/download_center/`。销售事实使用固定的《销售主题分析-按订单商品明细》页面，明确选择“创建时间”、目标业务日和“按订单商品明细”后计算并导出。下载中心每轮只读取 `.m-data-item` 的安全元数据，用固定 `.j-search` 刷新列表；匹配到本次任务的生成完成行后点击该行 `.J-download`，再由通用 Downloads API 和本机执行器接管文件。页面路径和选择器不进入 D1，也不由远端任务提供。

## 组件复用

- 复用 Chrome 自身的用户 Profile 和既有登录态，不要求开启调试端口或复制 Cookie。
- 复用快麦 `scanner.mjs`、`archive.mjs`、ingest API、归档清单和领域投影。
- 复用数据中心同步卡片、状态徽标、表格、空态、错误态和人工配合动作。
- 现有 `browser_agent_tasks` 只服务带凭据的一次性连接身份识别，`user_insight_sync_runs` 只服务市场页面采集；两者不承担长期通用日计划，避免混入不兼容状态和权限。

## 新增组件

- `scripts/web-data-collector/schedule.mjs`：纯函数计算上海时区任务、补跑、资源范围和幂等键。
- `chrome-extension/company-data-collector/`：MV3 插件；包含固定 provider 注册表、白名单页面动作、下载监听、状态弹窗和本机桥接客户端。
- `scripts/web-data-collector/bridge.mjs`：仅监听 `127.0.0.1` 的安全桥接；使用配对密钥和固定扩展来源验证，不持有平台凭据。
- `scripts/web-data-collector/orchestrator.mjs`：领取任务并串联 adapter、下载、解析、入库、状态和通知。
- `scripts/web-data-collector/notification.mjs`：发送无敏感内容的 macOS 首次失败和 06:30 汇总通知。
- `scripts/web-data-collector/providers/<providerId>/`：provider 页面和文件适配器；首个目录为 `kuaimai`。
- `/api/platform/v1/web-collection/jobs`：设备心跳、任务领取、阶段更新、运行和游标接口。
- `web_collection_runners/jobs/runs/cursors/notifications`：通用 D1 运行元数据；provider 事实继续写入各自现有事实表。

每个 provider adapter 输出同一接口：

```js
{
  id,
  allowedOrigins,
  resources,
  classifyPage(page),
  exportResource(context),
  locateDownload(context),
  ingest(file, context)
}
```

adapter 不直接写 D1，不计算全局计划，也不能执行来自远端字符串的 URL、选择器或 JavaScript。插件只接收 `providerId/resourceType/businessDate/jobId` 等安全任务字段，具体页面地址和动作来自扩展包内版本化代码。

快麦下载中心匹配规则固定为：

- `order_items`：文件名以“快麦ERP交易订单明细导出”开头。
- `orders`：文件名以“快麦ERP交易订单导出”开头，且不得包含“订单明细”。
- `sales_items`：文件名以“销售主题分析-按订单商品明细-”开头。
- 申请导出时间必须位于当前任务开始时间容差窗口之后；多行满足时选择申请时间最近的一行。
- “导出完成”才点击下载；明确失败返回 `KUAIMAI_EXPORT_GENERATION_FAILED`；等待超过三分钟返回 `KUAIMAI_DOWNLOAD_CENTER_TIMEOUT`。
- 错误结果只回传安全错误码，不回传平台追踪码、任务文件完整路径、页面正文或行备注。

## 页面状态

- 加载：读取设备心跳、今日任务和最近成功游标。
- 空数据：解释未启用 adapter 或今日尚未到 05:00。
- 错误：区分设备离线、未登录、人工验证、页面变化、下载、解析、网络和 D1。
- 无权限：可查看安全状态但不能启停 adapter 或登记设备。
- 禁用：显示 adapter 未启用、未安装或缺少首次登录。
- 成功：显示资源业务日期、完成时间、行数和批次；成功历史默认折叠。

## 响应式与钉钉 WebView

沿用数据同步页的笔记本和 390px 单列布局。钉钉 WebView 只显示状态和恢复说明，不尝试控制本机 Chrome。Mac 通知点击后打开数据同步页面。

## 交互文案

- “公司 Mac 离线”
- “等待首次登录”
- “等待人工验证”
- “页面结构变化，已停止采集”
- “昨天数据尚未全部入库”
- “打开公司采集 Chrome”

## 无障碍

任务状态同时使用文字和图标。错误操作可键盘聚焦；成功历史使用原生 details/summary；通知不是唯一错误渠道。颜色保持 WCAG AA。

## 视觉验收

检查数据接入 adapter 状态、数据同步的设备离线、等待登录、部分失败、全部成功和无任务状态；覆盖 1440、1180、640、390px 和钉钉 WebView。

## 数据流与故障恢复

LaunchAgent 启动并守护本机 loopback 执行器。执行器向 jobs API 上报心跳，05:00 后创建昨天任务；扩展通过 `chrome.alarms` 最多每分钟向本机领取任务。adapter 在当前 Chrome 打开固定页面并触发官方导出；对于异步生成平台，adapter 继续在代码登记的下载中心完成任务匹配和官方下载。扩展通过 Downloads API 将完成结果回传本机，通用下载器确认文件稳定后交给 provider ingest。每个阶段回写 D1；只有 provider ingest 完整成功才完成任务和推进游标。下载文件解析出的首尾业务日期必须同时等于任务业务日期，否则以 `WEB_COLLECTION_BUSINESS_DATE_MISMATCH` 失败，避免快麦控件显示值与内部查询状态不一致时误覆盖事实。

任务租约超时后可恢复，幂等键阻止重复任务，文件哈希和 provider ingest 阻止重复事实。控制面在执行器重复提交同一计划时检查失败任务：仅下载、网络和本机处理类安全错误进入自动恢复，按第 1、2 次失败分别等待 5、15 分钟，最多领取 3 次；重新排队时清除租约、runner 和旧错误但保留 attempt。每次成功、失败、等待人工或页面结构变化都追加安全运行记录，重试不得覆盖旧尝试；数据同步页把运行记录与任务的来源、资源和业务日关联展示。登录/验证保持页面打开并等待员工确认，`schema_changed` 等待适配器修复；修复发布时提升资源 `scheduleVersion`，由新幂等键创建一条可审计的新任务。通知使用独立去重记录；通知失败不影响任务结果。

数据同步页的人工兜底始终使用当前失败任务的 `resourceType`，不再把订单、订单明细和销售主题明细统一写死为 `order_items`。旧失败任务和新版本任务同时保留，页面按业务日期、资源和更新时间展示真实状态。

## 安全边界

本机桥接只监听 `127.0.0.1`：请求带 `Origin` 时要求固定扩展来源；Chrome MV3 Service Worker 未发送 `Origin` 时仍要求随机配对密钥，缺少或错误密钥一律拒绝。远端 runner token 只存在 macOS Keychain。代码注册表固定允许域名、入口、动作类型和选择器版本，D1 任务不能下发任意 URL、选择器或脚本。插件不申请 Cookie、历史记录或网络拦截权限；Cookie、页面正文、截图、网络响应和个人信息不进入日志、API、D1 或通知。

快麦订单采集必须选择“下单时间”，销售主题明细必须选择“创建时间”；二者都写入业务日期的上海时区完整日范围，并在查询/计算前后回读校验起止值。值未生效时以对应的 `KUAIMAI_*_TIME_RANGE_NOT_APPLIED` 失败，不允许按默认日期继续导出。点击订单导出入口后，插件必须在弹窗内逐项确认代码登记的非个人经营字段，不能继承快麦账号的默认模板；字段缺失或未生效时停止。订单明细沿用其官方导出结构。配置完成后才点击“立即导出”并进入下载监听。

## 测试策略

- 纯领域测试：05:00、昨天范围、关机补跑、全部资源、幂等键、瞬时错误退避、3 次上限、人工状态不自动恢复、06:30 汇总和通知去重。
- MV3 适配器测试：校验最小权限、固定来源、固定动作注册表，并用安全页面探针覆盖成功、未登录、验证、结构变化、下载超时和域名拒绝。
- 快麦下载中心测试：覆盖订单/订单明细文件名互斥、任务时间窗口、生成中、生成失败、完成下载和三分钟超时。
- API/迁移测试：认证、权限、任务租约、状态转换、游标、心跳、通知和旧数据兼容。
- provider 契约测试：第二个测试 adapter 不复制共享运行时；快麦 adapter 保持订单创建时间和敏感字段剔除。
- 真实验收：独立 Chrome 首次登录、05:00 任务补跑、Mac 通知、D1 任务与事实、数据同步页面分别留证。
