# 平台连接 PRD

## 文档状态

- 状态：开发中
- 负责人：总经办 / 平台管理员
- 最近更新：2026-07-19

## 背景与问题

系统已经检查钉钉和快麦的运行配置，也会在缺少 `DINGTALK_APP_KEY`、`DINGTALK_APP_SECRET` 等配置时阻断发布；但数据中心只能登记非敏感资料，管理员没有安全的应用内配置入口。结果是老板可以看到“缺少配置”，却不能直接处理，测试、预览和生产还可能读取不同配置。

## 目标

- 最高权限管理员在数据中心通过平台 Logo 进入钉钉或快麦的专属配置页。
- 老板只需填写中文表单并点击“保存并验证”，不接触环境变量、数据库或加密术语。
- 凭据只在服务端加密保存，浏览器永远不能读取已保存明文。
- 钉钉、快麦适配器和环境就绪检查读取同一份公司连接，消除“检查来源”和“运行来源”不一致。
- 保存失败或验证失败不替换当前可用配置。
- 所有配置变更保留不含凭据值的审计记录。

## 非目标

- 不保存验证码、扫码结果、Cookie、个人账号密码或浏览器会话。
- 不在本期实现阿里云 OpenAPI 调用；阿里云只展示真实的“准备接入”状态。
- 不允许普通员工或运营人员维护公司级平台凭据。
- 不提供凭据明文查看或复制功能。
- 不让应用持有 Cloudflare 账号级管理令牌，也不由应用修改 Cloudflare 项目设置。

## 用户与权限

- 已登录员工：可查看平台名称、连接状态、最近验证时间和安全错误摘要。
- 最高权限管理员：可新增、更换、验证、停用平台连接。
- 只读身份：只能查看状态。
- 服务端适配器：只能按平台读取已启用凭据，不具备列出明文或跨平台读取能力。

## 当前流程

1. 老板在说明书中看到环境阻断和变量名。
2. 数据中心与内部平台资料都拒绝保存凭据。
3. 只能由开发人员进入 Cloudflare 后台分别维护环境配置。
4. 平台适配器只读环境变量；预览和生产容易发生配置漂移。

## 目标流程

1. 老板进入“数据中心 → 平台连接”。
2. 页面展示钉钉、快麦、阿里云的 Logo、用途和真实状态。
3. 点击钉钉或快麦进入专属配置页。
4. 已保存字段显示“已配置”，输入框保持空白；只填写需要替换的字段。
5. 点击“保存并验证”，服务端合并当前配置、校验必填字段并执行只读连接测试。
6. 验证通过后才加密保存并切换为当前连接；页面显示验证时间与操作者。
7. 环境阻断面板重新检查后，把对应能力标记为已就绪。

## 业务规则

- 公司连接不区分生产和预览记录；所有绑定同一生产 D1 的运行环境读取同一份当前连接。本地开发仍通过服务端生产网关或本地环境回退，不直接下载密文。
- D1 当前连接优先于旧 Cloudflare 环境变量；停用当前连接后自动回退旧环境变量。
- `PLATFORM_CREDENTIAL_MASTER_KEY` 只保存在 Cloudflare Secret；缺少或格式错误时禁止保存和解密。
- 每个平台只有一个当前公司连接，使用整数版本做乐观锁。
- 保存请求中的空字段表示“保持原值”，不能清空必填字段。
- 候选配置必须先通过只读连接测试，失败时不更新当前密文和版本。
- 钉钉连接测试只申请应用 access token；快麦连接测试只查询平台时间。
- 任何响应、日志、审计和错误都不能包含凭据、请求 URL 中的密钥参数或完整平台响应。
- 停用只改变启用状态并保留密文，便于回退；重新启用仍需验证。

## 数据定义

### 当前连接

- `platform_id`：注册表平台 ID，首期为 `dingtalk`、`kuaimai`。
- `ciphertext`、`iv`、`algorithm`、`key_version`：AES-256-GCM 加密结果与密钥版本。
- `configured_fields`：只记录已配置字段名，用于遮罩和就绪检查，不记录值。
- `version`：乐观锁版本。
- `enabled`：是否优先供适配器使用。
- `verified_at`、`verified_by`：最近一次通过只读连接测试的时间和操作者。
- `updated_at`、`updated_by`：最近保存时间和操作者。

### 审计

- `platform_id`、`action`、`changed_fields`、`result`、`request_id`、`actor_id`、`actor_name`、`created_at`。
- `changed_fields` 只包含字段名；审计表禁止存储字段值和外部原始响应。

## 异常与边界

- 没有 D1、没有主密钥、密文损坏：返回安全中文错误，保持旧环境变量回退。
- 版本冲突：返回 409，页面刷新当前状态并要求重新提交。
- 平台超时、限流或权限不足：返回可重试标记与安全摘要，不保存候选配置。
- 首次配置失败：保持“未连接”，页面保留输入内容供用户修正，但密码字段不能写入浏览器存储。
- 已有配置更新失败：继续使用旧版本，页面明确显示“原连接未受影响”。
- 普通员工直接调用写接口：服务端返回 403。
- 阿里云等未接入平台：按钮禁用并说明当前缺少真实适配器。

## 验收标准

- 最高权限管理员能在数据中心配置钉钉应用凭证和应用密钥，成功后页面只显示两个字段均已配置。
- 钉钉配置成功后，不设置旧环境变量也能完成登录配置和服务端 access token 获取。
- 快麦配置成功后，状态、拉取和刷新接口读取同一份保险箱配置。
- 普通员工 GET 只获得脱敏状态，PUT 返回 403。
- 任意 API 响应、D1 审计、前端状态和控制台日志均不出现测试凭据值。
- 验证失败后，旧配置版本、密文和适配器行为保持不变。
- 环境就绪检查把保险箱已配置字段视为满足对应配置，并在缺少主密钥时保持阻断。
- 页面覆盖加载、空数据、错误、无权限、禁用、保存中、验证失败和成功状态。
- 笔记本、窄屏和钉钉 WebView 中可用键盘完成选择、填写、保存和返回。

## 上线与回滚

- 上线前应用 `migrations/0003_platform_credentials.sql`，并在生产与预览设置同一个 32 字节 Base64URL 主密钥。
- 首次上线不迁移旧环境变量值；旧变量继续作为回退，管理员可在页面重新录入并验证。
- 上线后分别验证钉钉、快麦的生产就绪与只读连接。
- 回滚应用版本时保留新表；旧代码继续读取旧环境变量。
- 若新连接异常，可在页面停用当前连接或回滚应用版本，不删除密文和审计。
