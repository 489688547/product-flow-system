# 品牌内容协同 PRD

## 文档状态

- 状态：首版已实现，本地验收通过，待部署与外部接入验收
- 业务负责人：品牌内容负责人
- 产品负责人：经营执行平台负责人
- 最近更新：2026-07-18

## 背景与问题

剪辑、编导和运营目前分别在钉钉表格、投放后台、NAS 目录和即时沟通中完成内容生产。生产记录能够说明“做了什么”，投放后台能够说明“花了多少钱”，但两者没有稳定的内容主键和责任交接，导致负责人难以持续回答以下问题：

1. 视频已经制作完成，为什么没有发布、没有素材 ID 或没有获得有效测试？
2. 某个产品的新素材表现弱，是上线时间不足、投放分配不足、内容方向不对，还是产品链接本身承接弱？
3. 同一素材在不同账户、投放体系和模式中产生的数据如何去重、归因并与公司总盘对平？
4. 下一轮应给哪个产品补几条素材、复刻什么方向、分配给哪个账户、由谁负责？
5. 信息流 GMV/ROI 与品牌自然内容增长如何分别提升，又如何共同沉淀可复用的内容资产？

历史复盘已经暴露出反复纠错的高风险口径：公司总盘与局部账户小计混用、GMV 与用户实付混用、产品链接消耗与素材归因消耗混用、乘方与全域漏算、商品与直播目标筛选错误、近期上线素材与成熟素材混评。新 App 必须把这些校验固化为系统约束，而不是依赖分析人员记忆。

## 目标

1. 每条内容从 Brief 创建起获得唯一 `contentId`，并稳定关联产品、人员、NAS 文件、发布记录和平台素材 ID。
2. 打通“需求 → 脚本 → 剪辑 → 审核 → 发布 → 学习期 → 有效测试 → 放量/替换”的责任闭环。
3. 从数据中心读取标准化投放与自然内容数据；数据未对平时显示覆盖率和缺口，不输出伪精确结论。
4. 将产品链接基准、素材表现、上线时长和投放分配同时纳入判断，区分“学习中、未获测试、内容偏弱、产品承接弱、分配错误”。
5. 为负责人生成可执行的补充决策：产品、数量、方向、账户、责任人、截止时间和复盘时间。
6. 团队效能同时衡量按时交付、有效上线、有效测试、进入第二梯队和内容效率，不以单一 ROI 或产量直接评价个人。
7. 自然内容账号单独展示播放、完播、互动、收藏、涨粉等品牌指标，不与付费投放 ROI 混为同一排名。

## 非目标

- 不在品牌内容协同 App 中直接抓取抖音、巨量千川、快手或小红书页面；采集、清洗、口径与历史回填由数据中心负责。
- 不替代公司 NAS，不把原视频重复上传到 D1、R2 或浏览器存储。
- 不保存 NAS、投放平台或内容平台的账号密码、Cookie、验证码和浏览器会话。
- 不代替剪辑软件、脚本编辑器或平台发布后台。
- 不自动调整广告预算、创建投放计划或发布内容。
- 不根据一次低消耗、短学习期或单一 ROI 自动奖罚人员。
- 首版不建设跨公司、多租户或对外客户协作能力。

## 用户与权限

### 品牌内容负责人

查看全部产品、账户、素材和团队队列；创建内容 Brief；确认补充决策；维护内容规则；查看团队效能。不能绕过数据质量状态把“学习中”强制标成“内容失败”。

### 编导

查看分配给自己及被授权产品的 Brief、脚本、反馈和复盘；维护脚本版本、内容方向和交付状态；不能修改投放事实数据。

### 剪辑

查看分配任务和 NAS 资产；登记剪辑版本、封面、完成时间和送审记录；不能修改发布记录或投放事实数据。

### 运营

维护平台发布记录、账户、产品链接、发布时间和素材 ID；处理缺 ID、未发布、未获测试和错误分配；不能修改编导、剪辑已经确认的历史交付记录。

### 管理层

查看内容经营结果、补充决策、数据覆盖和团队效能，不默认获得内容规则或任务编辑权限。

权限基于现有钉钉组织身份。前端禁用状态必须解释权限来源，服务器仍需校验页面写入、内容范围和操作角色。

## 当前流程

1. 品牌或运营通过聊天、表格安排内容。
2. 编导和剪辑在 NAS 中保存文件，通过文件名或消息说明版本。
3. 运营发布后人工填写素材 ID，部分记录缺失或重复。
4. 投放数据在多个账户、投放体系和目标页面分别查看或导出。
5. 负责人临时把素材台账、产品链接和投放数据拼接，手工判断人员和产品表现。
6. 口径错误或上线时长遗漏后重新统计，补素材任务再通过聊天分配。

## 目标流程

1. 负责人基于产品经营与内容缺口创建 Brief，系统生成 `contentId` 并指定主编导、主剪辑、运营和截止时间。
2. 编导提交脚本，剪辑登记 NAS 文件与版本，负责人完成审核；每次状态变化记录操作人和时间。
3. 运营发布内容，填写平台、账号、产品链接、发布时间和素材 ID；同一内容可产生多个平台发布记录和多个投放素材 ID。
4. 数据中心按日返回公司/账户/模式/产品链接/素材层事实、对平状态和数据截止时间。
5. 系统按发布时间计算学习期，把成熟且测试不足的素材路由给运营，把成熟且获得有效测试但表现偏弱的素材路由给内容复盘。
6. 负责人确认补充决策后生成下一轮 Brief；原素材、复刻方向、人员和复盘日期保持可追溯关系。
7. 自然内容和付费投放分别复盘，在产品和内容主档中沉淀可复用的钩子、卖点、结构和人群标签。

## 业务规则

### 主档与发布记录

- `ContentItem` 表示一次内容创作，创建后生成不可复用的 `contentId`。
- 一个 `ContentItem` 只指定一名主编导、一名主剪辑和一名主运营，可登记协作者但不重复计算团队总量。
- 一个 `ContentItem` 可以关联多个 `Publication`，用于同一内容跨平台、跨账号或重新发布。
- 一个 `Publication` 可以关联多个平台素材 ID；素材 ID 在同一平台和账户范围内必须唯一。
- 产品使用现有 `productId`；无法映射的产品记录进入数据问题队列，不自动猜测。

### 状态分离

生产状态独立维护：`brief → scripting → editing → reviewing → ready → published → archived`。

数据状态独立计算：

- `missing_id`：已发布但缺少平台素材 ID。
- `waiting_sync`：已登记 ID，但数据中心尚未返回数据。
- `learning`：自发布时间起未满平台配置的完整学习期；抖音首版默认 3 个完整自然日。
- `untested`：已过学习期，但累计消耗未达到有效测试阈值；抖音首版默认 50 元，可按平台配置。
- `tested`：已过学习期且达到有效测试阈值。
- `inconsistent`：重复 ID、总盘未对平、产品链接未映射或来源口径冲突。

生产状态不得被投放表现自动回退；数据状态不得由人工直接覆盖。负责人可以记录人工判断，但必须保留原因和数据版本。

### 投放层级与对平

抖音首版保留下列完整维度：

`账户 × 乘方/全域 × 推商品/推直播 × 商品自选/全店托管 × 放量/控成本 × 产品链接 × 素材 ID`

- 公司、账户、投放体系和目标小计由数据中心提供。
- 全域、乘方、GMV、用户实付、产品链接消耗和素材归因消耗分别展示，不互相替代。
- 同一素材出现在多个模式时，内容数量只计一次，各模式消耗与 GMV 分别保留并按约定汇总。
- 数据覆盖率未达到 100% 或对平差额超过数据中心阈值时，页面显示差额和影响范围；系统不得输出“确认是内容问题”。

### 判断顺序

负责人判断固定按以下顺序执行：

1. 检查产品、发布时间、平台、账户、投放目标和素材 ID 是否完整。
2. 检查数据中心截止时间、对平状态和覆盖率。
3. 检查是否仍在学习期。
4. 检查成熟素材是否获得有效测试。
5. 对比同产品链接、同账户和同龄素材表现。
6. 区分投放分配、内容方向、产品承接和数据缺失。
7. 形成“继续观察、追加测试、制作变体、新做方向、暂停复刻、修复数据”的决策。

系统建议只提供证据与下一步，不自动创建高成本制作任务。负责人确认后才进入内容作战台。

### 团队效能

- 产量按主责任人统计，协作者单独展示，不重复累计团队总量。
- 未发布、缺 ID、学习中和未获有效测试的内容不用于内容能力结论。
- 编导和剪辑至少展示交付数、按时率、上线率、有效测试率、进入第二梯队数量和成熟样本表现。
- 运营至少展示发布及时率、ID 完整率、有效测试覆盖率、数据问题关闭时长和账户分配执行率。
- 人员比较默认限定同产品、同平台、同账户角色和相近上线时间；样本不足时显示“暂不排名”。

### NAS

- 原文件保留在公司绿联 NAS；系统只保存受控共享目录下的相对路径、文件名、大小、修改时间、媒体时长、缩略图引用和版本关系。
- 首版通过公司 Mac 的 SMB 挂载与本地只读索引器读取目录；WebDAV 作为远程或兼容方案。
- MCP 仅作为可选 AI 搜索入口，默认只读且限制在指定素材目录；正式 App 不依赖 MCP 才能运行。
- NAS 离线时保留已有索引和业务记录，禁止把索引缺失解释为文件已删除。

## 数据定义

### ContentItem

- `id`: 稳定内容主键。
- `productId`: 现有产品档案 ID。
- `purpose`: `product_sale | live_traffic | brand_content`。
- `title`, `brief`, `contentDirection`, `hook`, `sellingPoints[]`。
- `directorId`, `editorId`, `operatorId`, `collaboratorIds[]`。
- `productionStatus`, `dueAt`, `createdAt`, `updatedAt`。
- `sourceContentId`: 复刻或变体来源。

### ContentAssetVersion

- `id`, `contentId`, `version`, `nasRelativePath`, `fileName`。
- `fileSize`, `durationSeconds`, `modifiedAt`, `thumbnailRef`。
- `reviewStatus`, `reviewComment`, `createdBy`, `createdAt`。

### Publication

- `id`, `contentId`, `platform`, `accountId`, `productLinkId`。
- `publishedAt`, `platformContentId`, `materialIds[]`, `url`。
- `publishingPurpose`, `createdBy`, `createdAt`, `updatedAt`。

### ContentPerformanceSnapshot

由数据中心提供并保持只读：

- `asOfDate`, `platform`, `accountId`, `deliverySystem`, `marketingGoal`。
- `selectionMode`, `deliveryStrategy`, `productLinkId`, `materialId`。
- `spend`, `gmv`, `actualPayment`, `impressions`, `clicks`, `orders`, `roi`。
- `contentViews`, `completionRate`, `interactions`, `favorites`, `followersGained`。
- `reconciliationStatus`, `coverageRate`, `metricVersion`, `sourceUpdatedAt`。

付费投放字段和自然内容字段允许部分为空；无数据使用 `null`，不得使用零伪装完整数据。

## 异常与边界

- 数据中心不可用：生产协同继续可用，指标显示上次成功时间，暂停生成新的效果判断。
- 数据未对平：显示差额、缺失账户和影响指标，相关内容进入数据问题队列。
- 重复素材 ID：阻断确认归因，保留冲突记录并要求运营处理。
- NAS 离线：显示“索引暂不可用”，允许复制已有路径，不删除版本记录。
- 文件被移动：索引器按稳定文件指纹尝试匹配；无法匹配时进入“文件失联”，不自动改成已删除。
- 发布后修改素材 ID：保留变更历史，下一次同步重新归因，不覆盖旧版本审计。
- 同一内容跨平台：每个平台独立计算学习期和数据状态。
- 快手、小红书首版数据：通过数据中心标准导入读取；缺少平台字段时明确显示覆盖范围。
- 历史内容：允许批量导入内容主档、人员、发布时间、NAS 路径和素材 ID，重复项先进入预览，不直接写入。

## 验收标准

1. 创建 Brief 后生成唯一内容 ID，并能指定产品、主编导、主剪辑、主运营和截止时间。
2. 编导、剪辑和运营只能执行其权限范围内的状态动作；服务端拒绝越权写入。
3. 用户可以从左侧“品牌内容协同”分组进入九个页面；右侧不出现重复的顶部二级导航。
4. 素材资产页可查看 NAS 相对路径、版本、缩略图状态和文件元数据；NAS 离线时仍能查看已索引记录。
5. 同一内容可以建立多个发布记录；重复平台素材 ID 会进入数据问题队列且不重复计算。
6. 抖音发布时间不足 3 个完整日的素材显示“学习中”，不会进入人员效率排名或内容失败判断。
7. 成熟但消耗不足默认阈值的素材显示“未获有效测试”，责任动作指向运营分配，不指向编导或剪辑能力。
8. 数据中心未对平时，投放复盘页显示覆盖率、差额和受影响范围，并禁用“确认内容问题”。
9. 产品内容作战表同屏展示产品链接基准、新素材表现、上线时长、数据状态和下一步动作。
10. 补充决策必须包含产品、数量、方向、账户、负责人、截止时间和复盘时间，确认后生成可跟踪任务。
11. 团队效能页不重复累计协作者产量，样本不足时显示“暂不排名”。
12. 快手和小红书在缺少自动同步时仍可展示数据中心导入的标准数据及数据截止时间。

## 上线与回滚

- 首版使用独立路由与独立数据表，不修改已有产品全周期状态文档。
- App 注册、左侧导航和 API 通过功能开关控制；关闭后现有经营、产品、供应链和数据中心页面不受影响。
- 历史导入先预览、再确认；失败批次不产生部分写入。
- 数据中心接口不可用时回退到只显示协同与上次成功快照，不回退到浏览器直接抓取平台。
- NAS 索引器停止时不影响云端业务记录；重新连接后执行增量扫描。
- 回滚时隐藏 App 入口并停止索引与同步任务，保留品牌内容数据表供恢复，不删除历史记录。
