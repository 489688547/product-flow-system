# 数据中心 App PRD

## 文档状态

- 状态：开发中
- 负责人：总经办 / 数据管理员
- 最近更新：2026-07-18

## 背景与问题

公司已有产品全周期和供应链管理两个业务 App，但销售、投放和店铺数据仍分散在快麦、巨量引擎及各电商后台。不同页面重复处理时间口径、平台映射和数据新鲜度，其他 App 也无法稳定复用同一份经营事实。

## 目标

1. 增加与供应链管理、产品全周期同级的数据中心 App。
2. 以现有 `product_sales_daily` 为唯一销售日事实，提供统一总览、分析和查询契约。
3. 固定销售统计为订单创建时间、`Asia/Shanghai`、默认截止昨天。
4. 常规经营分析默认排除“其它”等未识别平台，同时保留原始事实。
5. 建立数据源、用户洞察、指标、质量、同步和消费关系的管理入口，为受控浏览器采集器提供稳定边界。

## 非目标

- 本期不自动控制 Chrome、不下载平台文件、不执行历史回填。
- 本期不接入巨量千川真实投放事实；未接入指标明确显示无数据。
- 不保存账号密码、验证码、Cookie、Token 或浏览器会话。
- 不改变现有 `/api/sales` 调用方，不复制销售事实表。
- 不部署生产、不写入真实平台凭证。

## 用户与权限

- 总经办：查看全部数据，维护数据源、设置和质量规则。
- 运营部：查看经营和产品分析，维护被授权的数据源与设置。
- 财务部：查看销售、退款、成本和毛利，只读数据源配置。
- 产品部、供应链及采购：查看授权经营事实与数据质量，只读配置。
- 其他部门：默认不显示数据中心导航。
- 服务端对 GET、POST 分别执行查看和编辑权限，`readonly` 会话不能写入。

## 当前流程

1. 运营从快麦或店铺后台导出数据。
2. 销售数据通过设置页导入或旧快麦链路写入 `product_sales_daily`。
3. 产品页自行读取和计算销售结果。
4. 管理层需要跨平台或跨产品观察时再次导出和整理。

## 目标流程

1. 用户从业务 Apps 或主侧边栏进入数据中心。
2. 数据总览默认展示本月截至昨天的经营事实及新鲜度。
3. 数据分析按时间、平台、产品和 SKU 下钻。
4. 数据接入登记平台、账户标签、后台地址、采集方式和状态，不登记凭证。
5. 指标、质量、同步记录和数据服务页面解释口径、异常和消费关系。
6. 产品全周期及其他 App 继续使用现有销售事实，并逐步迁移到稳定的数据中心查询契约。

## 业务规则

1. 主导航顺序为公司经营、供应链管理、产品全周期、数据中心、平台。
2. 数据中心 App ID 为 `data-center`，入口为 `data-overview`；一级导航包含 `data-insights` 用户洞察。
3. 销售业务日期使用订单创建时间 `create_time`，时区固定 `Asia/Shanghai`。
4. 默认查询区间为昨天所在月份 1 日至昨天；今天不进入正式经营统计。
5. `其它`、`其他`、`未知`、`未知平台` 和空平台不进入常规分析。
6. 缺失指标显示“无数据”或 `—`，不得以 0 伪装完整数据。
7. 数据源记录只接受平台、名称、账户标签、HTTPS 后台地址、采集方式、计划时间和状态。
8. 变更数据源或设置必须记录操作人和时间。

## 数据定义

- 销售事实：复用 `product_sales_daily` 的 69 码、日期、平台、销量、销售额、净销售额、毛利、退款、成本及发货前后退款。
- 数据中心元数据：数据源、执行器、同步批次、来源文件元数据、维度映射、指标定义、质量问题、消费订阅和审计记录。
- 新鲜度：最后数据日期达到查询结束日且未超过设置阈值时为正常。
- 覆盖率：来源、产品映射和日期覆盖完整度；缺少来源时不返回伪造完整值。

## 异常与边界

- 无 D1：生产 API 返回 501；本地预览的元数据保存到本机 JSON，销售数据回退浏览器 IndexedDB。
- 无销售数据：页面显示可执行空状态，不展示虚假趋势。
- 数据过期：保留上次成功数据并提示最后数据日期。
- 无权限：隐藏导航之外，服务端仍返回 403。
- 非法日期、超长区间或非 HTTPS 地址：返回稳定中文校验错误。
- 字段变化或登录失效：本期只展示预定义状态，不伪造同步成功。

## 验收标准

1. 数据中心八个入口全部位于产品全周期之后、平台之前。
2. 业务 Apps 显示产品全周期、供应链管理、数据中心三项。
3. 2026-07-18 打开默认总览时，请求范围为 2026-07-01 至 2026-07-17。
4. 同一范围存在抖音和“其它”数据时，总览只计算抖音，原 `product_sales_daily` 行不被删除。
5. 巨量数据未接入时广告消耗和 ROI 显示 `—` 与明确未接入说明。
6. 数据源表单不存在密码、Cookie、Token 和验证码字段。
7. 总经办和运营可保存配置，财务及产品只读，未授权部门 API 返回 403。
8. `npm run lint`、治理检查、集成检查、完整测试和构建全部通过。

## 上线与回滚

- 本期为新增路由、表和 API，不替换 `/api/sales`，可通过移除导航注册回滚前端入口。
- 元数据表为新增表；回滚时停止读取即可，不删除已写记录。
- 销售查询只读现有事实表，回滚不涉及销售数据迁移。
- 未经单独确认不执行 Cloudflare 部署。
