# 数据中心 App PRD

## 文档状态

- 状态：已实现并发布
- 负责人：总经办 / 数据管理员
- 最近更新：2026-07-20
- 替代规则：本文替代 2026-07-18 版本中“不保存账号凭证、不自动控制浏览器”的限制

## 背景与问题

公司已有产品全周期、供应链管理等业务 App，但销售、店铺、投放和内部系统数据分散在快麦 ERP、抖音电商、巨量引擎、快手、淘系、拼多多、小红书、京东、NAS 和其他后台。不同 App 重复处理登录、导出、时间口径、平台映射和数据新鲜度，管理层无法确认各页面是否基于同一份经营事实。

快麦 API 不能覆盖全部所需数据，因此不再作为唯一或优先主链路。数据中心需要同时支持开放 API、浏览器已登录页面、文件导出和内部系统凭证，并向其他 App 提供稳定数据库接口。

## 目标

1. 将数据中心作为独立业务 App，位于产品全周期之后、平台之前。
2. 数据接入采用内置平台 Logo 目录；点击平台后打开专属配置弹窗，不再使用一个通用登记表单。
3. 在 D1 建立共享的加密凭证保险箱，支持平台账号、API 密钥、浏览器会话和内部系统账号。
4. 由固定公司 Mac 上的本地采集器按计划使用 API、文件导出或已登录 Chrome 页面采集数据。
5. 每天 `07:30` 拉取截止昨天的数据，并支持按月拆分的全部历史回填。
6. 销售统计固定使用订单创建时间 `create_time`、`Asia/Shanghai`，常规经营判断排除“其它”。
7. 其他业务 App 只读取数据中心数据库/API，不直接依赖外部平台、浏览器或凭证。
8. 准确呈现“可配置、等待验证、已接通、同步异常”等状态，不能把平台卡片存在误报为真实接入。

## 非目标

- 不恢复独立“数据分析”页面，也不在数据中心罗列全部原始销售行。
- 不代替用户在外部平台执行下单、退款、调价、投放或其他业务写操作。
- 不绕过平台验证码、扫码、滑块、设备风控或服务条款。
- 不把 OTP、短信验证码、二维码内容或人工验证答案持久化。
- 不在日志、Git、URL、浏览器本地存储或普通 API 响应中暴露明文凭证。
- 不因本期新增能力改变现有 `/api/sales` 调用方或删除历史事实。

## 环境与兼容边界

- Cloudflare Pages + D1 是共享口径的正式运行边界；本地 Node 服务不把 JSON 文件伪装成公司共享定义或计算结果。
- 本地缺少 D1 时，`/api/platform/v1/data-standards*` 返回结构化 501；页面只展示 11 项内置只读目录，新增、编辑、计算和重算不可用。
- 旧 `metricDefinitions` 只作为迁移兼容读取；浏览器缓存和旧 `/api/data-center` 写入都不能覆盖共享定义。
- 数据总览默认读取共享计算结果。事故回滚只能在构建时显式设置 `VITE_DATA_CENTER_LEGACY_OVERVIEW_ROLLBACK=1`，页面必须显示“兼容回滚口径”；回滚不删除口径版本、结果、任务或审计。
- 生产迁移、备份、结果生成和部署属于独立授权动作；本地验证不能替代 Production 或钉钉 WebView 验收。
- 未经单独授权不部署生产、不迁移真实凭证、不启动生产定时采集。

## 用户与权限

- 总经办平台管理员：维护连接器、加密凭证、采集器、权限、同步计划和一次性迁移；可在再次确认后查看或复制被授权的明文。
- 运营数据管理员：维护获授权的店铺/投放连接器和同步计划；可直接发布运营部负责的数据口径；默认可替换凭证但不可查看已保存明文。
- 财务、供应链：读取授权经营事实、质量和同步状态；可直接发布本部门负责的数据口径，默认不能读取或编辑凭证。
- 产品部：读取授权经营事实、口径、质量和同步状态；默认不能发布口径或读取、编辑凭证。
- 内部系统保险箱授权人：仅查看被明确授予的 NAS、邮箱、财务或政务/SaaS 条目；权限按条目或范围分配。
- 本地采集器：使用独立机器身份和短时任务授权，只能取得当前任务所需的凭证，不能遍历保险箱。
- 其他部门：默认不显示数据中心导航，服务端仍执行权限校验。

页面可见、连接器编辑、凭证替换、凭证查看/复制、采集器取用和权限管理是不同权限；隐藏按钮不是授权边界。

## 用户流程

### 新增平台连接

1. 用户进入“数据接入”，看到内置平台 Logo、能力说明和当前状态。
2. 点击平台卡片，系统根据平台定义打开专属配置弹窗。
3. 用户填写平台对应的店铺/账户名称，并按需补充网页登录或 API 凭据；系统根据已填写字段自动判断接入方式，API 凭据优先于网页登录，均未填写时使用平台支持的文件导出。
4. 密码、密钥、Token、Cookie 或会话信息在服务端加密后写入 D1；列表只返回是否已配置和脱敏提示。
5. 需要短信、扫码、验证码或设备确认时，连接状态进入“等待人工验证”，页面说明在指定 Mac/Chrome 完成操作。
6. 验证完成后用户点击重试，或采集器检测到登录恢复后继续原任务。

### 管理数据口径

1. 用户进入“数据口径”，按销售和货流查看已发布定义、当前版本、覆盖状态和责任部门。
2. 有权限的责任部门使用受控公式构建器新增口径或编辑现有口径；页面不接受任意 SQL、JavaScript 或代码表达式。
3. 保存前系统校验字段权限、单位、循环依赖、除零、日期范围和样本结果；保存即发布新版本并要求设置生效日期。
4. 历史查询使用当时生效版本，修改不会静默重算历史。用户需要重算时明确选择日期范围，成功批次才切换为当前结果。
5. “删除”只归档口径；历史版本、结果、依赖和审计继续保留。
6. 数据总览只读取已发布口径结果；结果缺失或不完整时展示覆盖原因，不回退到组件内的另一套公式。

### 每日采集

1. Cloudflare 在 `07:30` 创建截止昨天的同步任务；销售目标时间为前一天 `23:59:59`。
2. 固定公司 Mac 的采集器领取短时任务和最小范围凭证。
3. API 型连接器直接调用服务端适配器；网页型连接器优先使用平台导出，其次读取已确认的页面数据。
4. 数据校验、标准化、去重后写入 D1；历史成功数据仅在新批次完整校验后原子替换。
5. 单一来源失败不阻塞其他来源；页面保留上次成功时间、数据日期和安全错误摘要。
6. 其他 App 继续读取最近成功的数据库事实，不感知登录或采集细节。

### 历史回填

1. 按平台可提供的最早日期至昨天建立月份任务。
2. 每月任务可以暂停、继续、重试或重跑；幂等键避免重复累计。
3. 回填与日常增量使用同一标准化和质量规则。
4. 页面展示月份、来源、覆盖日期、行数、异常数和最后结果，不展示原始凭证或敏感响应。

### 钉钉账密表迁移

1. 一次性读取钉钉 AI 表格“账密表”的 `总表`（表 ID `hERWDMS`），忽略 `总表 (副本)`（表 ID `pvMRuGR`）。
2. 在内存中清洗空行、分类、去重并映射到连接器或内部保险箱。
3. 操作人确认导入预览后，敏感字段直接加密写入；日志只记录数量、目标类型、冲突和操作者。
4. 导入不持续同步钉钉表，也不在仓库生成含明文的中间文件。

### 内部系统保险箱

- NAS：地点、设备/服务名、协议、地址、端口、共享目录、用户名、密码、只读要求、负责人。
- 邮箱：邮箱地址、服务商、网页登录地址或 IMAP/SMTP、用户名、密码/应用专用密码、二次验证说明。
- 财务系统：系统名称、公司主体、网址、账号、密码、证书/Key 引用、负责人、用途与复核日期。
- 政务/SaaS：系统名称、组织/租户、网址、登录标识、密码、验证方式、负责人、用途与复核日期。
- 自定义内部系统：由管理员选择受控字段类型，不允许自由 JSON 绕过敏感字段分类和审计。

内部保险箱与数据连接器共享加密存储和权限服务，但在信息架构、列表和授权范围中分开，避免把内部基础设施误当成经营数据源。

## 业务规则

1. 主导航顺序为公司经营、供应链管理、产品全周期、数据中心、平台。
2. 数据中心 App ID 为 `data-center`，入口为 `data-overview`，共七个入口且没有“数据分析”。
3. 销售业务日期来自订单创建时间 `create_time`，时区固定 `Asia/Shanghai`。
4. 默认查询区间为昨天所在月份 1 日至昨天；今天不进入正式经营统计。
5. `其它`、`其他`、`未知`、`未知平台` 和空平台不进入常规分析，但原始值保留用于质量审计。
6. 缺失指标显示“无数据”或 `—`，不得以 0 伪装完整数据。
7. 默认每日计划为 `07:30`；销售重拉最近 7 天，投放默认重拉最近 3 天，可由连接器策略调整。
8. 失败任务不得覆盖最近一次成功事实；重复批次必须幂等。
9. OTP、短信验证码、扫码内容和滑块结果永不持久化；密码、密钥、Token、Cookie 和可复用会话只允许加密持久化。
10. 普通读取只返回 `hasSecret`、脱敏尾部、更新时间和健康状态；明文必须走独立受控动作。
11. 连接器目录中的“可配置”不代表“已接入”；只有通过真实连通性验证且有成功同步证据才能显示健康。
12. 所有凭证创建、替换、查看、复制、采集器取用、失败解密和权限变更均写追加式审计。
13. 口径使用稳定 `metricCode`；修改生成带生效日期的新版本，已发布版本不可覆盖。
14. 口径公式只能使用白名单字段、指标、聚合、过滤和运算符；缺失数据不补零。
15. 新增、发布、归档、预览失败、重算和结果批次切换均追加审计。
16. 责任部门可直接发布本部门口径；总经办可维护全部口径和责任部门；删除统一为归档。

## 数据定义

- 销售事实：继续复用 `product_sales_daily` 的 69 码、日期、平台、销量、销售额、净销售额、毛利、退款、成本及发货前后退款。
- 连接器定义：平台、Logo、支持能力、授权方式、字段 schema、验证说明、数据集和适配器状态；由代码版本管理，不含实例凭证。
- 连接器实例：店铺/账户名称、自动推断的采集方式、后台地址、计划、执行器、状态、最后成功时间、凭证引用，以及由登录会话生成的创建/更新身份审计。
- 加密凭证：用途和权限元数据、密文、随机 IV、算法、密钥版本、更新时间；敏感 payload 不拆成明文字段。
- 运行任务：计划范围、执行器、状态、尝试次数、数据覆盖、幂等键、错误码和最后心跳。
- 内部保险箱：分类、主体、位置/系统、用途、负责人、复核日期和加密凭证引用。
- 新鲜度：最后成功数据日期达到查询结束日且未超过阈值时为正常。
- 覆盖率：来源、产品映射和日期覆盖完整度；缺少来源时不返回伪造完整值。
- 数据口径：稳定代码、分类、责任部门、单位、周期、状态和当前版本。
- 口径版本：生效日期、展示公式、受控公式 AST、字段与指标依赖、数据来源和发布身份；发布后不可覆盖。
- 口径结果：口径及版本、周期、允许维度、数值、覆盖率、可信等级、估算标识、数据截止时间和计算批次。
- 口径计算任务：预览之外的计算或重算范围、目标版本、状态、进度、幂等键和安全错误。

## 状态模型

- `unconfigured`：内置连接器存在，但尚未建立实例。
- `pending_validation`：配置已保存，尚未完成首次真实验证。
- `waiting_verification`：需要短信、扫码、验证码、滑块或设备确认。
- `running`：正在采集、转换或写入。
- `healthy`：最近计划成功且数据覆盖到目标日期。
- `stale`：超过计划时间仍未覆盖目标日期。
- `login_required`：可复用登录状态失效，需要重新登录。
- `schema_changed`：页面、导出列或响应结构变化。
- `failed`：网络、下载、解析、映射、解密或写入失败。
- `disabled`：人工暂停。

`waiting_verification` 不算失败；完成验证后继续同一业务任务。任何非成功状态均保留上次成功事实。

## 异常与边界

- 无 D1 或无加密主密钥：禁止保存凭证，页面显示环境缺失；只读经营数据可按已有降级规则继续。
- D1 容量接近阈值：先告警和统计增长，再决定扩容或归档；不得静默丢弃标准事实或凭证。
- 采集器离线：任务保持等待，显示设备最后心跳；不把未执行误报为成功。
- 登录或风控：进入等待人工验证/需要登录，不自动循环提交验证码。
- 平台改版：进入 `schema_changed`，保留上次成功数据并阻止不完整新批次覆盖。
- 解密失败或密钥版本缺失：不返回密文细节，记录安全审计并要求管理员处理。
- 无权限：页面隐藏之外，API 返回 403；通过 ID 猜测不能读取条目元数据或明文。
- 一次性导入冲突：保留来源行指纹和冲突数量，必须人工选择合并、跳过或新建，不覆盖现有凭证。
- 公式非法：未知字段、任意代码、单位冲突、循环依赖、除零缺陷或超限查询在发布前阻止。
- 口径数据未覆盖：定义可以发布，但结果保持空值并说明缺失来源，不生成模拟值。
- 历史重算失败：保留当前有效批次和旧结果，不切换消费者读取源。

## 验收标准

1. 数据接入页展示八个内置经营连接器及 Logo，点击任一连接器打开对应字段弹窗；页面不存在通用新增数据源表单。
2. 巨量引擎与巨量千川只有一个连接器入口，配置中可区分账户类型和数据集。
3. 内部系统保险箱支持 NAS、邮箱、财务、政务/SaaS 和受控自定义类型，并与经营连接器分区。
4. 经营连接器弹窗使用平台对应的店铺/账户名称区分实例，不要求填写公司主体、负责人或手工选择接入方式；创建人和更新人由服务端登录会话自动留痕，客户端伪造字段会被拒绝。
5. 密码、密钥、Token、Cookie 或会话保存后，D1 不含对应明文，普通 GET 不返回明文。
6. OTP、短信验证码、二维码或滑块结果不会写入 D1、日志、本地存储或审计详情。
7. 未完成真实验证的连接器显示待验证而非健康；验证码流程显示等待人工验证并可恢复。
8. 本地采集器只能用短时任务授权取得单个任务所需凭证，取用行为可审计。
9. 每日任务默认 `07:30`、数据截止昨天；销售写入和报表使用 `create_time`、`Asia/Shanghai`，默认排除“其它”。
10. 同步失败保留最近成功事实，重复运行不重复累计，历史回填可按月暂停和重跑。
11. 钉钉账密表导入只读取 `总表`，忽略副本；导入日志不含明文和原始敏感响应。
12. 财务、产品、供应链和普通运营不能通过普通页面或 API 查看已保存明文；越权请求返回 403。
13. 数据中心仍只有七个左侧入口，不恢复数据分析 Tab。
14. UI 覆盖键盘、焦点、加载、空、错误、禁用、等待验证、过期、响应式和钉钉 WebView 状态。
15. “指标管理”统一改为“数据口径”，保留 `data-metrics` 路由兼容历史链接。
16. 首批五项销售和六项货流口径可查看；净销售额、毛利、销量、退款率和毛利率均由共享口径结果驱动。
17. 授权责任部门可以新增、直接发布本部门新版本并归档；越权写入由服务端返回 403。
18. 公式构建器阻止任意代码、未知字段、循环依赖、单位冲突和除零缺陷，并使用同一执行器进行预览和正式计算。
19. 新版本按生效日期使用；历史不静默重算，显式重算成功后才切换有效批次。
20. 归档不删除历史；每个结果返回口径版本、覆盖率、可信等级、估算标识和数据截止时间。
21. lint、治理、集成、环境能力、完整测试和构建全部通过。

## 发布、迁移与回滚

- 实施先新增表、Secret 和 API，再启用 UI；迁移可重复执行，不修改现有销售事实。
- 首次生产启用前检查 D1 容量、创建数据库备份、配置 `PLATFORM_CREDENTIAL_MASTER_KEY` Cloudflare Secret，并验证只读环境就绪；旧名 `DATA_CREDENTIAL_MASTER_KEY` 只保留服务端兼容读取。
- 钉钉账密表只在授权后的迁移窗口执行；先预览数量与分类，再确认加密写入。
- 可通过停用采集器和连接器入口回滚；已加密凭证与审计记录保留，不自动删除。
- 密钥轮换先增加新版本、迁移密文并验证，再撤销旧密钥；不能通过覆盖 Secret 直接使旧数据不可解密。
- 未经单独确认不执行生产部署、数据库迁移、真实凭证导入或外部平台动作。
- 数据口径先新增向后兼容表和共享 API，再迁移现有两项定义、并行对账五项销售结果；差异未解释前不切换总览读取源。
- 口径读取切换使用可回滚开关；回滚只恢复旧读取，不删除新版本、结果、任务或审计。
