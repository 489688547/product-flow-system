# 数据中心 App 功能设计书

## 用户任务

管理层快速判断本月至昨天的销售、利润、退款和数据异常；运营按平台、产品和 SKU 下钻；数据管理员登记来源并处理口径、质量和同步状态。

## 信息层级

- 主信息：净销售额、销量、退款、毛利、截止日期、覆盖率和异常。
- 辅助信息：每日趋势、平台贡献、产品贡献、筛选条件和数据来源。
- 低频信息：指标版本、同步批次、消费订阅、保留期和阈值。

## 页面结构

主侧边栏新增独立“数据中心”分组，位于产品全周期之后。分组包含数据总览、数据分析、数据接入、指标中心、数据质量、同步记录、数据服务和设置。页面不再增加二级侧栏或页内重复导航。

数据总览采用紧凑指标带、趋势与贡献主区、异常列表；数据分析采用固定筛选栏和单一结果表；设置和接入使用无嵌套卡片的矩阵结构。

## 交互流程

1. 进入总览即按昨天计算默认范围并读取销售事实。
2. 点击刷新只重新读取数据，不触发外部平台同步。
3. 分析页修改日期后显式应用；平台、产品和 SKU 筛选即时作用于已读取数据。
4. 新增数据源需填写安全字段并通过 HTTPS 校验，保存后显示“已停用”或既有真实状态。
5. 本地采集器未连接时，“登录后立即重试”保持禁用并解释原因。
6. 设置保存后展示成功反馈；失败保留草稿并显示恢复方式。

## 组件复用

- `PageHeader`：统一页面标题和刷新操作。
- `Button`：主操作、普通操作和禁用原因。
- `DataTable`：平台、产品、指标、质量、同步和订阅列表。
- 现有状态色与骨架规则：保持供应链和经营平台一致。
- Recharts：仅用于每日净销售额和毛利趋势，不增加装饰性图表。

## 新增组件

- `DataCenterAppPage({ section })`：装配 Provider 数据和八个工作区，不包含领域计算。
- `DataOverview`：展示经营摘要、趋势、贡献和异常；只消费标准摘要。
- `DataAnalysis`：管理本页筛选和分组；不直接请求 API。
- 其他六个 Workspace：保留数据中心业务文案和组合逻辑，不进入 `src/ui`。

这些组件目前只有一个真实消费者，决定为“局部保留”；不预测性抽取共享组件。

## 页面状态

- 加载：保持页面结构的三段骨架，无中心旋转图标。
- 空数据：解释尚无可分析销售数据，并指向数据接入或现有销售导入。
- 错误：显示安全中文摘要，保留缓存数据和刷新入口。
- 无权限：导航不可见且 API 返回 403；不在页面内伪装有权限。
- 禁用：按钮同时展示禁用原因。
- 成功：保存后显示最近更新时间和同步状态。
- 过期：保留数值，突出数据截止日期和需要刷新/接入的原因。

## 响应式与钉钉 WebView

- 桌面：指标带和主次两列，表格内部水平滚动。
- 900px：主次区改为一列，指标带两列，来源卡两列。
- 640px：指标、筛选、来源和操作改为一列，页头按钮全宽。
- 390px：不得出现页面级横向溢出；表格滚动仅发生在 DataTable 容器。
- 保持动态视口、安全区、键盘焦点和钉钉 WebView 可用性。

## 交互文案

- 页面说明：`统一经营数据接入、口径、质量、分析与跨 App 服务。`
- 口径：`销售口径：订单创建时间 · Asia/Shanghai · 默认不含其它`
- 空数据：`当前范围没有可分析的正常经营数据。`
- 广告未接入：`巨量数据尚未接入`
- 凭证规则：`不保存密码、Cookie、验证码或浏览器会话。`
- 本地采集禁用原因：`本地浏览器采集器尚未连接。`

## 无障碍

- 导航和分组使用现有语义，选中项保留 `aria-current="page"`。
- 图表提供文字摘要，指标状态不只依赖颜色。
- 表单具有关联 label、错误说明和焦点可见样式。
- 表头不拆字，日期、数值和操作列不异常换行。
- 骨架动画支持 `prefers-reduced-motion`。

## 视觉验收

需要检查数据总览有数据、空数据、过期、错误；数据接入空列表、已登记、表单错误、禁用；同步记录空和失败。宽度为 1440、1280、900、640 和 390px，并单独检查钉钉内嵌视口。
