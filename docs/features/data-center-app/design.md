# 数据中心 App 功能设计书

## 设计原则

数据中心的首页解决“数据是否新鲜且可信”，数据接入解决“平台怎样接进来”，内部保险箱解决“公司系统账号怎样安全托管”。三者不混成一张全量表，也不恢复已删除的数据分析页。

平台卡片表示系统支持配置该平台，不代表已经成功接入。所有状态由连接实例、真实验证和同步证据产生。

## 信息架构

主侧边栏中的数据中心位于产品全周期之后、平台之前，保留七个入口：

1. 数据总览
2. 数据接入
3. 指标中心
4. 数据质量
5. 同步记录
6. 数据服务
7. 设置

“数据接入”内部采用两个并列工作区：经营数据连接器、内部系统保险箱。使用页面内分段切换，不增加主侧边栏入口。

## 数据接入首页

### 经营数据连接器

顶部说明数据范围、每日计划和安全方式。主体为八个内置连接器卡片：抖音电商、巨量引擎、快手生态、淘系、拼多多、小红书生态、京东/京麦、快麦 ERP。

每张卡片包含：

- 平台 Logo、名称和一句能力说明；
- 已建立的店铺/账户数量；
- 支持的采集方式标签，例如 API、网页、文件导出；
- 当前汇总状态与最近成功时间；
- 主操作“添加连接”或“管理连接”。

状态顺序按处理优先级展示：等待人工验证 > 结构变化 > 失败 > 登录失效 > 过期 > 运行中 > 待验证 > 健康 > 未配置。颜色只辅助状态，必须同时有文字和图标。

连接器 Logo 使用本地受控资源并提供文本替代，不远程加载跟踪资源。Logo 尺寸统一，品牌色只用于图标区域，不把整张卡片染成品牌广告。

### 内部系统保险箱

保险箱首页以类型卡片和条目列表组合：NAS、邮箱、财务系统、政务/SaaS、自定义内部系统。顶部明确“加密保存”“明文查看受控”“所有取用留痕”。

列表只显示名称、类型、主体/位置、负责人、复核日期、是否已配置凭证和最近使用时间；不显示用户名全值、密码长度、Token 前缀或可推断敏感值的信息。

NAS 条目突出地点、协议、地址、共享目录和只读策略；杭州与广州可作为独立条目，不以一个共享密码覆盖多个设备。

## 平台专属配置弹窗

### 结构

弹窗使用统一外壳，但表单 schema 来自连接器定义：

1. 标题区：Logo、平台名、能力范围、关闭按钮。
2. 连接身份：使用平台专属标签填写店铺或账户名称；不显示通用连接名称、公司主体或负责人字段。
3. 平台专属字段：通用字段在身份区展示，网页登录和 API 凭据分别分组展示。
4. 自动识别：只读提示当前推断的 API、网页登录或文件导出方式；API 凭据优先，未替换凭据的既有连接保留原方式。
5. 人工验证提示：短信、扫码、验证码、滑块、设备确认的处理说明。
6. 同步设置：数据集、执行器、每天 `07:30` 或自定义计划、历史起始日期。
7. 安全说明：哪些值会加密保存、哪些值永不保存、谁能取用。
8. 固定底栏：取消、保存并验证。保存失败保留用户草稿；关闭前对未保存敏感输入二次确认。

### 平台差异

- 抖音电商：支持邮箱/手机号等登录标识、密码、后台地址和人工验证方式。
- 巨量引擎：统一包含巨量千川，选择广告主/千川账户类型、账户 ID、API 或网页登录，以及投放数据集。
- 快手生态：选择小店或广告账户，填写后台地址和相应登录方式。
- 淘系：选择天猫/淘宝、店铺主体和网页登录方式，明确扫码/设备验证可能性。
- 拼多多：店铺主体、登录标识、后台地址和验证方式。
- 小红书生态：选择店铺或乘风账户，填写主体、后台地址和登录方式。
- 京东/京麦：店铺、京麦账号、后台地址和验证方式。
- 快麦 ERP：同时提供 API 与网页字段；API 显示 App Key、App Secret、Token，网页显示 ERP 地址和账号密码，保存时自动判断接入方式。

字段类型由定义白名单产生，例如 `text`、`url`、`email`、`password`、`secret`、`select`、`date`，不能由客户端提交任意敏感 JSON。

### 对话框行为

- 打开后焦点落在标题或首个错误字段，焦点被限制在弹窗内；Esc 关闭前遵循未保存确认。
- 错误信息与字段关联，保存按钮在请求期间禁用并显示进度。
- 普通编辑时敏感字段显示“已保存”，留空表示保持原值；不会把服务器明文回填到 input。
- “替换凭证”明确进入替换模式；“查看/复制”是单独动作、单独权限和单独审计。
- 390px 宽度时弹窗改为全屏抽屉，底栏固定且不遮挡键盘输入。

## 人工验证流程

验证码不作为配置字段。采集器识别到短信、扫码、验证码、滑块或设备确认时：

1. 当前任务进入 `waiting_verification` 并停止自动重试。
2. 页面和系统通知显示平台、实例、指定设备、等待开始时间和“去处理”说明，但不传递验证码内容。
3. 授权人员在固定 Mac 的 Chrome 完成人工验证。
4. 页面点击“已完成，继续”，或采集器检测到目标业务页恢复后继续原任务。
5. 超时后任务保持可恢复状态；不会覆盖上次成功事实，也不会把等待人工误记为失败。

## 查看与复制凭证

凭证列表和详情默认只返回元数据。具有 `credential:reveal` 权限的用户发起查看或复制时：

1. 页面展示条目、用途和审计提示并要求再次确认；会话不满足近期认证要求时先重新认证。
2. 服务端检查条目范围和身份，解密后仅在本次 HTTPS 响应返回所需字段。
3. 响应设置 `Cache-Control: no-store`，前端只在内存中短暂展示，超时、失焦或关闭后清空。
4. 审计记录条目 ID、字段类别、动作、操作者、用途、时间和结果，不记录明文。

运营数据管理员默认只有替换权限，没有查看权限。内部保险箱的查看权限按条目或范围授权。

## 组件边界

### 共享平台能力

加密凭证保险箱有数据连接器、内部系统保险箱和本地采集器三个真实消费者，抽取为共享平台能力：

- `/api/platform/v1/credential-vault`：受控创建、更新和读取脱敏元数据；
- `/api/platform/v1/credential-vault/:id/reveal`：近期认证后的明文查看/复制；
- `/api/platform/v1/credential-vault/:id/task-grants`：向指定采集器签发最小范围短时取用授权；
- `functions/api/platform/_shared/credentialCrypto.js`：只负责加解密和密钥版本；
- `functions/api/platform/_shared/credentialAuthorization.js`：只负责条目范围与动作授权；
- `functions/api/platform/_shared/credentialAudit.js`：只负责脱敏追加式审计。

API 的完整契约在实施阶段写入 `docs/platform/apis/credential-vault-v1.md` 并加入契约测试。

### 功能局部能力

- `src/features/data-center`：连接器目录、配置弹窗、保险箱页面和任务状态组合。
- `src/domain/dataCenterConnectors.js`：纯连接器定义、字段 schema、状态优先级和配置校验，不依赖 React 或网络。
- `src/state`：只调用内部 API，管理加载、草稿、错误和刷新；不保存明文到 localStorage。
- `functions/api/data-center`：连接实例、同步任务、经营事实和 provider adapter；功能不能直接调用外部平台。

连接器卡片和平台专属字段目前只由数据中心使用，保持功能局部；对话框、按钮和基础表单继续复用 `src/ui`。

## 加密与密钥管理

- Cloudflare Secret `DATA_CREDENTIAL_MASTER_KEY` 保存当前 256 位主密钥，代码和 D1 均不保存主密钥明文。
- 使用 Web Crypto AES-256-GCM；每次写入生成独立随机 96 位 IV，认证附加数据绑定条目 ID、用途和密钥版本。
- D1 保存 `ciphertext`、`iv`、`algorithm`、`key_version` 和非敏感元数据；敏感 payload 在加密前统一序列化。
- 修改敏感字段时生成新 IV 和新密文；不重复使用 nonce。
- 普通 GET 永不解密；服务端仅在受控 reveal 或 task-grant 路径解密，并尽快释放内存引用。
- 日志、异常、审计、指标和追踪禁止记录请求体、解密值、密文或可复用会话。
- 密钥轮换采用新增版本、批量重加密、抽样验证、切换活动版本、撤销旧版本的顺序；回滚窗口内保留旧密钥。

Cloudflare Secret 保护的是“数据库泄漏”风险，不代表平台管理员天然可以查看全部明文；权限、近期认证和审计仍然必须执行。

## 采集器信任边界

固定公司 Mac 上的采集器不持有全库主密钥。首次登记时创建机器身份和公钥，服务器只保存公钥及令牌哈希。

1. 采集器用机器令牌领取已分配任务。
2. 服务器验证机器、任务、实例和有效期，解密所需凭证后用采集器公钥重新封装。
3. 短时 task grant 绑定任务 ID、连接器 ID、字段范围和过期时间，不能用于列出其他条目。
4. 采集器仅在内存中解封，任务结束、失败或超时即清除引用和临时文件。
5. Chrome Profile、下载目录和日志使用公司专用路径；不扫描无关标签页、浏览历史或个人 Profile。
6. 采集器只读外部业务数据，不代表用户执行平台写操作。

## 数据与存储模型

继续使用现有 `data_sources`、`data_runners`、`data_sync_runs` 和 `product_sales_daily`，通过迁移增加或建立以下表：

### `data_connector_instances`

保存连接器 ID、店铺/账户名称、自动推断的采集方式、后台地址、数据集、计划、执行器、凭证引用、状态、最后验证/成功/数据日期、创建与更新时间。创建人和更新人只从认证会话写入，客户端不能提交；旧主体和负责人列保留用于兼容既有数据，但经营连接器弹窗不再编辑。URL 不允许携带用户名、密码或 Token。

### `credential_vault_entries`

保存条目 ID、范围类型、分类、名称、主体引用、密文、IV、算法、密钥版本、凭证 schema 版本、状态、创建/更新人和时间。删除采用禁用/归档，不立即物理删除审计依据。

### `credential_vault_permissions`

保存条目或范围、主体类型、稳定组织身份、允许动作和有效期。允许动作至少分为 `metadata:read`、`credential:replace`、`credential:reveal`、`credential:grant`、`permission:manage`。

### `credential_vault_audit`

追加保存条目 ID、动作、字段类别、操作者/采集器、用途、结果、request ID 和时间；不保存请求体、值、密文或外部原始响应。

### `data_runner_identities`

保存采集器 ID、设备标签、公钥、机器令牌哈希、版本、状态、最后心跳和吊销时间，不保存原始机器令牌。

### `data_runner_task_grants`

保存 grant ID、任务/连接器/采集器、字段范围、令牌哈希、签发/过期/消费/吊销时间；授权短期有效且默认只可消费一次。

### `internal_vault_items`

保存类型、名称、主体/地点、地址、协议、共享目录或租户等非敏感字段、负责人、用途、复核日期、凭证引用和状态。

结构化经营事实、连接配置、加密凭证和审计均可保存在 D1。实施迁移前记录当前数据库大小、预估凭证与同步日志增长；达到容量告警线后再决定分区或归档，不以容量猜测阻止本期设计。

## API 行为

- API 使用公司会话或采集器机器身份；服务端执行动作级授权。
- 写入使用幂等键和字段白名单；更新带版本号，旧版本返回 409。
- 未配置 D1 返回 501；未配置加密主密钥时凭证写入/取用返回 503，非敏感列表仍可读取。
- reveal 和 task grant 限流；连续越权或解密失败进入安全日志。
- 外部平台超时不回传原始响应，返回稳定错误码和是否可重试。
- 连接器验证和同步结果与配置保存分离，保存成功不等于平台连通。
- CORS 不向任意来源开放，凭证响应禁止缓存。

## 同步状态与恢复

状态机使用 PRD 中的十个状态。每次任务保留目标日期、最后心跳、当前阶段、尝试次数、最后安全错误和上次成功引用。

- 网络、限流和临时下载失败可指数退避重试。
- 登录、验证码和风控进入人工处理，不自动重试。
- schema 变化停止写入该批次并产生数据质量问题。
- 数据校验通过后才在事务中替换目标范围；失败批次不改变消费者所见的最后成功事实。
- 每个来源独立推进，一个来源失败不阻塞其他来源。

## 页面状态

- 加载：保留连接器网格和列表结构的骨架，无中心旋转图标。
- 空：解释尚未配置连接或保险箱条目，提供唯一明确主操作。
- 错误：显示安全中文摘要、request ID 和恢复动作；不清空已加载数据。
- 禁用：按钮旁给出权限、环境或状态原因。
- 待验证：显示需要操作的平台、设备和等待时长，不显示验证码内容。
- 过期：保留上次成功时间和数据日期，突出下一步。
- 无权限：导航与动作按权限隐藏/禁用，API 仍返回 403。

## 响应式、WebView 与无障碍

- 1440/1280px：连接器四列或三列，保险箱类型与条目并列。
- 900px：连接器两列，详情区改为单列。
- 640px：连接器和工具栏一列，操作按钮可换行但主操作保持清楚。
- 390px：页面无横向溢出，配置弹窗为全屏抽屉，底栏避开安全区和软键盘。
- 所有卡片操作可通过键盘到达；卡片本身不用嵌套可点击元素造成冲突。
- 弹窗有语义标题、描述、错误关联、焦点圈和焦点恢复。
- 状态不只依赖颜色；Logo 有替代文本；动画遵守 `prefers-reduced-motion`。
- 在钉钉 WebView 单独验证 Cookie、全屏弹窗、键盘遮挡、滚动和返回行为。

## 视觉与行为验收矩阵

至少验证以下场景：

- 连接器：未配置、待验证、健康、运行中、登录失效、等待人工验证、结构变化、失败、过期、禁用；
- 弹窗：首次新增、已有密文的普通编辑、替换凭证、字段错误、保存失败、无权限、未保存关闭；
- 保险箱：空列表、NAS 多地点、复核过期、查看权限不足、查看超时自动清空；
- 采集器：离线、领取任务、授权过期、心跳丢失、任务恢复；
- 页面宽度：1440、1280、900、640、390px 及钉钉内嵌视口。

## 文案

- 页面说明：`统一连接经营平台和内部系统，安全托管凭证，并把可信数据提供给所有业务 App。`
- 口径：`销售口径：订单创建时间 · Asia/Shanghai · 默认不含其它`
- 加密说明：`敏感信息加密保存；验证码不会被保存；查看与采集取用全程留痕。`
- 未配置：`尚未添加连接`
- 待验证：`配置已保存，等待首次验证`
- 人工验证：`需要在指定公司电脑完成验证后继续`
- 健康：`已接通，数据已更新至昨天`
- 失败保留：`本次同步失败，业务 App 继续使用上次成功数据`
