# 商品主数据治理工作台 PRD

## 文档状态

- 状态：开发中
- 负责人：数据中心 / 运营部 / 产品部 / 供应链
- 最近更新：2026-07-23

## 背景与问题

现有商品主数据页同时承载商品查找、ERP 档案、经营销量、数据更新和跨 App 关联，所有信息在同一层级展开。主表行高过大，低频说明重复出现；销售事实截止不足和未匹配销售汇总只以弱提示展示，用户无法直接进入处理流程。日期和平台筛选会立即触发服务端查询，选择过程可能造成无效读取。

## 目标

1. 默认以商品档案治理和异常处理为主，经营数据作为判断商品状态的上下文。
2. 让销售覆盖不足、未匹配销售汇总和商品档案缺口成为可见、可操作的任务。
3. 主表保持一行一个商品和稳定行高，完整 SKU、组合、销量与关联信息进入详情弹窗。
4. 日期和平台先形成查询草稿，只有明确点击“查询数据”后才读取服务端聚合。
5. 保留 ERP 文件导入、只读快麦状态、分页、权限和现有商品目录 API 契约。

## 非目标

- 不新增或修改 D1 表。
- 不新增快麦 API 能力，不把未打通的同步按钮伪装为可用。
- 不在本期反向编辑快麦商品、SKU、分类、品牌、成本或组合关系。
- 不把商品主数据改造成独立商品销售分析 App。
- 不自动猜测未匹配销售汇总应归属的商品。

## 用户与权限

- 已登录员工可查看商品基础信息、经营数据和数据质量状态。
- 成本继续由服务端权限控制；接口未返回时页面显示无权限或暂无数据，不补零。
- 总经办与运营部可导入 ERP 商品文件。
- 只读用户可查询和查看详情，不显示文件导入操作。
- 未匹配销售汇总由有权限用户前往“数据同步”处理。

## 当前流程

1. 用户在一组并列指标中判断商品数、销量、销售额和同步时间。
2. 用户修改平台或日期后页面立即重新读取。
3. 用户在高行高表格中同时阅读 SKU、组合、分类、销量、平台和成本。
4. 用户看到未匹配数量，但缺少就近处理入口。

## 目标流程

1. 用户先看到商品数、库存单位、有销量商品、商品档案待完善和销售未匹配数量。
2. 有销售未匹配或覆盖不足时，点击“去处理”进入数据同步对应异常。
3. 用户选择任务视图，或通过搜索和本地筛选缩小商品范围。
4. 用户修改平台和日期草稿，点击“查询数据”后更新经营事实。
5. 用户在紧凑主表快速浏览；点击“查看详情”阅读完整 SKU、组合、平台、成本、关联和来源信息。
6. 有权限用户通过“更新数据”导入 ERP 商品文件，预览后确认写入。

## 业务规则

1. 销售继续使用订单创建时间、`Asia/Shanghai`，默认排除“其它/其他”。
2. 销售事实只按已应用的平台和日期范围聚合；查询草稿不得改变现有结果。
3. 未匹配销售汇总不计入任何商品，必须保留数量和处理入口。
4. 商品档案待完善包括：无 SKU、缺库存单位编码、组合商品缺组成、缺分类、缺品牌。
5. 未关联产品全周期属于业务关联状态，不与商品档案缺口混算。
6. 非标准格式的内部唯一码仍是有效身份，不因长度或前缀进入异常。
7. 主表零销量明确显示“该范围暂无销售”，不显示伪造的零经营结论。

## 数据定义

- `catalogIssueCount`：至少存在一个商品档案缺口的主商品数。
- `unmatchedRowCount`：所选销售事实范围内无法唯一匹配到商品的聚合行数。
- `coveredProducts`：所选范围内存在销量或净销售额的商品数。
- `view`：`all`、`attention`、`bundle`、`unsold`、`unlinked`。
- `salesDraft`：尚未应用的 `from`、`to`、`platform`。
- `salesQuery`：已经应用并由商品目录 Provider 请求服务端的范围。
- 商品与销售事实更新时间分别显示，不互相替代。

## 异常与边界

- 空目录：解释导入 ERP 商品文件的下一步，不显示加载失败。
- 查询无结果：保留筛选，提供清除条件。
- 销售覆盖不足：保留商品档案和上次成功经营数据，显示截止日期与处理入口。
- 读取失败：保留上次成功数据，提供重新加载。
- 无写权限：不展示上传控件，详情与查询保持可用。
- 快麦未打通：只显示真实状态和文件导入路径。
- 商品包含大量 SKU 或组件：主表只显示摘要，详情展示完整列表。

## 验收标准

1. 顶部状态带不再以销量和销售额抢占商品治理主层级。
2. 未匹配销售汇总数量可点击进入 `#data-sync`。
3. 页面包含“全部商品、待完善、组合商品、未售商品、未关联产品”任务视图。
4. 平台或日期改变后不立即替换 `salesQuery`；提交查询后只触发一次应用。
5. 表格一行一个商品，完整信息通过可键盘操作的详情入口打开。
6. 详情显示商品档案、SKU/库存单位、组合关系、经营数据、业务关联和数据来源。
7. 1280px、900px、640px 和 390px 下无页面级横向溢出；主表仅在自身容器滚动。
8. 加载、空数据、错误、无权限、禁用和成功状态均有明确中文反馈。

## 上线与回滚

本次无数据库迁移和外部平台契约变化。回滚时恢复原商品工作区和样式即可；商品目录、销售事实、导入批次和跨 App 关联数据均不删除。生产发布前单独验证 Cloudflare 页面、真实 D1 读取和钉钉 WebView。
