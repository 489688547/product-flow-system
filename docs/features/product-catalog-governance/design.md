# 商品主数据治理工作台设计

## 用户任务

用户进入页面首先判断商品目录是否完整、是否存在销售匹配问题，然后查找商品、确认经营表现并查看完整档案。数据更新是低频管理操作，不与日常查询争夺主操作。

## 信息层级

1. 主信息：商品身份、库存单位、有销量覆盖、档案待完善、销售未匹配。
2. 辅助信息：所选范围销量、净销售额、分类品牌、成本和业务关联。
3. 低频信息：全部 SKU、组合组件、平台拆分、来源时间和导入操作。

## 页面结构

1. 连续状态带：商品、库存单位、有销量商品、档案待完善；右侧显示商品档案与销售事实时间。
2. 异常行动条：存在覆盖不足或未匹配汇总时展示原因、影响和“去处理”。
3. 任务视图：全部商品、待完善、组合商品、未售商品、未关联产品。
4. 查询工具栏：搜索、本地分类/类型/关联筛选、平台与确认式日期范围、查询数据、更新数据。
5. 商品表：商品、SKU 摘要、类型/组成、分类品牌、销量/净销售额、成本、状态与详情。
6. 商品详情：大尺寸弹窗分区展示档案、SKU、组合、经营、关联和来源。

## 交互流程

- 搜索、类型、分类、关联和任务视图属于当前目录即时筛选。
- 平台和日期写入草稿；日期浮层确认仅完成草稿，点击“查询数据”才应用到 Provider。
- “待完善”仅筛选有商品档案缺口的商品；销售未匹配不猜测归属，跳转数据同步。
- 点击“查看详情”打开弹窗；关闭、按 Esc 或点击遮罩后焦点返回原按钮。
- 文件导入继续先解析预览，再由用户确认写入。

## 组件复用

- `DateRangePickerField`：复用确认式日期范围和快捷项。
- `HeaderFilter`：复用平台、类型、分类和关联筛选。
- `DataTable`、`TablePagination`：复用表格与分页。
- `Modal`：复用焦点陷阱、Esc、遮罩关闭和焦点返回。
- `Button`：复用加载、禁用、焦点和主次操作。

## 新增组件

### `ProductCatalogDetailDialog`

- 职责：只读展示单个目录商品的完整信息。
- 属性：`product`、`linkedProduct`、`supplierCount`、`catalogUpdatedAt`、`salesUpdatedAt`、`onClose`。
- 状态：商品为空时不渲染；字段缺失显示明确说明。
- 键盘：继承 `Modal` 的焦点管理；关闭按钮与分区内容按正常顺序访问。
- 复用边界：仅商品主数据工作区使用，不升级为共享平台组件。

## 页面状态

- 加载：保持页面标题，状态带和表格使用骨架。
- 空数据：引导有权限用户导入 ERP 文件，普通员工看到只读说明。
- 错误：保留上次成功表格，显示安全错误和重新加载。
- 无权限：隐藏上传操作，成本由服务端继续裁剪。
- 禁用：快麦状态说明真实限制并指向 ERP 文件路径。
- 成功：导入成功继续显示商品/SKU 数量和时间。

## 响应式与钉钉 WebView

- 1280px 以上显示完整表格；首列和表头保持清晰。
- 900px 以下降低次要列宽，详情承担完整阅读。
- 640px 以下查询栏纵向排列，任务视图横向滚动，操作按钮满宽。
- 390px 下页面不横向溢出，表格在自身容器滚动，详情弹窗贴合动态视口。

## 交互文案

- 异常主操作：“去处理未匹配数据”。
- 查询主操作：“查询数据”。
- 更新次操作：“导入 ERP 商品文件”。
- 空销量：“该范围暂无销售”。
- 组合缺口：“组合关系待补齐”。
- 档案缺口：“资料待完善”。

## 无障碍

- 任务视图使用按钮和 `aria-pressed`。
- 异常行动条使用可见图标、文字和链接，不只依赖颜色。
- 详情按钮有完整商品名称的无障碍标签。
- 表头不折行，数字列使用等宽数字。
- 加载、错误和查询状态分别使用 `aria-busy`、`role="alert"` 和 `role="status"`。

## 视觉验收

- 1440×900：默认全部商品、异常行动条、完整表格。
- 1280×800：筛选和操作不互相挤压。
- 900×700：主表可横向滚动，页面本身不溢出。
- 390×844：工具栏纵向、任务视图可滚动、详情弹窗可关闭。
- 状态：正常、覆盖不足、未匹配、空目录、无搜索结果、读取错误、只读、导入预览。
