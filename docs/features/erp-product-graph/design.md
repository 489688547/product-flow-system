# ERP 商品关系图功能设计书

## 用户任务

数据中心维护者需要确认 ERP 商品是否已经完整读取，组合装由哪些库存最小单位构成、比例是多少；产品和供应链人员需要用同一关系理解产品用料、库存余额和成本。

## 信息层级

- 主信息：商品名称、主商家编码、普通/组合类型、库存组成、同步状态。
- 辅助信息：库存单位编码类型、规格、比例、成本和业务关联。
- 低频信息：来源系统 ID、最近详情同步时间和异常代码。

## 页面结构

继续使用数据中心“商品主数据”单一页面，不增加页内侧栏或二级页面：

1. 顶部连续指标带：可售商品、库存单位、组合商品、组件关系、数据问题、最近同步。
2. 搜索和筛选：商品/编码搜索、状态、类型、分类、业务关联。
3. 单一高密度表格：商品、库存单位、库存组成、分类品牌、成本、状态、业务关联。
4. 同步反馈直接显示当前阶段、已读详情/总候选、组件数和异常数。

## 交互流程

1. 点击“同步快麦商品”后按钮进入忙碌状态。
2. 客户端调用同一同步端点；服务端先同步列表，再分批读取组合详情。
3. 返回 `complete:false` 时客户端携带 `cursor` 继续，页面显示进度。
4. 全部候选完成后刷新共享目录并显示完成摘要。
5. 失败时保留现有表格；用户可从相同游标重试。

同步是外部只读动作，不需要不可逆确认；不会修改 ERP。

## 组件复用

- 复用 `ProductCatalogProvider` 作为三个 App 的共享读取和同步编排边界。
- 复用 `DataTable`、`HeaderFilter`、`TablePagination` 和 `Button`。
- 商品表仍保留在数据中心功能内，不抽成通用业务表。

## 新增组件

不新增业务 UI 基础组件。新增纯领域模块 `productCatalogGraph.js`，负责组合标准化、递归展开和库存/成本计算；它不依赖 React、浏览器或网络。

## 页面状态

- 加载：保留现有骨架。
- 空数据：提示同步快麦或导入 ERP 文件。
- 错误：保留最后成功目录，显示失败阶段和重试。
- 无权限：可查看，禁用同步并解释权限。
- 禁用：同步中禁用重复点击。
- 成功：显示商品、库存单位、组合、组件和异常数量。
- 部分成功：显示覆盖率和未完成详情数，不把部分结果描述为完整。

## 响应式与钉钉 WebView

- 1440/1280px 显示完整列；900px 隐藏次要成本细节；640px 操作区换行；390px 仅表格容器横向滚动。
- “内部唯一码”“组合比例”必须横排；禁止单字竖排。
- 钉钉 WebView 使用动态视口和安全区，不增加固定高度弹窗。

## 交互文案

- `SKU / 69 码` 改为 `库存单位编码`。
- `标准 69 码` 与 `内部唯一码` 都是正常状态。
- `条码异常` 改为 `数据问题`。
- 组合摘要示例：`2 个组件：6978705011727 ×1；6977173969059 ×1`。

## 无障碍

- 同步进度使用 `role=status`；失败使用 `role=alert`。
- 所有状态同时包含文字，不只依赖颜色。
- 筛选、按钮和表格操作可键盘访问，并保留可见焦点。

## 视觉验收

- 数据中心商品主数据：有数据、同步中、部分失败、无权限、空状态。
- 组合装双组件和单组件比例 2 的表格可读性。
- 1440、900、640、390px 与钉钉 WebView。
