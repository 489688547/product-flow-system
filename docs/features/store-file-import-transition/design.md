# 店铺数据文件导入过渡设计

## 用户任务

用户在“数据接入 → 电商平台”确认每个平台未来通过原始文件导入，并知道当前需要先提供真实文件样例。

## 信息层级

- 主信息：平台 Logo、平台名称、覆盖的数据类型、状态“等待文件样例”。
- 辅助信息：接入方式“文件导入”；说明收到样例后支持原文件识别、预览和去重。
- 不展示：登录账号、密码、后台地址、负责人、浏览器状态和连接实例。

## 页面结构

电商平台分类使用现有目录网格，包含抖音、快手、淘系、拼多多、小红书和京东/京麦。巨量引擎属于广告平台，保持现有规划入口，不纳入本次店铺凭证清理。

## 交互流程

1. 用户打开电商平台分类。
2. 店铺卡片展示“文件导入”和“等待文件样例”。
3. 操作按钮禁用并显示“等待文件样例”，避免进入无实现的弹窗。
4. 后续拿到样例后，另一个功能周期增加上传、自动识别、首次字段映射和确认导入。

生产清理由管理员通过受控服务端动作执行：明确确认 → 校验权限与版本 → 销毁密文 → 终止任务 → 归档/删除关联元数据 → 写无秘密审计。

## 组件复用

- 复用 `ConnectorCatalog` 的响应式卡片、Logo、状态和无权限语义。
- 扩展目录卡片为只读等待模式，不建立新的电商专属卡片系统。
- 移除 `DouyinConnectionDialog` 与专属自动连接工作区的消费入口。

## 新增组件

不新增共享组件。目录卡片只增加业务中立的动作文案、禁用原因和空状态文案属性；第二个真实消费者出现前不抽取上传向导。

## 页面状态

- 加载：继续使用现有数据接入骨架。
- 空数据：卡片本身即为等待样例状态，不显示空连接列表。
- 错误：ERP/公司数据读取错误保持原逻辑；店铺目录不依赖旧连接 API。
- 无权限：仍可查看等待状态，不可执行管理动作。
- 禁用：按钮显示“等待文件样例”，title/disabled reason 解释需先提供平台原始文件。
- 成功：生产清理成功后普通列表不再出现旧抖店账号或店铺身份。

## 响应式与钉钉 WebView

继续使用三列、两列、单列目录网格；不增加弹窗，窄屏不会出现横向滚动。钉钉 WebView 只需验证卡片和禁用原因。

## 交互文案

- 状态：`等待文件样例`
- 接入方式：`文件导入`
- 按钮：`等待文件样例`
- 说明：`请先提供平台后台原始 XLSX / CSV；识别规则验证后再开放导入。`

## 无障碍

禁用动作同时提供可见状态和文字原因，不依赖颜色；卡片顺序与目录顺序一致，Logo 仍为装饰图片。

## 视觉验收

检查 1440、1180、640 和 390px；确认无账号密码、无网页登录标签、无横向溢出，键盘焦点不会进入禁用动作。
