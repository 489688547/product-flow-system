# 店铺数据文件导入过渡任务

## 执行规则

- 每项任务先写失败测试并确认失败原因，再写实现。
- 生产凭证销毁必须在部署和权限验证后执行。
- 文件解析器等待真实平台样例，不猜字段。

## 任务

- [ ] 记录产品与平台规则
  - 依赖：用户确认 B 方案和立即删除凭证。
  - 输出：PRD、设计、计划、任务和长期规则。
  - 验证：治理检查通过。
- [ ] 店铺目录切换文件等待状态
  - 依赖：产品规则。
  - 输出：店铺连接器仅 `export`，无登录字段和弹窗入口。
  - 失败测试：旧代码仍包含网页登录和账号密码时失败。
  - 验证：目录 UI 定向测试与响应式检查。
- [ ] 实现不可恢复凭证销毁
  - 依赖：共享保险箱。
  - 输出：密文/IV 清空、归档、版本锁和无秘密审计。
  - 失败测试：归档后密文仍存在时失败。
  - 验证：共享保险箱和两个连接 API 定向测试。
- [ ] 清理生产与本机运行时
  - 依赖：部署后的销毁接口。
  - 输出：店铺凭证为零、活动浏览器任务为零、LaunchAgent 停用。
  - 验证：生产只读检查和本机 launchctl 检查。
- [ ] 发布并合并
  - 依赖：Definition of Done 全部通过。
  - 输出：PR 合并到 main，生产页面与 API 验证通过。
  - 验证：分支基线、GitHub 检查和生产 readiness。
