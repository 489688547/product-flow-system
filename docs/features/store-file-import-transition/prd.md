# 店铺数据文件导入过渡 PRD

## 文档状态

- 状态：开发中
- 负责人：总经办 / 运营部
- 最近更新：2026-07-21

## 背景与问题

店铺后台登录会动态触发协议确认、邮箱验证码、滑块、扫码和设备确认。浏览器采集无法稳定做到无人值守，实际维护成本接近人工导出，且长期保存店铺账号密码增加不必要的安全风险。

## 目标

- 抖音、快手、淘系、拼多多、小红书和京东/京麦店铺入口统一改为文件导入方向。
- 在取得各平台真实原始文件前，页面明确显示“等待文件样例”，不开放虚假的导入动作。
- 删除现有店铺登录凭证的可恢复密文，结束残留浏览器任务并停用公司 Mac 店铺采集器。
- 保留无秘密的审计记录；已有标准销售事实不删除。

## 非目标

- 本次不猜测各平台未知文件结构，不实现未经样例验证的解析器。
- 不改变快麦 ERP、钉钉、NAS 和广告平台已有的真实接入边界。
- 不删除历史销售事实、商品主数据或无秘密审计记录。

## 用户与权限

- 所有已授权员工可查看店铺平台处于“等待文件样例”。
- 只有总经办平台管理员可执行不可恢复的店铺凭证销毁。
- 凭证销毁必须使用服务端权限、当前版本和明确确认文案；浏览器隐藏入口不是权限边界。

## 当前流程

1. 用户保存店铺登录账号和密码。
2. 公司 Mac 领取浏览器任务并打开店铺后台。
3. 平台风险验证需要人工接管，任务容易长期停留或失败。

## 目标流程

1. 数据接入页展示各店铺平台 Logo、数据范围和“等待文件样例”。
2. 不再收集账号密码，不再创建店铺浏览器任务。
3. 管理员销毁既有店铺凭证并终止旧任务。
4. 后续收到真实 XLSX/CSV 样例后，另行评审平台原文件识别、预览、字段映射和入库实现。

## 业务规则

- 店铺平台的接入方式固定为 `export`；店铺连接器不得定义登录/API secret 字段。
- “等待文件样例”不等于已接通，也不等于导入器可用。
- 订单销售口径继续使用订单创建时间、Asia/Shanghai，并排除“其它/其他”。
- 销毁凭证后密文和 IV 必须被清空，关联连接归档，浏览器任务终止；审计只保留标识、动作和结果。

## 数据定义

- `credential_vault_entries`：店铺连接凭证销毁后清空 `ciphertext` 和 `iv`，记录 `archived_at`、`archived_by` 和新版本。
- `data_connector_instances`：店铺连接实例归档，不再出现在普通列表。
- `data_connections`：旧抖店专属连接脱敏并停用；关联店铺删除，活动任务结束，grant 删除。
- 标准销售与商品事实：保持不变。

## 异常与边界

- 版本冲突：拒绝清理并要求刷新，避免销毁错误对象。
- 凭证已销毁：幂等返回已完成，不尝试 reveal。
- 清理中断：优先保证秘密不可恢复；残留无秘密元数据可再次清理。
- 未提供文件样例：页面保持等待状态，不接受上传。

## 验收标准

1. 电商平台卡片不再展示网页/API 登录方式、账号密码或“添加连接”。
2. 店铺平台统一显示“文件导入”和“等待文件样例”。
3. 普通用户不能调用销毁操作；管理员确认后密文和 IV 清空。
4. 所有抖店浏览器任务不再可领取，本机采集器停用。
5. 历史销售事实和审计仍可读取。

## 上线与回滚

- 上线顺序：先部署只读店铺目录和销毁接口，再执行生产清理，最后停用本机采集器。
- 回滚只恢复目录展示，不恢复被销毁的凭证；如需重启网页登录必须重新取得用户授权并重新录入。
