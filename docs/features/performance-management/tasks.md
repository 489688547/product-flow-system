# 人事管理 App · 绩效管理模块执行任务

## 执行规则

- 每项任务只交付一个可独立验证的结果。
- 先写失败测试并确认失败原因，再写实现。
- 完成后记录实际验证命令和结果。
- 每次提交只包含当前任务文件。

## 任务

- [x] 规格与边界
  - 依赖：用户确认人事管理 App 内包含绩效、自评/自评分、建议分、主管定分和一次复核。
  - 文件：本目录四份文档、电商运营绩效证据边界。
  - 输入：已确认流程和现有公司激励/月报边界。
  - 输出：可执行规格。
  - 失败测试：功能文档缺失时治理检查不能覆盖本功能。
  - 实现步骤：拆分领域；确定角色、状态、证据和冻结；自审。
  - 验证：`npm run check:governance`。
  - 提交：`docs(perf): design performance management app`。

- [x] 领域模型
  - 依赖：规格。
  - 文件：`src/domain/performanceManagement.js`、领域测试。
  - 输入：周期、方案、证据、自评和主管评分。
  - 输出：校验、建议分、差异原因、一次复核和冻结纯函数。
  - 失败测试：权重、越权流转、重复复核和冻结覆盖未被拒绝。
  - 实现步骤：默认值；校验；计分；状态机；过滤范围。
  - 验证：领域测试通过。
  - 提交：`feat(perf): add performance domain`。

- [x] API 与存储
  - 依赖：领域模型。
  - 文件：`functions/api/performance-management*`、API 测试、`package.json`。
  - 输入：钉钉会话、领域动作和 D1。
  - 输出：过滤读取、动作、版本、审计和冻结快照。
  - 失败测试：匿名、越权、旧版本、第二次复核和冻结覆盖成功。
  - 实现步骤：建表；过滤；动作权限；快照；错误契约。
  - 验证：绩效 API 与平台回归通过。
  - 提交：`feat(perf): add performance APIs`。

- [x] App 与核心 UI
  - 依赖：API、运营证据出口。
  - 文件：Provider、App 壳、权限、五个工作区、UI 测试。
  - 输入：绩效状态、组织和业务证据。
  - 输出：人事、主管、员工完整流程。
  - 失败测试：评分敏感数据越界、差异原因不必填或任务完成直接等于达标。
  - 实现步骤：注册；角色首页；方案；自评；评分；复核；冻结。
  - 验证：角色流程和权限测试通过。
  - 提交：`feat(perf): add performance workflows`。

- [x] 视觉与发布
  - 依赖：完整功能。
  - 文件：样式、耐久文档、集成与全部测试。
  - 输入：五个页面和所有状态。
  - 输出：响应式、无障碍、部署验证和回滚记录。
  - 失败测试：移动端、焦点、空错禁用、无建议分和冻结状态缺失。
  - 实现步骤：视觉；键盘；文档；全套检查；生产冒烟。
  - 验证：Definition of Done 与生产页面通过。
  - 提交：`style(perf): finish performance delivery`。
