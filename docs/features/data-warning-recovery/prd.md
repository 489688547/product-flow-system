# 数据异常处理闭环 PRD

## 文档状态

- 状态：开发中
- 负责人：数据中心
- 最近更新：2026-07-23

## 背景与问题

数据总览右上角会提示最新销售日异常，但当前点击后只进入通用“同步记录”。页面虽然解释异常，却没有定位对应问题或给出真正可执行的处理入口。更严重的是，现有页面会自动调用尚未打通的快麦开放平台 API，与公司当前只采用 Chrome 插件网页采集和快麦官方文件导入的决定不一致。

## 目标

- 点击数据总览右上角异常提示后，进入“数据同步”并自动定位到该销售异常。
- 异常卡片读取公司 Mac 的 Chrome 采集设备与快麦任务状态，明确下一步由谁在什么设备完成什么操作。
- 异常识别后自动幂等排队一次 Chrome 采集；需要人工登录时提供“我已登录，重新触发”。
- 提供“检查 Chrome 采集”和“导入官方文件”两个兜底入口。
- 快麦开放平台 API 在数据接入、设置和商品主数据页面统一显示“未打通”，且页面不得自动或手动调用该 API。

## 非目标

- 本次不新增 Chrome 插件的页面选择器、资源类型或自动验证码能力。
- 本次不删除历史快麦 API 路由、凭据或已产生的同步记录。
- 本次不改变销售事实的订单创建时间、Asia/Shanghai、截止昨天和默认排除“其它”的口径。

## 用户与权限

- 具有数据中心查看权限的用户可以查看异常、采集设备和任务安全摘要。
- 只有原有销售数据编辑权限用户可以在“设置 → 销售数据源”完成官方文件导入。
- 页面不得返回或展示凭据、Cookie、页面内容、原始文件绝对路径或客户敏感信息。

## 当前流程

1. 总览发现截止昨天的目标业务日完全缺失，或最新销售日 GMV 与销量异常。
2. 页面自动请求快麦开放平台 API 补拉。
3. 用户点击警告进入通用同步页，但无法直接判断 Chrome 插件是否在线或去哪里补数。

## 目标流程

1. 总览发现昨天整日缺失或最新日数据疑似截断后，自动为目标日期幂等创建一次 Chrome `order_items` 采集任务，不触发快麦开放平台 API。
2. 用户点击“去处理”进入 `#data-sync/kuaimai-sales`，页面滚动并聚焦异常卡片。
3. 卡片读取 `/api/platform/v1/web-collection/jobs` 的安全状态，展示公司 Mac、Chrome、快麦订单明细任务和最近结果。
4. 若任务报告登录失效，用户在公司 Mac 的 Chrome 完成快麦登录后点击“我已登录，重新触发”；服务端将可恢复任务重新排队。
5. 采集器离线时在当前页重新检测心跳，不用跳转掩盖离线问题；只有任务明确报告快麦登录失效时才提供“打开快麦 ERP”。
6. 用户也可进入 ERP 数据接入查看采集器设置，或直接进入销售数据源导入官方文件。

## 业务规则

- 快麦开放平台 API 当前状态固定为“未打通”，不得因已保存历史凭据显示“已连接”。
- Chrome 插件与官方文件是当前仅有的快麦数据接入通道。
- 自动触发以 `kuaimai:order_items:<businessDate>:v1` 去重；重复打开总览不得生成重复任务。
- 人工重新触发仅允许总经办、数据中心和运营的非只读身份；运行中的任务不重复排队。
- “重新触发”表示进入公司 Mac 采集队列，插件在下一轮轮询时领取，不代表浏览器已经立即执行。
- 任务状态必须区分运行中、等待登录/验证、页面结构变化、失败、成功和尚无任务。
- 采集器离线时不得把“打开快麦页面”表述为恢复动作；页面只能重新检测心跳，并明确后台采集服务与网页登录是两层状态。
- 缺失字段显示为“—”或明确空状态，不补 0，不编造在线状态。
- 历史快麦 API 修复记录可保留在执行记录中，但不得作为当前自动修复能力展示。

## 数据定义

- `runners[]`：`id/name/status/version/chromeStatus/currentJobId/lastSeenAt`。
- `jobs[]`：`id/providerId/resourceType/businessDate/status/stage/attempt/errorCode/errorSummary/runnerId/startedAt/completedAt/updatedAt`。
- 销售异常目标资源优先匹配 `providerId=kuaimai`、`resourceType=order_items` 和异常业务日期。
- 只有当前查询范围截止日等于 Asia/Shanghai 的昨天时，才把目标日缺失判定为自动采集异常；历史范围查询不得误触发当天任务。
- 用户触发请求固定为 `action=trigger/providerId=kuaimai/resourceType=order_items/businessDate/force`，不得携带 URL、凭据或页面指令。

## 异常与边界

- 状态接口加载失败：保留异常说明，显示“采集状态暂时无法读取”，仍允许进入数据接入或导入文件。
- 无采集设备：明确提示需要公司 Mac 安装并启动采集器。
- 设备在线但无当天任务：自动排队一次；状态页提示等待 Chrome 插件领取。
- 需要登录、验证码或页面变化：显示对应安全错误摘要和恢复动作。

## 验收标准

- 总览异常提示文案包含“去处理”，链接为 `#data-sync/kuaimai-sales`。
- 昨天完全缺失时，数据总览必须显示“数据未同步”，数据同步页必须显示最近可信日期和已自动排队说明。
- 打开深链后焦点落在异常卡片，键盘用户可继续操作两个入口。
- 页面源码不再导入或调用 `requestSalesRepair`。
- 快麦平台连接显示“未打通”，设置页不再请求 `/api/kuaimai/status|pull|refresh`，商品页不再提供可点击的 API 同步按钮。
- Chrome 状态接口失败、空、运行、等待人工、成功和失败均有可理解文案。
- 自动触发同一日期不会重复建任务；登录后按钮可把 `waiting_human/failed/schema_changed/success` 任务重新排队并清除旧租约和安全错误。

## 上线与回滚

- 无数据库迁移，无新增变量或绑定。
- 回滚时恢复前端警告与连接展示即可；历史任务、归档和销售事实不受影响。
- 上线后分别验证本地页面、生产 API 只读状态和生产页面；未部署前不得宣称线上已修复。
