# 数据异常处理闭环功能设计书

## 用户任务

用户看到数据总览右上角警告后，需要立即知道异常是哪一天、Chrome 采集执行到哪里、自己是否需要登录/验证，以及可去哪里补数。

## 信息层级

1. 异常日期、异常程度和当前恢复状态。
2. 主操作随故障层级变化：离线时“重新检测采集器”、采集中“刷新采集进度”、登录失效时“我已登录，重新触发”；兜底操作为“导入官方文件”。
3. 设备、最近任务、阶段、尝试次数和安全错误摘要。
4. 原始归档、执行记录和其他待处理质量问题。

## 页面结构

- 总览右上角警告仍位于页头，但异常时文案以“去处理”收尾。
- `#data-sync/kuaimai-sales` 进入数据同步后，异常卡片置于质量摘要下方并获得焦点。
- 异常卡片内部先显示诊断与行动区，再显示 Chrome 设备和当天订单明细任务摘要。
- 数据接入的快麦 ERP 卡片区分“Chrome/官方文件可用”和“开放平台 API 未打通”。

## 交互流程

1. 点击右上角警告。
2. 路由切换到数据同步并滚动到销售异常卡片。
3. 根据状态选择：
   - 登录已经恢复：点击“我已登录，重新触发”，任务进入队列并等待插件下一轮领取。
   - 采集器离线：在当前页重新检测心跳；不把打开网页当成恢复采集器。
   - 登录失效：打开快麦 ERP 完成登录，再重新触发。
   - 需要立即补数或完整退款/成本：进入 `#settings/sales-data`。
4. 设置页深链自动滚动并聚焦“销售数据源”。

## 组件复用

- 复用 `PageHeader`、`Button`、`DataTable` 和已有 `status-badge`。
- 复用现有 Web Collection `/jobs` 契约：GET 读取安全状态，POST `action=trigger` 幂等排队或人工重排。
- 复用 `SalesDataSettings` 的官方文件解析与保存能力。

## 新增组件

- 不新增共享组件。
- `SyncRunsWorkspace` 增加局部 Chrome 恢复状态区；它只组合已登记的通用采集状态，不包含 provider 选择器或浏览器自动化。

## 页面状态

- 加载：异常卡片显示“正在读取 Chrome 采集状态”。
- 昨天整日缺失：显示“销售数据尚未同步”、最近可信日期和“系统已自动排队 Chrome 采集任务”。
- 触发中：主按钮禁用并显示“正在触发…”；成功显示排队结果，失败显示安全错误并保留所有兜底入口。
- 空数据：说明公司 Mac 尚无当天任务，并保留两个处理入口。
- 错误：显示安全错误，不隐藏补数入口。
- 无权限：数据接入或文件导入页面沿用原有权限禁用原因。
- 禁用：快麦 API 使用“未打通”文字和禁用操作。
- 成功：显示 Chrome 采集已完成，并提示刷新状态确认销售事实。

## 响应式与钉钉 WebView

- 笔记本宽度下诊断文案与按钮同排；窄屏下按钮纵向全宽。
- 状态详情允许换行，不出现横向溢出。
- 深链聚焦不依赖鼠标和悬停。

## 交互文案

- 总览：`⚠️ 2026年7月22日数据疑似不完整 · 去处理`
- API 状态：`快麦开放平台 API · 未打通`
- 离线主操作：`重新检测采集器`
- 采集中主操作：`刷新采集进度`
- 登录恢复操作：`我已登录，重新触发`
- 兜底操作：`导入官方销售报表`
- 登录异常：`请在公司 Mac 的 Chrome 中打开快麦并完成登录，然后点击“我已登录，重新触发”。`

## 无障碍

- 异常卡片使用 `role="alert"`、`tabIndex="-1"` 和可见焦点。
- 链接使用动作型可读文案，不能只显示图标。
- 状态不只依赖颜色；所有状态同时提供中文文字。

## 视觉验收

- 1440×900：总览警告、同步异常卡片、快麦 ERP 数据接入详情。
- 390×844：异常卡片按钮堆叠和焦点环。
- 设置页 `#settings/sales-data` 深链后的销售导入区域。
