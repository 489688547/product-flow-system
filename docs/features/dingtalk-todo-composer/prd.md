# 钉钉待办编排器 PRD

## 文档状态

- 状态：开发中
- 负责人：产品负责人
- 最近更新：2026-07-18

## 背景与问题

产品进度中的任务可以同步为钉钉待办，但当前流程仍要求用户记住群名，缺少“我的群”默认入口；发送前只能选择执行人，不能核验或调整待办标题、正文、优先级和具体截止时间。没有截止日期时入口直接禁用，也没有修复路径。群成员身份无法确认时只显示跳过人数，用户无法判断漏发对象。

## 目标

- 群聊模式默认展示当前用户创建或管理的群，并保留关键词搜索。
- 选择群后明确展示最终执行人、来源群、去重结果和跳过成员原因。
- 发送前可编辑标题、富文本正文、优先级、截止日期和具体时间。
- 没有任务截止日期时仍可进入弹窗并完成补录。
- 发送或更新钉钉待办后保存最后一次编排内容，下一次更新可继续编辑。
- 覆盖加载、空数据、授权失败、接口失败、提交禁用、键盘和窄屏状态。

## 非目标

- 不把待办直接发送到群会话；群聊仅作为执行人选择来源。
- 不创建、修改或解散钉钉群。
- 不在本次实现附件、子待办、评论、循环待办或独立提醒规则。
- 不改变其他来源待办的同步流程。

## 用户与权限

- 能编辑产品进度任务的企业成员可以打开编排器。
- 钉钉群列表和成员读取使用当前登录用户的授权令牌，只返回其可见群。
- 群成员必须精确映射到当前企业有效组织成员并具有 unionId，才可成为执行人。
- 服务端继续校验会话、令牌、请求方法和必填参数；前端隐藏或禁用不是权限边界。

## 当前流程

1. 用户为任务设置截止日期。
2. 点击“未同步”。
3. 按人员选择，或输入群名搜索并读取群成员。
4. 点击发送，系统用固定标题、纯文本正文、普通优先级和当天 18:00 创建待办。

## 目标流程

1. 用户点击“未同步/待更新”，无论任务是否已有截止日期都可打开编排器。
2. 系统生成标题、正文、普通优先级和截止时间默认值；已同步任务恢复上次编排内容。
3. 用户按人员选择，或进入群聊模式直接查看“我的群”，也可搜索其他可见群。
4. 选择群后，系统读取并合并可识别企业成员；用户可以逐人排除，并查看跳过成员及原因。
5. 用户核对最终执行人和待办预览，编辑标题、富文本正文、优先级、日期及时间。
6. 系统校验标题、正文、截止时间和至少一名执行人，再创建或更新同一条钉钉待办。
7. 成功后保存执行人和编排草稿；失败时保留输入并显示可恢复错误。

## 业务规则

- 优先级采用钉钉值：10 低、20 普通、30 较高、40 紧急，默认 20。
- 新待办默认截止时间为任务截止日 18:00；任务无日期时日期为空、发送按钮禁用但弹窗可用。
- 在编排器修改截止日期后，同步更新产品任务截止日期，保持本地任务与钉钉一致。
- 富文本 HTML 仅用于本地编辑和预览；同步钉钉时转换为保留段落、列表和链接文本的可读正文，避免发送 HTML 标签或内嵌图片数据。
- 多群重复成员按 unionId 去重；人工选择优先保留；用户明确排除后，同一弹窗内再次选择群也不自动加回。
- 群聊不是待办接收对象，最终只提交执行人 unionId。
- 更新已有待办时保持 todoId 与 sourceId，不重复创建。

## 数据定义

- `subject`：待办标题，1-1024 字符。
- `descriptionHtml`：本地富文本草稿，禁止内嵌图片和脚本。
- `description`：由富文本转换的钉钉正文纯文本。
- `priority`：10/20/30/40。
- `dueDate`：`YYYY-MM-DD`。
- `dueClock`：`HH:mm`，默认 `18:00`。
- `dueTime`：按 Asia/Shanghai 组合后的毫秒时间戳。
- `groups`：已选群的 ID、名称、成员数、跳过信息；不传给待办 API。
- `dingTodo.draft`：最后一次成功同步的标题、富文本正文、优先级、日期和时间。

## 异常与边界

- 我的群为空：展示解释性空态和搜索入口。
- 用户授权失效：显示重新授权链接，保留已填写草稿。
- 群成员包含外部联系人、离职人员、机器人或无法映射身份：不加入执行人，展示具体姓名与原因摘要。
- 新群搜索开始时清空旧结果；搜索失败不显示旧结果。
- 群成员读取期间锁定其他群选择，防止并发状态错乱。
- 组织缓存晚到时，保留用户已做的选择，只补齐尚未初始化的已同步执行人。
- 提交期间所有模式、搜索、添加和移除操作均禁用。

## 验收标准

- 打开群聊模式且搜索为空时，自动请求并展示我的群。
- 搜索“产品”时能展示匹配群，清空搜索后恢复我的群。
- 选择两个有重复成员的群，最终执行人不重复；移除一群不误删其他来源成员。
- 任务无日期时可打开弹窗，在弹窗内设置日期和时间后发送按钮启用。
- 预览显示最终标题、正文、优先级、完整截止时间和执行人数量。
- 更新已有待办时，优先级、正文、截止时间和执行人都进入更新请求。
- 键盘用户进入弹窗后焦点位于弹窗内，Tab 不离开弹窗，Escape 关闭并恢复触发按钮焦点。

## 上线与回滚

- 不新增数据库表；旧任务没有 `dingTodo.draft` 时使用任务字段生成默认值。
- 新字段均为可选，旧状态向后兼容。
- 回滚前端编排器和我的群 API 即可恢复旧流程，已同步钉钉待办不受影响。
- 发布前分别验证本地预览、Cloudflare Functions、生产登录态和钉钉 WebView；本任务不自动部署。
