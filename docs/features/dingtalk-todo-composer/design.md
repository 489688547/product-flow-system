# 钉钉待办编排器功能设计

## 用户任务

产品经理要在不离开产品进度页面的情况下，把当前任务交给正确的企业成员，并在发送前确认接收人、正文、优先级和截止时间。

## 信息层级

1. 主信息：待办标题、正文、截止时间、优先级、最终执行人。
2. 辅助信息：产品、责任部门、当前同步状态、来源群和跳过成员。
3. 低频信息：重新授权、搜索无结果、身份映射说明。

## 页面结构

- 弹窗宽度使用 `default`，分为“待办内容”“执行人”“发送预览”三个连续区块，不使用嵌套卡片。
- 内容区包含标题、优先级、截止日期/时间和紧凑型富文本编辑器。
- 执行人区保留按人员/按群聊分段控件；群聊默认显示我的群，输入关键词后替换为搜索结果。
- 预览区以只读摘要展示最终标题、正文摘要、截止、优先级和执行人数量。

## 交互流程

1. 打开时生成或恢复草稿，焦点进入标题。
2. 按群聊模式立即加载我的群；搜索使用 300ms 防抖。
3. 读取群成员期间只允许取消弹窗，不允许切换模式或修改选择。
4. 跳过成员在已选群旁显示“查看 N 人”，展开后展示姓名与原因。
5. 发送按钮随校验实时启用；提交期间整个表单禁用并显示同步状态。
6. 成功关闭并恢复触发按钮焦点；失败保留全部输入。

## 组件复用

- 复用 `Modal`、`Button`、`DatePickerField`、`RichTextEditor` 和现有组织人员选择样式。
- 扩展 `Modal` 的基础焦点管理，因为它是多个真实消费者共同使用的基础控件。
- 扩展 `RichTextEditor` 支持 `compact`、`disabled` 和 `allowImages=false`，不创建第二套编辑器。

## 新增组件

- `TodoComposerFields`：待办标题、正文、优先级、截止时间与校验；保持产品进度业务语义，不进入 `src/ui`。
- `TodoPreview`：只读发送摘要，仅用于产品任务待办。
- `GroupExecutorPicker` 扩展我的群、跳过成员详情、完整禁用与异步互斥。

## 页面状态

- 加载：我的群使用三行骨架；搜索和群成员读取使用行内状态。
- 空数据：我的群为空时提示“暂无可用群，可搜索其他群聊”；人员无匹配时显示明确空态。
- 错误：错误说明影响和重试/重新授权动作；不保留过期结果。
- 无权限：提示当前钉钉授权不足并提供重新授权。
- 禁用：提交期间所有控件禁用；发送按钮展示首个校验原因。
- 成功：关闭弹窗，任务操作状态变为已同步或已更新。

## 响应式与钉钉 WebView

- ≥720px：内容与执行人纵向排列，日期/时间/优先级同一行。
- <720px：字段单列，Modal 保持底部弹层规则，页脚按钮满宽且适配安全区。
- 群列表和执行人列表各自限高滚动，不让整个弹窗出现双重横向滚动。

## 交互文案

- 群聊空态：“选择最近使用的群，或搜索其他群聊。群聊仅用于带入企业执行人，不会向群发送消息。”
- 跳过提示：“N 名群成员未加入执行人。”
- 授权错误：“当前钉钉授权无法读取群聊，请重新授权后继续。”
- 发送预览：“将为 N 名执行人创建/更新 1 条钉钉待办。”

## 无障碍

- Modal 打开时聚焦首个可操作控件，Tab/Shift+Tab 在弹窗内循环，Escape 关闭，关闭后恢复触发元素焦点。
- 异步状态使用 `aria-live`；错误使用 `role=alert`。
- 分段按钮使用 `aria-pressed`；所有移除按钮包含人员或群名称。
- 触控目标不小于 40px，主要操作不小于 44px。

## 视觉验收

- 1440×900：默认我的群、已选群、发送预览、错误状态。
- 390×844：内容编辑、群列表、页脚按钮、键盘焦点与无横向溢出。
- 钉钉 WebView：安全区、授权跳转和返回后草稿恢复。
