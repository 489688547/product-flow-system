# 展示数据库环境 PRD

- 状态：已确认，待实施
- 日期：2026-07-23
- 产品负责人：公司最高权限账号
- 关联设计：`docs/features/demo-data-environment/design.md`

## 一、背景

当前系统只有正式业务数据库。对外展示时，真实业务结构、页面关系和操作流程有价值，但现有真实销售规模不适合直接作为展示口径。仅在页面上把数字乘二会造成详情、汇总、导入、同步和 AI 读取互相矛盾；在正式数据库中写入展示数据又会污染真实经营数据。

因此需要增加一套独立的展示业务数据库，并让最高权限用户在设置中为当前浏览器切换。展示数据库使用真实业务数据生成，销售类可加总事实扩大两倍，比例和均值保持原口径。真实数据采集仍可使用，但任何改变钉钉等外部平台状态的动作必须模拟。

## 二、目标

1. 提供一套与正式系统结构一致、物理隔离的展示 D1 数据库。
2. 最高权限用户可以在设置中切换正式数据库和展示数据库。
3. 切换只影响当前账号的当前浏览器会话，不影响其他浏览器和员工。
4. 展示数据库的销售数量、订单量和金额类可加总指标精确为来源的两倍。
5. 退款率、毛利率、客单价等比例和均值由转换后的基础事实重新计算，保持原口径。
6. 快麦、ERP 文件和浏览器采集等真实读取能力在展示环境仍可使用，结果只写展示数据库。
7. 钉钉待办、日历、审批等外部写动作在展示环境模拟成功，不调用真实 Provider。
8. 所有业务 App、共享 API 和 AI Skill 使用同一服务端数据环境路由，后续新增能力不能绕过。
9. 展示库刷新期间不提供半成品数据；失败时不回退正式库。

## 三、非目标

1. 不为每名员工创建独立展示数据库。
2. 不支持同一展示库内部的 A/B 双槽或无停机刷新。
3. 不在普通业务页面显示“演示数据”“展示数据”水印或提示条。
4. 不把展示环境当作测试环境；试验性写入继续使用本地沙箱。
5. 不把尚未接通的快麦开放平台 API 描述为已接通能力。
6. 不复制或展示平台凭证、会话、个人令牌、生产写入解锁和秘密审计。
7. 不允许普通用户管理数据环境。
8. 不保证刷新展示数据库期间可以继续浏览展示环境；用户已接受等待。

## 四、用户与权限

### 4.1 主要用户

当前主要用户为公司的最高权限账号。未来存在其他员工时，普通员工默认只能使用正式数据库，且看不到数据环境设置。

### 4.2 权限规则

- 只有状态为 active 的 `executive` 或等价最高权限身份可查看数据环境卡片。
- 切换环境、创建刷新任务、重试刷新和读取刷新详情均由服务端再次校验权限。
- UI 隐藏不构成授权边界。
- 本地线上模式必须使用服务端解析出的真实最高权限身份，不允许硬编码本地身份。
- 浏览器不得提交 D1 绑定名、数据库 ID 或任意目标库标识。

## 五、核心用户流程

### 5.1 首次准备展示数据库

1. 用户进入“设置”。
2. 系统显示当前为“正式数据库”，并显示展示数据库尚未就绪。
3. 用户点击“更新展示数据库”。
4. 服务端验证最高权限、正式库与展示库结构和容量。
5. 展示环境进入维护状态，系统分批复制、脱敏、转换并校验业务数据。
6. 设置页显示进度；用户可等待，刷新页面后也可继续查看同一任务。
7. 全部校验通过后，展示数据库变为可用。
8. 用户选择“展示数据库”并确认，页面重新加载后使用展示业务数据。

### 5.2 使用展示环境

1. 普通业务页面按原样展示，不出现展示标签。
2. 所有业务查询、编辑和同步通过统一路由访问展示数据库。
3. 真实采集按钮保持可见、可用。
4. 销售采集结果经统一转换后写入展示库。
5. 外部写动作返回与正式成功合同兼容的模拟结果，并只更新展示库状态。
6. AI 使用展示库业务上下文，但 Provider 配置和用量审计仍在正式控制库。

### 5.3 切回正式环境

1. 用户进入设置并选择“正式数据库”。
2. 服务端更新当前浏览器会话的数据环境选择。
3. 前端终止旧请求、切换缓存命名空间并重新加载。
4. 后续业务请求访问正式数据库。

### 5.4 刷新展示数据库

1. 最高权限用户在设置中点击“更新展示数据库”。
2. 展示环境立即进入维护状态，现有展示业务请求返回稳定维护错误。
3. 服务端按可续跑步骤从正式业务库重新生成唯一展示库。
4. 成功后环境版本递增并恢复可用。
5. 失败后保持不可用，不静默读取正式数据；用户可重试。
6. 正式数据库和正式用户会话不受影响。

## 六、产品规则

### 6.1 数据环境

- 环境枚举固定为 `production` 和 `display`。
- 未建立有效选择时默认为 `production`。
- `display` 只有在绑定存在、结构匹配、状态为 ready 时才能被选择和使用。
- 已选择 `display` 后，如果其状态变为 refreshing 或 failed，业务请求直接失败，不回退 `production`。
- 环境选择通过随机 HttpOnly 会话授权保存；数据库只保存其哈希。
- 同一浏览器会话的标签页共享选择；其他浏览器拥有独立选择。
- 数据环境版本每次成功刷新后递增。
- 业务写请求必须携带前端最近一次获得的环境版本；版本过期时拒绝写入。

### 6.2 正式控制面

以下数据始终保存在 `PRODUCT_FLOW_DB`：

- 登录会话、组织身份、角色和权限。
- 数据环境选择、刷新任务和安全审计。
- 平台连接保险箱、密钥密文和凭证版本。
- 本地线上个人令牌、生产写入解锁、写前快照。
- 采集任务的授权令牌和运行控制记录。
- AI Provider 配置、调用租约、Token 用量和安全审计。

### 6.3 可切换业务面

以下数据通过统一路由使用 `PRODUCT_FLOW_DB` 或 `DEMO_FLOW_DB`：

- 公司共享业务状态及分片。
- 数据中心业务元数据、数据标准、质量问题和业务同步结果。
- 销售日事实、商品主数据和经营指标。
- 产品、供应链、电商运营、品牌、人事、绩效和协同业务记录。
- AI Skill 查询的业务上下文和业务事实。

### 6.4 展示刷新

- 系统只有一个物理展示 D1，不建立 A/B 双槽。
- 同一时间最多存在一个活动刷新任务。
- 刷新采用服务端分步、游标化、幂等执行；浏览器只负责发起和轮询，不搬运业务记录。
- 刷新开始后清理并重建允许复制的展示业务表。
- 刷新任务关闭页面后仍可通过任务 ID 恢复。
- 相同来源版本与规则版本必须生成相同结果。
- 重复点击或重试不得造成数据累计为四倍。
- 成功前展示库始终不可用。
- 失败后保留错误阶段和安全结果码，禁止读取半成品。
- 运维恢复可使用 Cloudflare D1 Time Travel，但应用不保存 Cloudflare 控制令牌。

### 6.5 数据目录与脱敏

每张表必须在展示数据目录中声明以下策略之一：

- `copy`：原样复制业务字段。
- `mask`：复制前按字段级规则脱敏。
- `transform_sales`：使用销售两倍转换器。
- `recalculate`：不复制结果，基于已转换事实重新生成。
- `skip`：控制面、凭证、秘密或不适合展示的数据不得复制。

未登记表默认 `skip`，不得因为新增表而自动进入展示库。

手机号、地址、私人邮箱和其他个人识别字段必须确定性脱敏，以便关系稳定但不可还原。平台 Secret、Cookie、访问令牌、Authorization header、Provider 原始响应和浏览器采集授权不得进入展示库。

### 6.6 销售两倍规则

可加总字段乘以 2：

- 订单量、销售数量、净销量。
- 销售额、净销售额、退款额。
- 成本、毛利。
- 发货前退款、发货后退款。
- 其他经数据标准登记为 additive 的销售事实。

保持或重算字段：

- 日期、平台、商品、69 码、部门等维度保持不变。
- 单价、税率、指标定义和账期保持不变。
- 退款率、毛利率、转化率从转换后的分子分母重算。
- 客单价、件单价、平均成本从转换后的分子分母重算。
- 库存、采购、人员、项目状态等非销售事实不乘二。

必须满足：

- 展示销售数量和各金额总计精确为正式来源的 2 倍。
- 退款率、毛利率和均值在既定舍入精度内与来源一致。
- 现有数据恒等式继续成立。
- 上海时区、订单创建时间和平台分类口径不变。
- 来源只有聚合事实时只转换聚合事实，不虚构客户或订单明细。
- 来源存在明细时使用确定性展示 ID 复制关系完整的第二份记录。

### 6.7 真实采集

- 正式环境的采集结果写正式业务库。
- 展示环境的采集任务在创建时由服务端记录目标环境和目标环境版本。
- 运行器从控制库领取任务，根据服务端保存的目标环境解析业务库。
- 幂等键必须包含目标环境，避免正式和展示任务互相覆盖。
- 展示环境的销售事实通过统一转换器后写展示库。
- 采集任务不得接受调用方提交的绑定名或 D1 ID。
- 当前快麦主通道以集成注册表登记的官方文件和浏览器采集为准。

### 6.8 外部平台动作

展示环境下，所有会改变外部平台状态的动作都必须模拟，包括：

- 创建或更新钉钉待办。
- 创建日历日程。
- 发起或推进审批。
- 其他经集成注册表登记为 external write 的动作。

模拟器必须：

- 复用正式参数校验、权限、幂等和错误合同。
- 不请求真实 Provider 写权限，不调用真实写接口。
- 生成稳定且可识别的内部模拟 ID。
- 把页面需要的业务结果写入展示库。
- 在正式控制库记录不含秘密的 `DEMO_EXTERNAL_ACTION_SIMULATED` 审计。

真实读取动作可以继续读取 Provider，并把允许的业务结果写入当前业务库。

### 6.9 AI

- AI Provider URL、Secret、配置、租约和用量审计始终读取正式控制库。
- Prompt 组装、公司上下文和 Skill 业务查询读取当前业务库。
- 展示环境可以真实调用模型并产生真实 Token 消耗。
- AI 审计增加数据环境字段，但不记录 Prompt、答案、业务上下文、Skill 参数值或 Provider 原始响应。
- 未登记 `appId`/`featureId` 的 AI 功能继续 fail closed。

### 6.10 前端缓存与并发

- 内存缓存、`localStorage`、`sessionStorage` 和 IndexedDB 命名空间包含环境 ID。
- 环境切换时中止旧请求，清空当前内存状态并重新加载。
- 晚到的旧环境响应不得写入新环境状态。
- 每个业务写请求携带环境版本。
- 服务端对过期版本返回稳定 409 错误。
- 普通业务页面不读取或展示 D1 绑定名和数据库 ID。

## 七、设置页需求

### 7.1 卡片内容

- 当前数据环境。
- “正式数据库”和“展示数据库”选择项。
- “更新展示数据库”按钮。
- 最近成功更新时间。
- 来源数据截止时间和覆盖范围。
- 最近校验结果。
- 更新失败时的中文错误说明和重试入口。

### 7.2 状态

- 加载中。
- 正式数据库已选。
- 展示数据库已选。
- 展示数据库未准备。
- 切换中。
- 更新中。
- 更新成功。
- 更新失败且展示环境不可使用。
- 结构不一致。
- 无权限。

### 7.3 可访问性

- 环境选择支持键盘操作。
- 按钮有明确中文可访问名称。
- 更新进度和结果通过 `aria-live` 宣告。
- 禁用态说明原因。
- 焦点在切换或刷新后返回卡片标题。
- 满足 WCAG AA 对比度和常见笔记本宽度、窄屏布局。

## 八、数据定义

### 8.1 数据环境授权

| 字段 | 说明 |
|---|---|
| token_hash | 随机 HttpOnly 授权值的哈希 |
| actor_id | 当前最高权限身份 |
| environment_id | `production` 或 `display` |
| environment_version | 选择时确认的环境版本 |
| expires_at | 授权过期时间 |
| revoked_at | 撤销时间 |
| created_at / updated_at | 创建和更新时间 |

### 8.2 展示环境状态

| 字段 | 说明 |
|---|---|
| status | `empty`、`ready`、`refreshing` 或 `failed` |
| enabled | 是否允许使用展示环境 |
| version | 成功刷新后递增的环境版本 |
| active_job_id | 当前刷新任务 |
| rule_version | 展示转换规则版本 |
| source_updated_at | 正式来源截止时间 |
| coverage_json | 安全的表和日期覆盖摘要 |
| validation_json | 不含业务明细的校验摘要 |
| last_error_code | 最近安全错误码 |
| updated_by / updated_at | 操作者与时间 |

### 8.3 刷新任务

| 字段 | 说明 |
|---|---|
| job_id | 任务 ID |
| status | `queued`、`running`、`succeeded` 或 `failed` |
| stage | preflight、clear、copy、transform、recalculate、validate、activate |
| current_table | 当前表 |
| cursor_json | 可续跑游标 |
| source_version | 来源快照/修订标识 |
| rule_version | 转换规则版本 |
| counts_json | 安全计数 |
| last_error_code | 安全错误码 |
| started_at / finished_at | 运行时间 |
| actor_id | 操作者 |

### 8.4 环境审计

审计只记录操作者、环境、动作、任务 ID、版本、结果码和时间，不记录业务数据、凭证或 Provider 原始响应。

- `DATA_ENVIRONMENT_PERMISSION_DENIED`
- `DATA_ENVIRONMENT_BINDING_MISSING`
- `DATA_ENVIRONMENT_DISABLED`
- `DATA_ENVIRONMENT_NOT_READY`
- `DATA_ENVIRONMENT_MAINTENANCE`
- `DATA_ENVIRONMENT_REFRESH_FAILED`
- `DATA_ENVIRONMENT_SCHEMA_MISMATCH`
- `DATA_ENVIRONMENT_VERSION_CONFLICT`
- `DATA_ENVIRONMENT_REFRESH_IN_PROGRESS`
- `DATA_ENVIRONMENT_REFRESH_NOT_FOUND`
- `DATA_ENVIRONMENT_TABLE_NOT_REGISTERED`
- `DATA_ENVIRONMENT_VALIDATION_FAILED`

错误响应不得暴露 D1 数据库 ID、SQL、Secret、Cookie 或 Provider 原始响应。

## 十、验收标准

### 10.1 环境与权限

- 最高权限用户可以在设置中查看、切换和刷新展示数据库。
- 非最高权限 API 调用返回 403。
- 当前浏览器切换不影响另一浏览器会话。
- 无选择时使用正式业务库。
- 浏览器提交任意绑定名不会改变路由目标。
- 展示环境不可用时不回退正式库。

### 10.2 数据正确性

- 两套 D1 使用相同业务迁移版本且数据库 ID 不同。
- 展示刷新完成后，允许复制的业务表覆盖校验通过。
- 敏感控制表和凭证字段未进入展示库。
- 销售可加总总计精确为正式来源的 2 倍。
- 退款率、毛利率和客单价在既定精度内一致。
- 同一任务重复执行不会变成 4 倍。
- 展示库刷新失败时无法读取半成品。

### 10.3 全系统路由

- 公司状态、数据中心、销售、商品、供应链、电商、品牌、人事、绩效和协同业务 API 均使用统一业务库。
- 治理检查会拒绝新增业务路由直接解析 `PRODUCT_FLOW_DB`。
- 控制面模块仍只使用正式控制库。
- 切换后本地缓存和 IndexedDB 不串环境。
- 过期标签页写入返回 409。

### 10.4 集成

- 展示环境真实采集只写展示库。
- 正式环境真实采集只写正式库。
- 两个环境的采集幂等键互不冲突。
- 展示环境钉钉待办、日历、审批不调用真实 Provider。
- 正式环境对外动作维持现有行为。
- AI 展示调用读取展示业务上下文，Provider 配置和用量仍写正式控制库。

### 10.5 体验

- 普通业务页面没有展示数据标签。
- 设置页显示当前环境、更新时间、覆盖和校验状态。
- 刷新过程可等待、刷新页面后可恢复查看。
- 键盘、焦点、加载、空、错误、禁用和响应式状态通过验收。

## 十一、发布与回滚

### 发布顺序

1. 创建并迁移独立 `DEMO_FLOW_DB`。
2. 在正式控制库执行控制面迁移。
3. 配置 Pages 的正式与 Preview 双绑定，并通过环境一致性检查。
4. 发布服务端路由、刷新能力和外部写模拟。
5. 发布设置页入口。
6. 最高权限用户首次刷新并完成不变量校验。
7. 在本地线上、Preview、Git 部署和当前 Wrangler CLI 四条验证通道分别验收。

### 回滚

- 将展示环境状态设为 disabled，设置页隐藏切换入口。
- 已选展示环境的业务请求返回明确禁用错误，引导切回正式环境，不自动回退。
- 回滚应用代码不删除展示 D1 和审计。
- 正式业务库不需要数据恢复。
