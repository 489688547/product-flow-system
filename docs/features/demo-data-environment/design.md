# 展示数据库与数据环境切换设计书

- 状态：产品设计已确认，待文档评审
- 日期：2026-07-23
- 适用范围：设置、共享业务数据、数据采集、外部平台动作、AI 业务数据读取

## 用户任务

最高权限用户需要为对外展示准备一套与正式系统结构一致、数据关系可信的展示数据库，并可在设置中仅为当前浏览器会话切换数据环境。展示数据库使用真实业务结构和最新真实来源，销售类可加总指标扩大为两倍，比例和均值保持原口径；正式数据库不得被展示操作修改。

## 已确认决策

1. 使用独立 Cloudflare D1，而不是在正式库中增加环境字段，也不是查询时临时乘二。
2. 设置切换只影响当前账号的当前浏览器会话，不影响其他浏览器或其他员工。
3. 普通业务页面不显示“演示数据”或“展示数据”标识；当前选择、更新时间和切回入口只在设置页显示。
4. 整个系统的业务数据切换到展示库，控制数据始终留在正式控制库。
5. 展示库由最高权限用户在设置中手动更新；更新期间旧版本继续可用。
6. 快麦、ERP 文件、浏览器采集等已登记的真实数据获取能力保持可用。展示模式下仍读取真实来源，但转换后只写展示库。
7. 钉钉待办、审批、日历等对外写操作在展示模式下模拟执行，不调用真实平台。
8. 销量、订单量、销售额、净销售额、退款额、成本和毛利等可加总指标扩大两倍；退款率、毛利率、客单价等比例或均值重新计算并保持一致。

## 方案选择

### 采用：独立展示 D1 与统一数据环境路由

正式业务库使用现有 `PRODUCT_FLOW_DB`，展示业务库新增 `DEMO_FLOW_DB`。服务端认证完成后，由统一数据环境路由为业务存储选择目标数据库。登录、会话、权限、平台凭证、个人令牌和数据环境选择始终使用正式控制库。

选择该方案的原因：

- 物理隔离明确，展示写入无法通过普通业务路径修改正式业务数据。
- 两套库可执行同一批迁移，避免每张业务表增加环境字段。
- 展示库可以保存编辑、同步和模拟动作结果，跨页面关系比查询时临时转换稳定。
- 回滚只需关闭展示入口和路由，不需要恢复正式业务数据。

### 不采用：单库环境字段

该方案需要修改几乎所有表、索引、查询和历史迁移。任何未增加环境条件的查询都可能造成串库，迁移成本和正式数据风险过高。

### 不采用：查询时临时转换

该方案无法稳定支持编辑、导入、同步、历史记录、跨 App 关系和模拟外部动作，也不符合独立数据库的目标。

## 共享能力结论

平台能力评审结论为 `抽取共享能力`。

当前多个业务 App 和服务端存储模块直接解析 `PRODUCT_FLOW_DB`，数据库选择具有跨设置、数据中心、商品、销售、供应链、人事、协同和 AI Skill 的真实消费者。新增一个只服务于设置页的局部判断会造成绕过和串库风险，因此应抽取单一职责的服务端数据环境路由。

该共享能力负责：

- 从已认证服务端会话解析当前数据环境。
- 返回正式或展示业务数据库，不接收浏览器自报的数据库绑定名。
- 为每次业务请求提供稳定的环境 ID 和环境版本。
- 拒绝过期版本、切换前请求和缺失绑定。
- 输出安全审计字段，不输出 D1 ID、凭证或内部绑定详情。

该能力不负责：

- 登录、角色判断或业务权限本身。
- Provider payload 转换。
- 销售指标翻倍计算。
- 展示页面的视觉组合。

## 数据库与控制面

### 正式控制库

以下数据始终使用 `PRODUCT_FLOW_DB`，不随设置切换：

- 登录会话和组织身份。
- 角色、权限和最高权限判断。
- 当前浏览器会话的数据环境选择。
- 平台连接保险箱、密钥密文和安全审计。
- 本地线上账号个人令牌、生产写入解锁和写前快照。
- AI Provider 配置、Token 用量和受限调用审计。

当前浏览器的数据环境选择绑定到当前服务端会话，而不是全公司账号配置或普通 `localStorage`。同一浏览器会话的标签页共享选择，其他浏览器拥有独立选择。

### 业务数据库

以下业务数据通过数据环境路由选择 `PRODUCT_FLOW_DB` 或 `DEMO_FLOW_DB`：

- 公司共享状态及其分片。
- 数据中心元数据、数据口径、质量问题和同步记录。
- 销售日事实、商品主数据和经营数据。
- 产品、供应链、店铺运营、人事绩效和跨 App 协同业务记录。
- AI Skill 读取的业务事实。

展示库不保存会话、凭证明文或密文、个人令牌、生产解锁和秘密审计。手机号、收货地址、私人邮箱等个人敏感字段在复制到展示库前脱敏。

## 服务端路由与请求顺序

请求顺序固定为：

1. 正式控制库解析登录会话。
2. 服务端校验当前角色和路由权限。
3. 数据环境路由读取该会话的环境选择与版本。
4. 业务存储获得明确的目标 D1。
5. Provider adapter 执行已授权的真实读取或模拟写入。
6. 结果写入选定业务库，并记录不含秘密的控制面审计。

浏览器不能通过 Query、Header、请求体或本地状态直接指定 `PRODUCT_FLOW_DB`、`DEMO_FLOW_DB` 或 Cloudflare 数据库 ID。无环境选择时默认正式库；已选择展示库但绑定、版本或结构无效时直接拒绝请求，不静默回落正式业务库。

## 展示数据库更新

设置中的“更新展示数据库”创建受控全量刷新任务。任务按表和游标分批执行，支持安全续跑，避免 Pages Function 单次请求超时；设置页轮询安全状态，不在浏览器中搬运业务数据。

1. 校验最高权限身份、两库结构版本和展示库容量。
2. 读取正式业务库允许复制的业务表。
3. 对个人敏感字段脱敏，对销售事实执行确定性转换。
4. 写入展示库的候选版本。
5. 校验表覆盖、关系完整性、销售两倍不变量和比例不变量。
6. 全部通过后切换展示库当前版本；失败时保留上一版本。
7. 记录来源时间、数据覆盖、结果码和操作者，不记录敏感值。

同一份来源和规则版本必须生成相同的展示编号与相同结果，刷新页面不得随机变化。更新任务具有幂等键，重复点击不得产生四倍或重复累加。

## 销售数据转换规则

转换发生在标准销售事实写入边界，而不是页面显示层。所有已登记采集入口和手动全量更新复用同一转换器。

| 类型 | 字段示例 | 展示规则 |
|---|---|---|
| 可加总数量 | 订单量、净销量、销售件数 | 乘以 2 |
| 可加总金额 | 销售额、净销售额、退款额、成本、毛利、发货前退款、发货后退款 | 乘以 2 |
| 比例 | 退款率、毛利率、转化率 | 从转换后的基础字段重新计算，保持来源比例 |
| 均值 | 客单价、件单价、平均成本 | 从转换后的分子和分母重新计算，保持来源均值 |
| 维度 | 日期、平台、商品、69 码、部门 | 保持不变 |
| 单价与规则 | 商品单价、账期、税率、指标定义 | 保持不变 |
| 非销售事实 | 库存、采购、人员、项目状态 | 原样复制或按脱敏规则复制，不乘以 2 |

如果存在订单明细，系统使用确定性的展示订单 ID 生成第二份关系完整的记录，使订单数和销量真实增加；不得仅修改页面汇总。当前只具备日聚合的来源继续在聚合事实层精确乘二，不虚构不存在的客户明细。

校验要求：

- 转换后订单量、销量和各金额总计精确为来源的两倍。
- 退款率、毛利率和均值在既定精度内与来源一致。
- `销售额 - 退款额 = 净销售额`、`净销售额 - 成本 = 毛利` 等现有业务关系继续成立。
- 上海时区、订单创建时间口径和排除“其它”平台的现有报表约定不变。

## 真实数据采集

展示模式保留现有同步、导入和采集入口。

- 快麦使用集成注册表当前登记的官方文件与浏览器采集主通道；不得把尚未接通的开放平台 API 描述为可用能力。
- ERP 文件导入和网页采集仍经过现有服务端授权、adapter、幂等、游标和审计边界。
- Provider 读取成功后，标准事实写入器根据当前数据环境决定是否执行展示转换。
- 展示模式只改变写入目标和销售转换，不绕过真实来源的权限、超时、重试和错误处理。
- 单次同步只更新该数据源负责的范围，不重建无关业务表。

## 外部写操作与 AI

### 对外写操作

钉钉待办、审批、日历及其他会改变外部平台状态的动作，在展示模式下进入模拟 adapter：

- 复用正式请求校验、权限和幂等规则。
- 生成与正式成功合同兼容的安全结果。
- 只在展示业务库记录模拟状态和后续页面所需的数据。
- 不获取真实写入授权、不调用外部写接口。
- 控制面审计记录 `executionMode=display` 和安全结果码。

### AI

AI 功能继续调用已配置的真实模型 Provider，但业务上下文只从当前选中的业务库读取。展示模式的调用仍计入真实 Token 用量，控制面审计标记执行环境；提示词、回答、业务上下文和 Skill 参数值仍不得写入审计。

## 设置页

### 信息层级

主信息：

- 当前数据环境：“正式数据库”或“展示数据库”。
- 两项切换控件。
- “更新展示数据库”主操作。

辅助信息：

- 最近成功更新时间。
- 来源数据截止时间和覆盖范围。
- 最近校验结果。

低频信息：

- 失败详情与重试入口。
- 结构版本和就绪状态的中文摘要，不展示内部数据库 ID。

### 交互流程

1. 最高权限用户进入设置页的数据环境卡片。
2. 选择目标环境并确认。
3. 服务端更新当前浏览器会话的环境选择和版本。
4. 前端中止旧请求、清空内存数据、按环境隔离缓存并重新加载当前页面。
5. 切回正式库使用相同流程。

普通业务页面不增加展示标识、颜色条或水印。设置页是唯一可见的当前环境确认入口。

### 权限

只有 active executive 或等价的最高权限身份可以查看、切换和更新展示数据库。前端隐藏不是授权边界，所有操作均由服务端重新校验。

## 组件复用

- 复用现有 `SettingsPage`、`PageHeader`、`Button`、状态徽标和确认对话框。
- 数据环境卡片保持设置页现有信息密度，不建立新的全局导航或独立 App。
- 数据刷新状态复用现有加载、成功、错误和禁用语义；业务页面不感知数据库绑定名称。

## 新增组件

### `DataEnvironmentSettings`

职责：展示当前环境、切换环境、触发展示库更新并反馈状态。

状态：

- 加载当前环境。
- 正式库已选。
- 展示库已选。
- 展示库未就绪。
- 切换中。
- 更新中。
- 更新成功。
- 更新失败但旧版本可用。
- 无权限。

键盘行为：

- 环境选项可使用 Tab 聚焦和方向键切换。
- 确认、更新和重试按钮具有明确可访问名称。
- 切换后焦点返回卡片标题，更新结果通过 `aria-live` 宣告。

复用边界：该组件只负责设置页组合，不直接访问 D1、Provider 或计算销售指标。

## 页面状态与文案

- 加载：“正在读取数据环境…”
- 未就绪：“展示数据库尚未就绪，正式数据库不受影响。”
- 更新按钮：“更新展示数据库”
- 更新中：“正在读取真实业务数据并生成展示版本…”
- 成功：“展示数据库已更新，销售数量和金额已按规则生成。”
- 校验失败：“新版本未通过数据校验，已继续使用上一版本。”
- 结构不一致：“展示数据库结构落后于当前版本，请先完成数据库升级。”
- 无权限：“只有最高权限账号可以管理数据环境。”
- 过期请求：“数据环境已切换，请重新执行当前操作。”

## 缓存与并发

- 浏览器缓存、IndexedDB 和内存 Query Key 必须包含数据环境 ID。
- 切换时取消旧环境中的进行中请求，并使晚到响应失效。
- 每个业务写请求携带服务端下发的环境版本；版本不一致返回稳定冲突错误。
- 展示库更新、来源同步和页面编辑使用明确的幂等键。
- 同一展示刷新任务只允许一个运行实例。

## 错误处理

建议登记以下稳定错误：

- `DATA_ENVIRONMENT_PERMISSION_DENIED`
- `DATA_ENVIRONMENT_BINDING_MISSING`
- `DATA_ENVIRONMENT_SCHEMA_MISMATCH`
- `DATA_ENVIRONMENT_VERSION_CONFLICT`
- `DEMO_DATA_REFRESH_IN_PROGRESS`
- `DEMO_DATA_REFRESH_FAILED`
- `DEMO_DATA_INVARIANT_FAILED`

读取型真实采集可沿用既有安全重试策略；写请求和环境切换不自动重试。错误响应不得返回 D1 ID、绑定细节、凭证、Provider 原始响应或堆栈。

`DEMO_EXTERNAL_ACTION_SIMULATED` 是模拟外部动作的安全成功结果码，只进入兼容响应和审计，不作为错误提示。

## 环境与集成影响

### 环境依赖

- 新增 Cloudflare D1 绑定 `DEMO_FLOW_DB`。
- Preview 与 Production 必须明确绑定展示数据库；缺失时展示功能为阻断状态，但正式业务库继续可用。
- 两个业务数据库执行同一批业务迁移并通过结构一致性检查。
- `docs/platform/environment-capabilities.json`、生成清单、环境就绪检查和 Pages 环境一致性脚本必须同步更新。
- 本地沙箱仍是测试性写入的唯一环境，不因展示库而废除。

### 集成路由

- `cloudflare-d1`：新增展示业务数据库、结构一致性和数据刷新。
- `cloudflare-pages`：新增绑定、会话级路由、中间件和就绪检查。
- `kuaimai`：真实读取后按当前环境写入；当前主通道仍为官方文件与浏览器采集。
- `erp-file-import`：真实文件导入后按当前环境写入。
- `dingtalk`：展示模式对外写动作进入模拟 adapter，登录和组织控制面仍使用正式路径。

实现 PR 必须声明以上 `Integration-Impact`，并把数据环境路由、绑定、错误合同、模拟动作和迁移规则反写到平台文档、集成注册表或 ADR。

## 安全与隐私

- 展示库不得保存平台密钥、会话 Token、Cookie、个人令牌、生产解锁或凭证明文/密文。
- 真实数据读取继续使用现有服务端权限和 Provider adapter，不向浏览器下发凭据。
- 个人敏感字段在进入展示候选版本前脱敏。
- 模拟外部动作不得调用真实写接口。
- 所有环境选择、更新、同步和模拟动作写入无秘密审计。

## 验收与测试

### Domain

- 每个登记的可加总销售字段精确乘二。
- 比例和均值从转换后的基础字段重算并保持一致。
- 转换幂等，同一来源重复处理不会变成四倍。
- 明细复制保持外键、商品、日期和平台关系。

### API 与权限

- 未登录、非最高权限和只读身份无法切换或更新。
- 浏览器不能伪造环境 ID 或绑定名。
- 正式与展示请求访问各自 D1。
- 环境版本冲突、缺失绑定和结构不一致返回稳定安全错误。
- 展示模式真实采集只写展示库。
- 展示模式外部写动作不调用 Provider。

### 数据隔离

- 展示更新、同步、编辑和模拟动作前后，正式业务表内容与版本不变。
- 一个浏览器的选择不影响另一个浏览器或其他账号。
- 缓存、旧标签页、晚到响应和并发请求不得串库。
- 控制数据始终读取正式控制库。

### UI

- 设置页覆盖加载、未就绪、切换中、更新中、成功、失败、无权限和禁用状态。
- 键盘、焦点、`aria-live`、窄屏、笔记本宽度和钉钉 WebView 可用。
- 普通业务页面不出现展示标识。
- 切换后当前页面重新读取目标环境数据。

### 环境与发布

- 环境能力清单、生成模块、Wrangler 3.x Pages Functions 兼容测试和两库迁移检查通过。
- 本地沙箱、正式库只读路径、展示库读写路径、Preview 和 Production 分别验证。
- 线上验证至少核对一个日期范围：可加总销售指标为来源两倍，退款率和毛利率一致，正式库未变化。

## 上线顺序

1. 创建展示 D1，不接入用户流量。
2. 对展示库执行当前全部业务迁移。
3. 部署环境能力、数据环境路由和权限接口，但保持设置入口关闭。
4. 生成第一版展示数据并完成不变量校验。
5. 开启最高权限设置入口。
6. 验证切换、真实采集、模拟外部动作和正式库不变。

## 回滚

- 关闭设置入口和展示数据环境功能开关。
- 服务端拒绝新的展示切换；现有展示会话要求重新选择正式库。
- 保留展示 D1 供排查，不让业务路由继续访问。
- 正式业务库、登录控制面和 Provider 凭证不需要恢复或迁移。
- 如单个采集入口不兼容，先禁止该入口的展示写入，不影响正式路径和其他展示能力。

## 非目标

- 不把展示库作为开发测试环境；开发试验仍使用本地沙箱。
- 不允许普通员工或外部访客管理数据环境。
- 不把尚未接通的快麦开放平台 API 伪装为已接通。
- 不为普通业务页面增加展示水印、横幅或颜色主题。
- 不改变现有销售时间口径、平台排除规则和正式数据。
