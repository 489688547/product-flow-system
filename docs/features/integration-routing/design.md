# 外部平台地图交互设计

## 信息架构

入口位于“说明书 → 平台能力 → 外部平台地图”。现有说明书左侧目录保持不变，选中该文档时中间正文替换为专用平台地图，右侧目录不再占用空间。

## 页面结构

1. 顶部说明当前数据由公开注册表和登录后内部资料合并。
2. 搜索框支持平台名称、能力、问题和关键词。
3. 状态筛选显示全部及四种状态的数量。
4. 列表行显示平台名称、状态、能力摘要和资料完整度。
5. 详情区显示能力、问题、依赖关系、官方文档和内部资料。
6. 总经办用户可在详情内展开编辑区；保存期间禁用按钮并显示明确状态。

## 状态

- 加载：公开资料立即可见，内部字段显示骨架占位。
- 空结果：解释当前搜索或筛选没有匹配，并提供清除筛选。
- 内部 API 错误：显示非阻断警告，公开内容照常展示。
- 内部资料缺失：列出可由管理员补充的字段。
- 过期：最近验证时间超过 180 天时显示需复核。
- 无编辑权限：不渲染编辑控件，服务端仍独立鉴权。

## 视觉与可访问性

- 复用说明书的字号、间距、颜色和按钮体系。
- 使用列表与详情分栏，不堆叠同尺寸卡片。
- 状态同时使用文字和颜色，不只依赖颜色。
- 搜索、筛选、平台选择和编辑表单均保留键盘焦点。
- 窄屏按搜索、筛选、列表、详情的顺序单列排列。
