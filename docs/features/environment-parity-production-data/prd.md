# 环境一致性与生产数据访问 PRD

## 文档状态

- 状态：开发中
- 负责人：总经办 / 平台维护
- 最近更新：2026-07-18

## 背景与问题

测试、本地预览和生产环境目前依靠人工分别配置 Cloudflare Pages、D1 与钉钉应用。现有发布说明只提醒 Production 和 Preview 都应配置，但没有可执行检查，因此会出现测试环境能够同步钉钉、生产环境却缺少应用配置或数据库绑定的问题。另一方面，本地测试账号需要读取真实业务数据，并在明确授权时直接修正生产数据库；当前本地状态接口仍以本地文件为主，缺少稳定、可审计的生产数据通道。

## 目标

- 用一份机器可检查的环境能力清单定义本地、预览和生产的必要配置。
- 在说明书中用中文展示当前环境、生产环境和各集成能力的就绪状态，不展示密钥值。
- 本地测试默认实时读取生产 D1 的真实公司状态。
- 当前指定的钉钉最高权限账号可以显式解锁 15 分钟生产写入，直接修改生产 D1。
- 每次跨环境生产写入具备版本冲突检查、写前快照、审计和回滚能力。
- 本地测试中的钉钉待办、消息、审批、日历等外部副作用默认禁止，数据库写入权不等于外部操作权。
- PR、构建和部署验证能发现能力清单、生成模块、生产运行配置或 D1 表结构缺失。

## 非目标

- 不复制生产密钥到仓库、浏览器状态或 D1 业务记录。
- 不让所有总经办账号自动获得跨环境生产写入权。
- 不在本期开放测试环境真实创建钉钉待办、日历或审批。
- 不为每个外部平台重建独立环境检查页面。
- 不改变生产环境内已有业务角色与正常写入流程。

## 用户与权限

- 已登录员工：可查看脱敏后的环境能力状态。
- 平台管理员：可查看缺少的变量名、绑定名和修复说明。
- 生产数据读取账号：必须存在于 D1 `production_data_access_tokens`，并具有 `read` 能力。
- 生产数据写入账号：必须同时具有 `executive` 身份、D1 令牌 `write` 能力和有效的 15 分钟写入解锁。
- 初始只向当前指定的最高权限账号签发令牌；稳定匹配使用钉钉 `userId` 与 `unionId`，不使用姓名。
- 外部平台真实操作使用独立能力，本期测试环境不签发。

## 当前流程

1. 开发者在本地或 Preview 测试功能。
2. 钉钉变量和 D1 绑定由人工在 Cloudflare 分环境维护。
3. 测试通过后部署；生产缺少配置时只能在运行中发现。
4. 本地状态写入本地文件或浏览器缓存，无法形成受控的生产数据修正和审计。

## 目标流程

1. 开发和 CI 校验环境能力清单及其生成代码一致。
2. 本地服务使用个人生产数据令牌，通过生产网关实时读取公司状态。
3. 普通本地操作保持只读；需要修正线上数据时，最高权限账号填写原因并输入确认短语解锁 15 分钟。
4. 每次生产写入先检查 `baseUpdatedAt`，再保存写前快照、写入数据并记录审计。
5. 发现错误时，最高权限账号从最近审计记录发起回滚；回滚同样生成快照和审计。
6. 发布后运行生产就绪检查；缺少 D1、钉钉变量或必要数据表时，发布验证失败。

## 业务规则

- 环境能力清单是配置名称、适用环境、严重级别和检查方式的唯一事实源；密钥值永不进入清单。
- 本地生产数据令牌只保存在被忽略的 `.env` 中，生产数据库只保存 SHA-256 哈希。
- 解锁确认短语固定为“修改线上真实数据”，原因长度为 4 至 200 个字符。
- 解锁有效期固定 15 分钟；锁定、过期、撤销或令牌失效后立即拒绝写入。
- 写入必须携带最近读取到的 `baseUpdatedAt`；不一致返回 409，不覆盖新版本。
- 快照保存最近 30 次跨环境写入，超过数量按最早记录清理。
- 审计不保存访问令牌、Cookie、密钥或完整请求头。
- 数据库写入授权不允许调用钉钉、快麦刷新等外部副作用接口。

## 数据定义

- `environment-capabilities.json`：能力 ID、中文名称、关联平台、适用环境、必需变量、D1 绑定和表结构。
- `production_data_access_tokens`：令牌哈希、钉钉稳定身份、能力列表、有效期和撤销时间。
- `production_write_unlocks`：解锁令牌哈希、访问令牌哈希、原因、开始和过期时间。
- `production_data_snapshots` / `production_data_snapshot_parts`：写前状态版本、更新时间和分片载荷。
- `production_data_audit`：动作、来源环境、操作者、原因、写前写后版本、请求 ID、结果和时间。
- 环境状态响应只返回配置名称与布尔状态，不返回配置值。

## 异常与边界

- 缺少本地生产令牌：保留本地文件模式并明确提示未连接生产数据。
- 令牌不存在、过期或撤销：返回 401，不透露账号是否存在。
- 有读取权但无写入权：返回 403。
- 未解锁或解锁过期：返回 423，并提示重新解锁。
- 基线版本冲突：返回 409，要求刷新真实数据后重新操作。
- D1 或钉钉配置缺失：就绪状态标为阻断；不回显任何值。
- 快照或审计写入失败：业务写入不执行，避免出现不可追溯修改。
- 外部平台写操作来自本地测试：返回稳定错误 `EXTERNAL_ACTION_DISABLED_IN_TEST`。

## 验收标准

- 清单缺字段、生成模块漂移或集成平台 ID 不存在时，CI 失败。
- 生产就绪接口在缺少 `PRODUCT_FLOW_DB`、`DINGTALK_APP_KEY` 或 `DINGTALK_APP_SECRET` 时返回 `ready: false` 和缺少项名称。
- 未认证访问就绪状态被拒绝；有效员工会话只得到脱敏结果。
- 有效读取令牌可以实时读取生产公司状态；数据库中不存在原始令牌。
- 未解锁、错误账号、过期解锁和旧版本写入全部被拒绝。
- 有效最高权限账号解锁后可以写入，产生快照和审计，并可回滚。
- 本地钉钉真实写操作保持禁止。
- 说明书可查看环境状态，覆盖加载、错误、缺失、就绪、无权限和解锁状态。

## 上线与回滚

- 先部署兼容代码和幂等建表，再通过受控命令签发当前账号令牌并写入本地 `.env`。
- 部署后先执行只读生产就绪检查，再执行一条可回滚的测试字段修改与回滚验收。
- 回滚应用时撤销生产数据令牌并删除本地 `.env` 令牌；新增表可保留，不影响旧版本。
- 若跨环境写入异常，立即撤销令牌并回滚到写前快照；不删除审计。
