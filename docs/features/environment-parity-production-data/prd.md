# 本地线上账号一致性 PRD

## 文档状态

- 状态：已评审，开发中
- 负责人：总经办 / 平台维护
- 最近更新：2026-07-20

## 背景与问题

当前本地开发同时存在 Vite、旧 Node 预览和 Pages Functions 三种运行方式。本地 Pages Functions 使用硬编码的“周荣庆”假身份，且统一拒绝非 GET 请求；旧 Node 预览又单独阻止钉钉待办、日历和快麦刷新。结果是本地看到的数据、身份、权限和外部动作与线上不一致，功能只能部署后才能完整验证。

当前只有一名开发者，线上尚无其他业务用户。产品负责人已明确授权：本地开发使用自己的真实最高权限账号，直接读写线上业务数据，也可以执行该账号在线上允许的钉钉、快麦等真实外部动作。开发与线上唯一允许的差异是代码是否已经部署。

## 目标

- 本地统一使用 Pages Functions 运行完整后端，不再使用假身份或功能残缺的本地 API 实现。
- 本地通过仅存在于 `.env` 的个人令牌，还原 D1 中当前账号的真实组织身份和最高权限。
- 本地读取、创建、修改和删除业务数据时，走与线上相同的 API、D1 表、版本检查和业务审计。
- 本地可以发送钉钉待办、创建日历、同步组织和触发快麦同步等真实动作，权限与线上同账号一致。
- 页面始终明确显示“本地代码 · 线上真实环境”和当前真实账号，避免误认成沙盒。
- CI 锁定本地身份、数据、外部动作和非本地域名安全边界，所有分支继承同一规则。

## 非目标

- 不把个人令牌、平台密钥或 Cookie 放入浏览器、仓库、日志或公开文档。
- 不让任意本地用户、任意令牌或任意非本地主机获得最高权限。
- 不绕过各业务 API 自己的角色、版本、幂等、审计或平台权限校验。
- 不承诺对外部平台动作提供自动回滚；钉钉、快麦按各自已有幂等和错误契约处理。
- 不删除现有受控生产数据网关；它保留为运维修复与回滚工具，但不再是本地日常开发主链路。

## 用户与权限

- 本地线上账号：个人令牌必须存在于 `production_data_access_tokens`，具备请求方法所需的 `read` 或 `write` 能力，并与 `product_flow_org_members` 中 active、`executive` 的 `userId` 与 `unionId` 同时匹配。
- 本地 GET/HEAD 请求需要 `read` 能力；POST/PUT/PATCH/DELETE 请求需要 `write` 能力。
- 通过校验后，中间件注入真实 `corpId`、`userId`、`unionId`、姓名、部门、职务和角色，`loginMode` 为 `local-online-account`。
- 钉钉、快麦等真实动作不单独授予“本地特权”；请求进入现有线上路由后，继续使用真实账号角色、目标用户范围、平台连接和提供商权限校验。
- 非 localhost、127.0.0.1 或 ::1 的请求永远不能使用本地线上账号模式。

## 当前流程

1. 本地 UI 在会话失败时降级为浏览器硬编码的最高权限假账号。
2. Pages Functions 看到 `LOCAL_LIVE_D1_PREVIEW=1` 后注入另一个硬编码假账号。
3. 中间件对本地非 GET 请求统一返回 403。
4. 旧 Node 预览为部分 API 提供本地文件模拟，并对钉钉和快麦真实动作返回 `EXTERNAL_ACTION_DISABLED_IN_TEST`。
5. 开发者必须部署后才能验证完整的身份、D1 写入和外部平台动作。

## 目标流程

1. 开发者双击“启动服务”或执行标准命令，Vite 与 Pages Functions 共同启动，浏览器只访问 `http://127.0.0.1:8127`。
2. Pages Functions 只在回环主机且 `LOCAL_ONLINE_ACCOUNT_MODE=1` 时，从服务端环境读取 `PRODUCTION_DATA_ACCESS_TOKEN`。
3. 中间件对令牌做 SHA-256 哈希，在远程生产 D1 校验令牌与真实组织身份，并按请求方法校验 `read` 或 `write`。
4. 校验成功后注入真实线上会话，后续路由不区分本地和已部署版本。
5. 页面显示固定风险条，标明当前是本地代码、线上真实环境、真实账号及“操作立即生效”。
6. 令牌缺失、失效、账号不再 active 或最高权限被撤销时，本地直接阻断并给出中文修复提示，不回退到假账号。

## 业务规则

- `LOCAL_ONLINE_ACCOUNT_MODE` 只是显式开关，不构成授权；个人令牌、回环主机、D1 身份和能力校验缺一不可。
- 个人令牌原文只由 Wrangler 从被忽略的 `.env` 加载到服务端环境，浏览器响应、前端 bundle 和日志不得出现令牌值。
- 中间件每次请求校验当前组织身份，权限撤销后下一次请求立即失效。
- 本地线上账号模式不要求额外 15 分钟解锁，因为它不是跨环境代理写入，而是在本地运行正式应用并以真实账号登录；业务 API 原有的版本检查、快照或审计继续执行。
- 数据库能力与外部动作仍是不同执行边界，但两者都由同一个真实线上会话进入各自现有权限校验，不再按“本地”统一禁止外部动作。
- 旧 Node 预览不再作为支持的完整业务运行时；标准本地入口必须启动 Vite + Pages Functions。
- 生产域名即使误配 `LOCAL_ONLINE_ACCOUNT_MODE` 和个人令牌，也不能触发本地账号注入。

## 数据定义

- `LOCAL_ONLINE_ACCOUNT_MODE`：本地线上账号模式显式开关，值为 `1`。
- `PRODUCTION_DATA_ACCESS_TOKEN`：个人原始令牌，仅存在于被忽略的 `.env` 和本地 Worker 环境。
- `production_data_access_tokens`：保存令牌哈希、稳定身份、能力、有效期和撤销状态。
- `product_flow_org_members`：真实公司身份来源，使用 `corp_id`、`user_id`、`union_id`、`name`、`department`、`title`、`role`、`active`。
- 本地会话增加 `loginMode: "local-online-account"`，不新增数据库表。

## 异常与边界

- 缺少开关：按正式线上会话流程处理，不注入本地账号。
- 缺少个人令牌：返回 `LOCAL_ONLINE_TOKEN_REQUIRED`，不回退假身份。
- 令牌无效、过期或撤销：返回 `PRODUCTION_TOKEN_INVALID`。
- 读令牌发起写请求：返回 `PRODUCTION_CAPABILITY_REQUIRED`。
- 账号停用、稳定身份不匹配或不再是最高权限：返回 `PRODUCTION_ROLE_REQUIRED`。
- D1 未绑定：返回明确的本地线上环境配置错误。
- 钉钉或快麦配置缺失：由现有提供商路由返回对应稳定错误，不伪装成功。
- 外部平台成功但浏览器断线：按原有幂等键、查询或同步状态恢复，不能自动重复创建。
- 非本地主机：忽略本地线上模式并要求正式钉钉会话。

## 验收标准

- 本地 GET 会话返回 D1 中真实账号的姓名、稳定身份、部门、职务和 `executive` 角色，不含硬编码身份。
- 本地 POST/PUT/PATCH/DELETE 在有效 `write` 令牌下进入目标路由，不再出现“线上 D1 本地预览为只读模式”。
- 本地钉钉待办、日历和快麦动作请求进入正式适配器，不再出现 `EXTERNAL_ACTION_DISABLED_IN_TEST`。
- 只读令牌不能写，失效令牌不能读，非本地主机不能借本地令牌登录。
- 浏览器源码、响应体、构建产物和日志中找不到个人令牌。
- 页面在桌面和窄屏都持续显示本地线上真实环境警示及真实账号。
- 标准启动入口只暴露一个用户访问地址，并支持前端热更新和 Functions 本地执行。
- 生产环境现有钉钉登录、业务权限和 API 行为不变。

## 上线与回滚

- 本次不新增 D1 表；部署前更新环境清单、集成注册表、生成模块、平台文档和 ADR。
- 本地先用只读接口验证真实账号，再使用可查询或幂等的方式分别验证一个数据库写入口和外部适配器；不得为了测试创建无业务意义的真实待办或日历。
- 部署后验证生产域名仍要求正式钉钉会话，并执行生产就绪检查。
- 回滚时恢复旧中间件与启动入口；撤销个人令牌或删除本地 `.env` 中的令牌可立即关闭本地线上账号模式，生产数据和外部平台不做迁移回滚。
