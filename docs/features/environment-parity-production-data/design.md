# 本地线上账号一致性设计书

## 用户任务

产品负责人在本地修改尚未部署的代码时，需要直接用自己的真实线上账号验证完整业务：查看和修改线上数据、发送钉钉待办、创建日历、同步组织和触发快麦同步。用户不需要理解多套环境，只需要明确当前运行的是“本地代码”，但所有数据和动作都是真实线上行为。

## 信息层级

1. 主信息：`本地代码 · 线上真实环境`、真实账号姓名、`所有操作立即生效`。
2. 辅助信息：数据、钉钉、快麦等能力是否就绪，以及缺少的配置名称。
3. 低频信息：令牌状态、生产数据网关审计和运维修复入口。

## 页面结构

- 应用主内容顶部新增全局环境警示条，所有业务 App 都可见，不只在数据中心出现。
- 警示条显示真实会话中的姓名，不读取或展示令牌信息。
- “说明书 → 平台能力 → 环境与生产数据”继续展示详细能力状态；默认说明书仍是“使用手册”。
- 旧的“解锁 15 分钟生产写入”保留给受控生产数据网关，不作为本地线上账号日常操作步骤。

## 交互流程

1. 标准本地启动器同时启动 Vite 和 Wrangler Pages Functions；Vite 在 `127.0.0.1:8127` 提供页面和热更新，并把 `/api` 代理到内部的 Wrangler `127.0.0.1:8132`。
2. 页面调用 `/api/auth/session`，中间件在服务端验证个人令牌并返回真实账号。
3. 验证期间保持加载态；验证失败展示明确错误和“检查本地线上账号配置”入口，不自动进入假账号。
4. 验证成功后显示全局警示条，所有业务按钮按真实角色正常启用。
5. 数据写入与外部动作继续使用原按钮、确认和成功反馈；不增加一套“测试动作”按钮。
6. 外部平台失败时展示原平台安全错误、请求 ID 和重试建议；成功后使用原业务记录或同步状态确认结果。

## 组件复用

- 复用应用现有顶部状态条、会话对象和说明书能力卡，不引入第二套权限组件。
- 复用 Pages Functions 的 `context.data.session` 作为所有 API 的唯一身份来源。
- 复用 `productionDataAccess` 的令牌哈希、过期、撤销和组织身份校验，扩展其返回字段，不复制验证逻辑。
- 复用钉钉、快麦正式适配器；业务组件不得直接调用提供商。

## 新增组件

### `LocalOnlineEnvironmentBanner`

- 职责：当 `sessionUser.loginMode === "local-online-account"` 时持续说明真实环境风险。
- 属性：`sessionUser`。
- 展示：环境名称、账号姓名、立即生效提示；不展示令牌、userId 或 unionId。
- 状态：正常显示；未进入本地线上模式时不渲染。
- 键盘：纯状态信息不进入 Tab 顺序，使用 `role="status"` 和可读文本。
- 复用边界：只表达运行环境，不承担权限判断或动作确认。

## 页面状态

- 加载：登录页保持会话检查提示，业务操作不可用。
- 成功：警示条显示“本地代码 · 线上真实环境 · 周荣庆 · 所有操作立即生效”。
- 配置缺失：显示缺少的变量名 `LOCAL_ONLINE_ACCOUNT_MODE`、`PRODUCTION_DATA_ACCESS_TOKEN` 或 `PRODUCT_FLOW_DB`，不展示值。
- 令牌失效：显示“本地线上账号授权已失效，请更新个人令牌”。
- 无权限：显示“当前令牌对应账号已停用或不再具有最高权限”。
- 外部平台未就绪：对应按钮按原有状态禁用或请求后显示平台错误，不降低为模拟成功。
- 网络错误：保留重试按钮，不回退本地假账号。

## 响应式与钉钉 WebView

- 警示条位于内容流顶部，不固定遮挡导航。
- 1024px 以上单行显示；窄于 760px 时分成环境、账号和风险两行。
- 390px 宽度允许文案换行，不产生横向滚动。
- 线上钉钉 WebView 不显示本地警示条，现有安全区和动态视口行为不变。

## 交互文案

- 警示标题：`本地代码 · 线上真实环境`
- 警示正文：`当前账号：{name}。数据修改、钉钉和快麦操作都会立即在线上生效。`
- 配置失败：`本地线上账号未连接，请检查个人令牌和生产 D1 绑定。`
- 权限失败：`当前个人令牌不能代表最高权限账号，请重新授权。`
- 禁止使用：`本地线上账号模式只能在本机地址使用。`

## 无障碍

- 警示条文字与背景达到 WCAG AA；风险不只靠颜色表达。
- 会话失败使用 `role="alert"`，成功环境信息使用 `role="status"`。
- 重试按钮具备可见焦点；焦点顺序保持导航、警示、页面主操作。
- 不使用闪烁或自动动画。

## 视觉验收

- 1440×900：任一业务 App，显示单行本地线上环境警示和真实账号。
- 1024×768：侧栏展开与折叠时警示条不挤压主操作。
- 390×844：警示条换行且页面无横向溢出。
- 生产域名与钉钉 WebView：不显示本地警示，原登录与布局不变。

## 服务端设计

- `authorizeProductionToken(rawToken, db, { capability })` 负责哈希、令牌状态、能力和真实组织身份校验，返回完整身份与 tokenHash。
- `localOnlineAccountSession(request, env)` 只在回环主机、显式开关和 D1 绑定存在时调用上述函数；能力由 HTTP 方法映射。
- 中间件先处理 OPTIONS，再尝试本地线上账号；成功后直接进入原路由。失败返回稳定安全错误，不继续正式会话或假身份回退。
- 非本地主机跳过这段逻辑，继续现有公开路径、正式 Cookie 会话和替代鉴权路径。
- 标准启动脚本由一个父进程管理 Vite 与 Wrangler，退出时同时清理子进程。
