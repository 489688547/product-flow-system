# 环境一致性与生产数据访问设计书

## 用户任务

平台管理员需要快速回答“测试和生产是否具备相同能力”，并在本地查看真实业务数据。当前指定的最高权限账号需要在明确知情时短时解锁，对生产数据库进行受审计的修正与回滚。

## 信息层级

1. 主信息：当前环境、生产是否可用、阻断项、生产写入是否解锁。
2. 辅助信息：按钉钉、Cloudflare Pages、D1、快麦等能力展示配置与表结构状态。
3. 低频信息：最近生产数据审计、写前版本、回滚入口和修复说明。

## 页面结构

在“说明书 → 平台能力”增加“环境与生产数据”文档。正文由专用状态面板替代普通 Markdown：顶部为环境总状态；中部为能力卡片；底部仅对最高权限账号显示生产写入控制与最近审计。默认仍打开“使用手册”，不改变说明书入口。

## 交互流程

1. 页面加载当前运行环境就绪状态。
2. 本地服务配置个人令牌时，同时读取生产就绪状态与最近审计；无令牌只显示配置方法。
3. 最高权限账号点击“解锁生产写入”，填写原因并输入“修改线上真实数据”。
4. 成功后顶部持续显示红色“正在修改线上真实数据”和剩余时间；可随时锁定。
5. 本地业务修改通过现有 `/api/state` 保存，由本地服务转发到生产数据网关。
6. 版本冲突时提示刷新，不提供强制覆盖。
7. 最近审计的“回滚”需要再次输入确认短语；成功后立即刷新状态和审计。

## 组件复用

- 复用 `PageHeader`、现有按钮、表单、状态色和说明书双栏布局。
- 复用 `canManagePermissions` 识别最高业务角色，但生产数据能力最终由服务端令牌表判定。
- 环境能力卡片保持业务中立，后续钉钉、快麦、阿里云能力复用同一组件。

## 新增组件

### `EnvironmentReadinessPanel`

- 职责：加载并展示脱敏环境状态、生产写入状态和审计。
- 状态：加载、就绪、警告、阻断、无权限、网络错误。
- 键盘：表单按文档顺序聚焦；状态不只依赖颜色；解锁和回滚确认对话区域获得焦点。
- 复用边界：只消费标准环境状态契约，不读取提供商密钥或直接访问 D1。

### `ProductionWriteControl`

- 职责：解锁、锁定、显示剩余时间与原因。
- 属性：`access`、`onUnlock`、`onLock`、`busy`、`error`。
- 禁用：非本地测试、无 `write` 能力、环境存在阻断项或请求进行中时禁用。

### `ProductionAuditList`

- 职责：展示最近 30 次跨环境写入和可回滚项。
- 回滚只对成功写入且快照存在的记录可用。

## 页面状态

- 加载：三条骨架卡片，并使用 `aria-busy`。
- 空数据：说明尚未配置生产数据令牌，给出本地 `.env` 配置名称，不展示值。
- 错误：展示稳定错误中文、请求 ID 和重试按钮。
- 无权限：说明当前账号只能查看环境状态。
- 禁用：按钮旁展示具体原因，例如“生产 D1 未就绪”或“写入解锁已过期”。
- 成功：显示检查时间、已解锁到期时间、写入审计号或回滚结果。

## 响应式与钉钉 WebView

- 笔记本宽度使用两列能力卡，低于 760px 改为单列。
- 长配置名允许换行，审计表在窄屏改为卡片，不产生页面横向滚动。
- 红色生产写入条固定在面板顶部而非浏览器视口，避免遮挡钉钉 WebView 导航。

## 交互文案

- 页面标题：`环境与生产数据`
- 正常：`生产环境配置完整`
- 阻断：`生产环境缺少必要配置，不能完成发布验收`
- 解锁按钮：`解锁 15 分钟生产写入`
- 确认短语：`修改线上真实数据`
- 解锁警示：`接下来的测试修改会直接写入线上数据库，并记录审计。`
- 冲突：`线上数据已被其他人更新，请刷新后重新操作。`
- 外部动作：`测试环境只允许修改数据库，不能执行真实钉钉操作。`

## 无障碍

- 状态图标均带文字；红、黄、绿满足 WCAG AA 对比。
- 解锁成功、过期和失败通过 `role=status/alert` 通知。
- 所有输入有可见标签；错误与输入通过 `aria-describedby` 关联。
- 不使用自动倒计时动画，只每分钟更新一次剩余时间文本。

## 视觉验收

- 1440×900：生产就绪、生产阻断、已解锁三种状态。
- 1024×768：说明书目录与两列能力卡。
- 390×844：单列卡片、解锁表单、审计卡片和错误提示。
- 钉钉 WebView：焦点、键盘、滚动和安全区。
