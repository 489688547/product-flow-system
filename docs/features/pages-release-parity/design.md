# Pages 发布一致性设计书

## 用户任务

员工打开生产域名时始终获得同一部署版本的 HTML、JS 和 CSS，不需要手动刷新或切换 URL。

## 信息层级

- 主信息：业务应用正常渲染。
- 辅助信息：稳定入口、无缓存响应和 hashed 资源状态。
- 低频信息：部署 ID、构建来源和恢复版本。

## 页面结构

不修改页面结构。发布层继续使用 `/cloudflare-entry` 作为兼容入口，根路径重写到同一 HTML。

## 交互流程

浏览器请求入口 → Cloudflare 返回当前部署的无缓存 HTML → HTML 请求同一部署的 hashed JS/CSS → React 启动并完成钉钉会话验证。

## 组件复用

复用现有 Vite 构建、Pages `_headers`、`_redirects` 和动态分包恢复模块。

## 新增组件

无 UI 组件。新增一个构建后处理脚本，职责仅是补齐 `dist` 发布契约文件。

## 页面状态

- 加载：维持现有钉钉账号验证页。
- 错误：缺失发布文件在构建阶段失败，不进入用户页面。
- 成功：入口、主包和业务 API 处于同一生产部署。

## 响应式与钉钉 WebView

不改变响应式和 WebView 行为；WebView 同样受益于入口不长缓存。

## 交互文案

无新增用户文案。

## 无障碍

不改变现有语义、焦点和键盘行为。

## 视觉验收

桌面 Chrome 和现有钉钉登录态至少各验证一次生产入口可渲染；控制台不得出现主 JS/CSS 404。
