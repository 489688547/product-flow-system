# 钉钉供应链文件夹数据导入 PRD

## 文档状态

- 状态：生产接入
- 负责人：供应链部
- 最近更新：2026-07-18

## 背景与问题

供应链文件夹已经沉淀供应商清单、成品盘点、月末库存、原辅料库存和异常库存数据。开发测试环境曾导入这些库存快照，但生产环境只同步了采购与付款审批，导致供应商和库存页面仍为空。快麦库存接口尚未验证可用，不能把钉钉文件快照标记成自动 ERP 同步。

## 目标

- 将供应商清单中的名称、类别和供货范围写入供应商主档。
- 将 2026 年 4 月库存盘点写入库存轨迹，保留 ERP 数量、实盘数量和差异。
- 将 2026 年 5 月兰山仓、山西仓成品库存及金额写入库存轨迹。
- 将提野星两仓原辅料数量、单价和金额写入独立明细，不与成品数量相加。
- 将异常库存的可售天数、预计到货和处理状态写入风险清单。
- 所有记录保留钉钉文档、工作表、源行和导入时间，重复导入可幂等更新。

## 非目标

- 本次不声称已经接通快麦实时库存接口。
- 本次不修改钉钉原文件。
- 本次不保存银行卡、手机号、身份证附件、收款资料或完整表格原文。
- 本次不自动把名称相近的产品强行关联到产品全周期档案。
- 异常库存和清仓备注不参与库存资金计算。

## 用户与权限

- 总经办、供应链、采购可查看并维护全部库存数据。
- 财务可查看库存金额，但不负责维护风险和盘点源数据。
- 产品、运营可查看库存与异常风险，不可修改。

## 当前流程

1. 供应链在钉钉文档维护盘点和库存表。
2. App 库存盘点页需要人工上传格式单一的文件。
3. 未进入产品全周期档案的 SKU 会被拒绝，历史库存无法完整导入。

## 目标流程

1. 导入工具只读钉钉供应链文件夹内登记的指定表格。
2. 按来源表结构规范化供应商、成品盘点、原辅料和风险记录。
3. 使用文档、工作表和源行生成稳定 ID，写入供应链状态。
4. App 分区展示成品轨迹、原辅料明细和异常风险。
5. 条形码能够匹配产品档案时关联 productId；不能匹配时保留名称和条形码，标记待关联。

## 业务规则

- 成品月末表的“期末库存数量”是 ERP 数量，“实际库存”是校准数量，“总金额”是实际库存金额。
- 4 月盘点表的“erp数量”和“盘点数量”分别保留，差值以盘点数量减 ERP 数量计算。
- 原辅料按产品、物料、仓库分别存储；不同计量单位的数量不汇总。
- 异常库存只用于提示补货和到仓风险，不生成库存调整。
- 同一来源再次导入时按稳定 ID 覆盖，不重复累加。
- 钉钉表中已被数值格式截断的条形码不得用于自动关联。

## 数据定义

- `suppliers`：供应商名称、类别、供货范围和来源证据；不含收款与个人敏感字段。
- `inventorySnapshots`：成品 SKU 库存与盘点轨迹。
- `materialInventorySnapshots`：产品下的原料、包材、辅料库存明细。
- `inventoryRisks`：低库存、预计到货和异常处理状态。
- `sourceDocument/sourceNodeId/sourceSheet/sourceRow`：证据链字段。
- `productId`：能稳定匹配时填写，否则为空。
- `productName/skuCode`：保留源表显示值，供后续人工关联。

## 异常与边界

- 空行、说明行、案例行和非数字数量行跳过并计入跳过数。
- Excel 日期序号转换为上海业务日期。
- 科学计数法或末位被补零的条形码标记为不可靠，不自动关联。
- 钉钉读取或本地 API 写入失败时不覆盖现有成功数据。

## 验收标准

- 重复运行导入工具，四类记录数量不翻倍。
- 供应商页展示文件夹中的有效供应商，且生产状态中不存在银行卡、手机号和身份证附件字段。
- 库存盘点页能看到成品库存、原辅料库存和异常库存三个数据区。
- 每条记录能看到产品/物料、仓库、数量或金额、来源日期。
- 异常中的记录优先展示，已解除记录仍保留历史。
- 数据源中心明确显示“钉钉文件快照”，不显示为快麦自动同步。

## 上线与回滚

- 本地继续通过命令行适配器读取钉钉；生产写入复用已登录且有权限的 `/api/supply-chain` 边界，不新增 D1 直写旁路。
- 生产写入前保留原状态快照；回滚时恢复快照，或删除 `dataSource=dingtalk-supply-folder` 的本次记录。
- 写入后分别核对生产供应商、盘点、成品库存、原辅料和异常库存数量。
