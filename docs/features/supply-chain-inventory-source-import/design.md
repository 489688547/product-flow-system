# 钉钉库存数据导入设计书

## 用户任务

供应链人员在库存盘点页直接判断各仓成品是否账实一致、原辅料占用多少金额、哪些产品存在断货和到货风险。

## 信息层级

- 主信息：成品 ERP 数量、实盘数量、差异、库存金额。
- 辅助信息：原辅料名称、数量、单价、金额和所属产品。
- 风险信息：可售天数、预计到仓日期/数量、异常状态和备注。
- 证据信息：来源文档、工作表、源行和导入时间。

## 页面结构

库存盘点页保持一个主页面，依次展示导入入口、成品库存轨迹、异常库存与到货风险、原辅料库存明细。不存在二级页面或额外返回按钮。

## 交互流程

文件导入仍先预览再确认。钉钉批量导入由开发/运维脚本执行，重复运行只更新同一源行。用户可在表格中直接阅读，不需要进入详情页。

## 组件复用

- `DataTable`：三类数据表统一横向滚动和空状态。
- `Button`、上传控件：继续用于人工文件导入。
- `SupplyChainProvider`：继续负责状态加载与保存。

## 新增组件

不新增通用组件。库存工作区内增加风险和原辅料列定义；导入解析留在纯函数和脚本边界。

## 页面状态

- 加载：沿用供应链应用加载状态。
- 空数据：分别说明可导入的对应来源。
- 错误：导入脚本中止且不 POST；人工上传显示行级错误。
- 无权限：沿用供应链权限。
- 禁用：只读用户不显示上传入口。
- 成功：表格直接显示写入结果，数据源中心显示数量与最近更新时间。

## 响应式与钉钉 WebView

窄屏保留横向表格滚动，产品名和物料名不竖排；来源说明放在次级文本中，不挤占主要列宽。

## 交互文案

- 成品区：`成品库存与盘点轨迹`
- 风险区：`异常库存与到货风险`
- 原辅料区：`原辅料库存明细`
- 未映射：`待关联产品档案`
- 数据源：`钉钉文件快照`

## 无障碍

表格保留语义表头；状态不能只靠颜色表达；长备注允许换行，产品和类别保持横排。

## 视觉验收

- 1440px：库存盘点页三个数据区、表头对齐、金额和数量可读。
- 900px：横向滚动正常，产品名不逐字换行。
- 空状态和已有数据状态各检查一次。
