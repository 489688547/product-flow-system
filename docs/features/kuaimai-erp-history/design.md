# 快麦 ERP 历史数据采集设计

## 用户任务

数据负责人使用已登录快麦页面导出一个明确时间范围的资源文件，并将其安全、可恢复地补入数据中心。

## 信息层级

本阶段没有新增页面。命令行首先显示资源类型、时间范围、文件哈希、有效行、异常行和是否允许写入；详细字段错误写入本地脱敏报告。

## 页面结构

沿用快麦原有导出页面和数据中心现有数据接入入口，不增加产品导航。

## 交互流程

选择资源与时间范围，下载文件，运行预检，确认范围完整后执行上传，最后读取服务器返回的批次、成功数和异常数。失败后使用同一文件重试，依赖哈希和来源键幂等恢复。

## 组件复用

- 复用 `src/domain/xlsxLite.js` 流式读取 XLSX/CSV。
- 复用现有商品目录、销售事实和货流模型作为标准投影。
- 复用真实登录会话、Pages Functions 和 `PRODUCT_FLOW_DB` 授权边界。

## 新增组件

- 本机解析器：识别快麦表头并生成标准批次。
- ERP ingest API：鉴权、校验、幂等存储和投影编排。
- 快麦采集 Skill：规定页面、时间口径、预检、导入和恢复步骤。

## 页面状态

- 加载：命令行显示当前文件和分块进度。
- 空数据：报告零有效行并禁止上传。
- 错误：输出稳定错误码、影响范围和重试方式。
- 无权限：不上传文件，提示使用真实数据管理员会话。
- 禁用：预检未通过时写入命令不可用。
- 成功：返回批次 ID、写入数、重复数和异常数。
