# 快麦 ERP 历史数据采集执行任务

## 执行规则

- 每项任务只交付一个可独立验证的结果。
- 先写失败测试并确认失败原因，再写实现。
- 完成后记录实际验证命令和结果。
- 每次提交只包含当前任务文件。
- 不使用子代理。

## 任务

- [x] 建立 ERP 原始批次和记录存储契约
  - 依赖：无。
  - 文件：迁移、环境清单、集成注册表、迁移测试。
  - 输出：新增表、容量和回滚契约。
- [x] 实现受控幂等 ingest API
  - 依赖：存储契约。
  - 文件：领域标准化、API 存储、授权、路由和测试。
  - 输出：500 条以内的幂等分块写入接口。
- [x] 实现快麦文件预检和上传 CLI
  - 依赖：ingest API。
  - 文件：采集器脚本、样例 fixture 和测试。
  - 输出：`--check` 与受控上传命令。
- [x] 创建快麦 ERP 数据采集 Skill
  - 依赖：CLI 契约。
  - 文件：Skill、参考表和契约测试。
  - 输出：可发现、可执行、可恢复的流程指南。
- [ ] 完成订单样例与全历史补数
  - 依赖：Skill 和 CLI。
  - 输入：三个月内订单和归档订单导出。
  - 输出：覆盖到最新 ERP 的批次与数据质量报告。
- [ ] 扩展其余 ERP 资源
  - 依赖：订单链路验证。
  - 输入：商品、库存、采购、售后和基础资料样例。
  - 输出：对应表头映射、投影和补数批次。
