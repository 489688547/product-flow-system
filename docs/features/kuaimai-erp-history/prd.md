# 快麦 ERP 历史数据采集范围契约

## 文档状态

- 状态：开发中
- 负责人：数据中心
- 最近更新：2026-07-22

## 背景与问题

快麦开放平台凭据没有形成可验证的完整数据通道，现有系统只保存销售日聚合和部分商品目录。公司已登录的快麦 ERP 页面可以导出订单、商品、库存、采购、售后及基础资料，但缺少统一的批次、幂等、质量和 D1 落库契约。

## 目标

- 使用已登录 Chrome 导出快麦 ERP 文件，不依赖未验证 API。
- 首次补齐全部历史订单和订单明细，并逐步补齐商品、库存、采购、售后和基础资料。
- 保存可查询的标准字段、原始行 JSON、批次范围、文件哈希和异常。
- 提供项目内 Skill，使后续补历史与每日增量按同一流程执行。

## 非目标

- 不向快麦写回数据。
- 不保存密码、Cookie、验证码或浏览器会话。
- 本阶段不新增数据中心页面。
- 不把 XLSX/CSV 二进制文件存入 D1。

## 用户与权限

只有数据中心、供应链、财务和总经办的非只读真实组织账号可以导入。普通员工通过已有业务 API 读取经过权限投影的数据，不直接读取原始 ERP 行。

## 当前流程

人工进入快麦页面，选择时间，导出文件，再由不同页面分别解析；缺少统一的历史游标和写入审计。

## 目标流程

Chrome 导出文件后，本机采集器识别资源类型和字段，计算哈希、检查范围、分块调用受控 API；API 幂等保存批次与原始记录，并投影到现有销售、商品目录和货流模型。

## 业务规则

- 订单统计日期固定使用订单创建时间，时区为 Asia/Shanghai。
- 首次订单补数同时覆盖三个月内订单与归档订单。
- 当前租户已验证的首条归档订单创建时间为 `2024-02-13 00:57:18`，首轮覆盖到 `2026-07-21 23:59:59`；`2020-01-01` 至 `2022-12-30` 无订单。
- 原始记录保留平台值；正常经营报表继续排除“其它/未知”。
- 同一文件哈希和资源类型重复导入不得重复生成事实。
- 商品和当前库存是快照；订单、售后、采购和库存出入库是历史事实。
- 快麦未返回或未导出的记录不能推断为删除。

## 数据定义

- 批次：平台、资源类型、文件名、哈希、时间范围、行数、状态、采集时间、导入时间和操作者。
- 原始记录：资源类型、稳定来源键、业务发生时间、修改时间、店铺、仓库、内容哈希和原始 JSON。
- 客户信息只保存快麦导出的脱敏值；不得主动恢复或拼接明文姓名、电话和地址。
- 真实 2026-07-01 至 2026-07-15 订单商品明细样本含 157,217 条有效行，序列化记录约 339.84 MiB；在原始明细全量超过 D1 容量前必须停止写入，并把原文件转存 NAS/R2、D1 仅保留查询事实和聚合。

## 异常与边界

登录失效、验证码、页面改版、导出超时、缺列、日期范围不完整和重复文件必须停止当前批次并保留可恢复游标。部分成功批次不得标记完成。

## 验收标准

- 订单样例文件可解析、校验并重复导入两次而只产生一份事实。
- 创建时间跨日边界按上海时区落到正确日期。
- 缺少系统订单号、创建时间或订单商品标识时产生质量异常而不是猜测。
- API 未登录、无权限、无 D1、无幂等键和超大批次均返回稳定错误。
- Skill 能指导完成样例导出、校验、导入、断点恢复和结果核对。

## 上线与回滚

迁移仅新增表和索引。回滚时停止采集器和路由，按批次删除新增 ERP 原始记录；现有销售、商品目录和货流表不做破坏性迁移。
