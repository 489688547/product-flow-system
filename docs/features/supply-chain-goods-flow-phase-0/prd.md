# 供应链管理货流升级 PRD

## 文档状态

- 状态：已评审
- 负责人：总经办、供应链部、财务部
- 最近更新：2026-07-20
- 版本：v1.0

## 背景与问题

现有供应链管理已经保存供应商、钉钉采购申请与付款申请、产品供应关系、库存盘点、原辅料库存和质量问题，但主要仍是分页面记录。库存来自 ERP 或文件快照，采购实付来自钉钉审批，销量与成本来自快麦或销售导入，各来源尚未形成可重算、可审计的统一货流事实。

管理层需要回答的不是单一“还有多少货”，而是供应响应与需求波动之间的时间差：钱压在哪些产品和供应商、哪些 SKU 会断货、账面库存与线下实存差多少、采购付款与销售消耗怎样共同影响现金循环。

货流升级不新增第二个业务 App。现有“供应链管理”保留为唯一业务入口，底层抽取独立货流数据平台，由供应链、产品全周期、数据中心、经营驾驶舱和公司 AI 读取同一套商品、库存与指标事实。

## 目标

### 总体目标

- 以现金循环周期 `CCC = 库存周转天数 + 应收天数 - 应付天数` 作为货流总闸门。
- 把需求计划、采购供应、生产在途、库存、履约、逆向质量和现金循环串成一套可追溯链路。
- 供应商交期方差、预测偏差进入安全库存和评分公式，不能只使用均值。
- 把销量异动到采购、补货或投放决策的时差从周压缩到天。
- 人只承担新品首单量、供应商关系、延误或清仓取舍等判断；确定性同步、计算、匹配和预警由系统完成。

### Phase 0 目标

- 形成 ERP 库存每日快照，并保留来源、时间、SKU、69 码、仓库、数量和成本。
- 建立月度线下盘点流程，区分 ERP 账面库存、盘点实存和校准库存。
- 生成可追溯的 CCC、库存周转天数、断货率、库存现金占用和库龄指标。
- 财务按平台维护固定应收账期，并按生效日期版本化。
- 所有指标同时展示来源新鲜度、覆盖率和可信等级。
- 同一个 SKU 在供应链、产品全周期和数据中心使用统一商品 ID、SKU ID、名称和 69 码。

## 非目标

Phase 0 不交付以下能力，但为后续保留稳定数据边界：

- 13 周 P10/P50/P90 需求预测和 MAPE 复核队列。
- 供应商评分、调价决策卡和 A 类 SKU 双供应商强制规则。
- PO 五节点自动跟催、供应商微信消息和投放节流自动执行。
- 快递账单自动对账、退货原因 AI 聚类及批次召回决策。
- 快麦库存接口的自动同步承诺。当前注册表只确认订单和销售能力；库存能力必须经接口、权限、字段和生产结果验证后才能替换文件通道。

## 用户与权限

| 角色 | 查看范围 | 编辑或确认责任 |
| --- | --- | --- |
| 总经办 | 全部货流指标、金额和异常 | 查看跨部门例外；不替代业务部门日常录入 |
| 财务部 | 平台账期、采购付款、库存资金、盘点损益、CCC | 维护平台账期；确认金额调整；冻结月度 CCC |
| 供应链/采购 | 供应商、采购、在途、库存、补货和映射异常 | 维护业务资料；确认账实差异和采购关系 |
| 仓库 | 所属仓库盘点任务、账面数和实存数 | 录入或导入实存数量；不能改采购金额或财务账期 |
| 质量管理部 | 产品、批次、供应商、退货和质量成本 | 维护问题、归因和处理状态 |
| 产品/运营 | 产品维度库存、可售天数、断货风险和节流建议 | 只读财务数据；按后续流程确认业务取舍 |
| 数据管理员 | 来源、同步、字段映射、SKU/69 码映射和数据质量 | 修复数据接入；不能代替业务部门确认事实 |

金额、成本和财务字段继续由服务端按角色裁剪，隐藏页面元素不构成权限边界。

## 当前流程

1. ERP 或供应链文件提供库存快照，快麦或销售文件提供销量与成本。
2. 仓库线下盘点后，把实存结果维护在钉钉文档或导入文件中。
3. 部门提交采购申请；财务使用与采购关联的付款申请支付。
4. 供应链页面分别展示采购付款、库存资金、产品供应商和质量问题。
5. 管理层需要人工拼接库存、付款、销量、账期和盘点差异，无法稳定复算历史 CCC。

## 目标流程

1. ERP 每天按 SKU×仓库生成账面库存快照；同步失败保留上次成功结果并标记过期。
2. 钉钉采购申请与关联付款申请进入货流事实层；无采购关联的报销、工资等付款不得进入。
3. 快麦订单/销售或文件导入形成销售数量和销售成本事实。
4. 财务维护各平台固定应收账期及生效日期。
5. 系统每天生成库存、可售天数、库龄和现金占用；每月生成 CCC 草稿。
6. 每月由仓库完成线下盘点并导入实存，供应链确认差异，涉及金额调整时由财务确认。
7. 盘点确认形成追加式盘盈或盘亏事件，不覆盖 ERP 原始快照。
8. 财务检查覆盖率与异常后冻结 CCC；补录数据只能生成新计算版本。
9. 驾驶舱只把需要人判断的断货、深库存、账实差异和付款未关联例外推到前面。

## 业务规则

### 总闸门与指标树

- 驾驶舱顶部只显示 CCC、断货率、库存周转天数三个数字。
- 展开区显示库存现金占用、滞销占比、OTD、交期方差、采购价格指数、MAPE、履约成本和质量成本等分量；Phase 0 未具备的数据明确显示未覆盖。
- 初始库存阈值为安全库存 14 天、断货红线 7 天、库龄红线 90 天，后续按品类版本化校准。

### 双库存账本

- `账面库存` 来自 ERP 日快照，永不被盘点覆盖。
- `盘点实存` 是盘点日的线下确认数量。
- `校准库存` 在首次盘点前等于账面库存并标记未盘点；盘点后按下式计算：

  `校准库存(t) = 最近盘点实存(T) + [ERP 账面库存(t) - ERP 账面库存(T)]`

- 新盘点确认后，以新盘点日作为下一段校准锚点。
- 计量单位不同的原辅料不得直接合计数量；金额必须使用已登记单位成本或已确认损益。

### 指标口径

- 库存现金占用：`已分摊采购实付 - 累计销售成本 ± 已确认盘点损益`。
- 库存周转天数：`月均校准库存成本 ÷ 当月销售成本 × 当月天数`。
- 应收天数：按平台净销售额，对财务维护的平台固定账期进行加权；账期按生效日取值，不回写历史月份。
- 应付天数：按采购金额加权计算验收入库日至实际付款日；没有入库日期时使用采购审批通过日并标记估算。月末仍未付款的采购按验收入库日至月末的已占用天数计入未付分量，并与已付款分量分别展示，避免只统计已付款采购而高估付款速度。
- CCC：`库存周转天数 + 应收天数 - 应付天数`。
- 断货率：`核心 SKU 断货天数 ÷ 核心 SKU 应售天数`。核心 SKU 的 A/B/C 分类在 Phase 1 校准前使用可配置清单。
- 月度结果保存计算版本、公式版本、来源时间、覆盖率和冻结状态。

### 月度盘点确认

- 仓库确认实存数量，供应链确认账实差异，涉及库存金额调整时财务再确认。
- 盘盈和盘亏分别保存原因、责任人、确认链和金额影响。
- 已确认记录不得物理删除；纠错通过反向调整或追加更正记录完成。

### 数据完整性

- 来源记录使用 `source + sourceReference + sourceVersion` 幂等写入。
- SKU、69 码、采购单、付款单或供应商无法确定性匹配时进入异常队列，不静默猜测。
- 缺数据不补零。指标返回覆盖率、缺口原因和可信等级。
- 月度冻结后补录数据生成新计算版本，旧版本继续可查。

## 数据定义

### `goods_flow_event`

保存采购申请、付款、入库、出库、销售消耗、退货、盘盈、盘亏和更正等不可变事实。核心字段包括事件 ID、类型、商品/SKU、69 码、仓库、供应商、采购单、数量变化、金额变化、发生时间、来源、来源引用、来源版本和创建人。

### `inventory_daily`

按日期、SKU 和仓库保存 ERP 账面数量与成本、校准数量与成本、可售数量、可售天数、库龄桶、现金占用、最近盘点日期、来源新鲜度和可信等级。

### `stocktake_session` 与 `stocktake_line`

保存月度盘点任务、仓库、盘点日期、状态、盘点人、确认人，以及每个 SKU 的 ERP 数量、实存数量、数量差异、单位成本、金额差异和原因。

### `platform_receivable_terms`

保存平台、固定账期天数、生效起止日期、维护人、版本和变更原因。同一平台任一日期最多命中一个有效版本。

### `ccc_monthly`

保存月份、库存周转天数、应收天数、应付天数、CCC、断货率、库存现金占用、各来源覆盖率、可信等级、计算版本、公式版本、冻结时间和冻结人。

### `metric_exception`

保存映射缺失、同步失败、快照过期、盘点差异、付款未关联、账期缺失和指标不可计算等异常；包含影响范围、责任部门、严重度、状态和来源证据。

## 异常与边界

- 同步失败：保留最近成功数据，显示更新时间、失败来源和重试入口，不清空页面。
- 重复导入：幂等覆盖同一来源版本或忽略重复事件，不重复增加库存和金额。
- 部分成功：成功记录提交，失败记录进入异常队列并保存批次结果。
- 未盘点：允许使用 ERP 账面库存，但页面和指标明确标记未盘点校准。
- 付款未关联：不计入 SKU 或供应商库存资金，直到确定性关联完成。
- 平台账期缺失：对应平台不进入应收天数分子，CCC 标记覆盖不足，不能显示为完整结果。
- 销售成本为零或缺失：库存周转天数不可计算，不使用零替代。
- 外部平台超时：服务端执行有限重试、记录请求 ID 和安全错误码，不把原始响应或凭据返回浏览器。

## 验收标准

- 给定连续 ERP 库存数据，系统能按日形成 SKU×仓库快照，并追溯来源与更新时间。
- 给定一次月度盘点，仓库、供应链和财务能按职责完成数量、差异及金额确认；ERP 原始快照保持不变。
- 给定盘点日后的 ERP 库存变化，校准库存按最近盘点锚点正确延伸。
- 给定钉钉采购及其关联付款，采购实付可分摊到已确认 SKU 和供应商；无关联付款不进入库存资金。
- 给定多平台销售与账期，系统按平台净销售额正确计算加权应收天数，并使用当期有效版本。
- 月度 CCC、库存周转、断货率和库存资金可逐层下钻到来源记录，并显示覆盖率和计算版本。
- 同一 SKU 在供应链、产品全周期和数据中心返回相同商品 ID、SKU ID、名称与 69 码。
- 权限不足、空数据、重复导入、部分失败、映射缺失、冻结后重算和外部超时均有可验证结果。

## 实施顺序

- Phase 0：库存日快照、月度盘点校准、库存资金和 CCC 月报。
- Phase 1：A 类 SKU 的 13 周需求预测、MAPE 复核、供应商评分和调价决策卡。
- Phase 2：PO 五节点跟踪、履约对账、退货质量聚类，以及库存预警到投放节流、补货和放量的闭环。

## 上线与回滚

- 新货流数据表和 `/api/platform/v1/goods-flow/*` 作为旁路能力上线，现有 `/api/supply-chain` 保持兼容。
- 首次回填先在 Preview 生成对账报告，不覆盖或删除现有供应链状态。
- 页面通过功能开关切换到新读模型；指标不一致、覆盖率异常或权限错误时关闭开关并恢复旧页面读取。
- 回滚保留新表和事件以便审计，不继续写入；不得通过删除历史盘点、付款或计算版本回滚。
- 快麦库存能力在独立验证通过前继续使用 ERP/文件快照通道，不能以产品文案暗示实时同步。
