# 供应链管理货流升级功能设计书

## 用户任务

用户从唯一的“供应链管理”App 进入，先判断 CCC、断货率和库存周转是否异常，再下钻到 SKU、仓库、供应商、采购付款和盘点来源。仓库每月完成线下盘点，供应链确认账实差异，财务确认金额影响并冻结当月 CCC。

## 能力边界

平台能力评审结论为 `抽取共享能力`：

- 业务入口仍是供应链管理，负责页面、工作流和人的判断。
- 独立货流数据平台负责跨来源事实、统一商品主键、计算规则、版本、权限、审计和稳定 API。
- 产品全周期、数据中心、经营驾驶舱和公司 AI 是货流平台的读取方，不复制可编辑库存或指标。
- 现有供应链状态在迁移期保留；新平台通过兼容投影读取已有供应商、审批、盘点和质量记录。

依赖方向保持 `features -> ui/domain/state`、`state -> API client`、`functions/api -> middleware/provider adapter`。React 页面不得直接调用钉钉、快麦、ERP 或 D1。

## 数据流

1. 钉钉适配器规范化采购申请和关联付款；ERP/文件适配器规范化库存快照与盘点；快麦适配器继续提供已验证的订单与销售聚合。
2. 服务端按来源引用和版本执行幂等校验，将有效数据写入不可变货流事件和来源批次。
3. 商品映射服务使用统一商品 ID、SKU ID 和 69 码；无法确定性匹配的记录进入异常队列。
4. 日计算器生成 `inventory_daily`；月计算器生成可重算的 `ccc_monthly` 草稿。
5. 盘点确认生成盘盈或盘亏事件，随后重算受影响日期；冻结 CCC 时保存公式和来源版本。
6. 供应链页面通过 `/api/platform/v1/goods-flow/*` 读取投影；其他 App 只读相同契约。

## 信息层级

### 主信息

- CCC、断货率、库存周转天数。
- 今日需要人判断的断货、深库存、盘点差异和付款未关联例外。
- SKU×仓库的账面库存、盘点实存、校准库存和可售天数。

### 辅助信息

- 库存现金占用、库龄、平台账期、应收与应付分量。
- 数据覆盖率、可信等级、最近成功同步和最近盘点日期。

### 低频信息

- 来源批次、计算版本、公式版本、请求 ID、同步日志和更正链。

## 页面结构

供应链管理不增加“返回业务 Apps”按钮，也不保留只有一个内容区的二级层级卡。页面沿用产品全周期的左侧 Tab 体系：

1. 货流驾驶舱。
2. 需求计划。
3. 采购与供应商。
4. 生产与在途。
5. 库存管理。
6. 履约物流。
7. 逆向与质量。
8. 现金循环。
9. 同步与覆盖。
10. 规则设置。

Phase 0 开放货流驾驶舱、采购与供应商的兼容入口、库存管理、现金循环、同步与覆盖、规则设置；未交付环节显示明确的阶段说明，不伪造业务数据。

### 货流驾驶舱

- 顶部只放 CCC、断货率、库存周转天数三个指标卡。
- 指标卡显示环比、目标、覆盖率和校准比例，不显示无关指标。
- “今日优先处理”使用紧凑例外条或表格，默认只呈现需要人判断的事项。
- 下方展示 CCC/库存资金趋势和库存例外表；详细分量折叠展开。

### 库存管理

- 主体直接显示 SKU×仓库表格，不增加“库存档案”包裹层。
- 列包含产品、SKU、69 码、仓库、ERP 账面数、最近盘点实存、校准库存、差异、可售天数、库龄、库存资金、数据状态和操作。
- 原料、成品使用横向标签；禁止因列宽造成中文逐字竖排。
- 默认筛选异常，支持产品、仓库、库存类别、盘点状态和新鲜度筛选。
- 表格横向滚动时固定产品列和操作列，金额与数量右对齐。

### 月度盘点

- 流程为生成任务、线下盘库、导入实存、供应链确认差异、财务确认金额。
- 导入前显示字段映射、有效行、失败行和差异预览。
- 确认动作明确说明将生成追加式调整事件，不会覆盖 ERP 快照。
- 已确认盘点只允许追加更正；更正展示前后值、原因和责任人。

### 现金循环

- 默认显示当前月及最近 12 个月趋势。
- 展开 CCC 的库存、应收、应付三个分量，并列出覆盖率和不可计算原因。
- 财务在同页维护平台账期版本；生效日期冲突时禁止保存。
- 冻结前展示异常清单，冻结后显示冻结人、时间和计算版本。

## 交互流程

### 每日同步

页面读取最近成功投影后异步读取同步状态。同步失败不清空数据；用户可以进入“同步与覆盖”查看失败来源、影响范围和重试状态。只有授权角色能触发真实同步，数据读取权限不自动授予外部平台动作权限。

### 月度盘点

仓库下载或打开盘点任务，线下盘库后导入实存。系统预览账实差异；仓库确认数量，供应链确认原因，财务仅在存在金额影响时确认。任一阶段退回都保留活动记录。

### CCC 冻结与重算

系统生成月度草稿并列出覆盖缺口。财务补齐平台账期和金额确认后冻结。当历史来源补录时，页面提示可生成新版本；旧版本保持只读，用户可比较差异后决定是否用新版本替代当前展示版本。

应付分量同时展示已付款采购的实际付款天数和月末未付款采购的已占用天数。缺少验收入库时间的记录使用采购审批通过日估算，并在下钻明细和覆盖率中单独标识。

## 组件复用

- 复用现有 `DataTable`、`Button`、`Modal`、页面标题、状态徽标和供应链 App 左侧导航样式。
- 复用现有公司会话、服务端部门权限、请求错误投影和数据中心来源元数据。
- 现有供应链状态 Provider 仅用于兼容期，不继续承载新的跨 App 货流事实。

## 新增组件

### `MetricHeadline`

业务中立的顶部指标组件，输入标签、值、单位、趋势、目标、覆盖率、可信等级和点击动作。覆盖加载、不可计算、过期、无权限和正常状态；键盘可聚焦并进入指标详情。达到两个真实消费者后进入 `src/ui`，否则先保留在供应链功能内。

### `DataFreshnessBadge`

展示来源最近成功时间、是否过期和覆盖率。颜色不是唯一信息，始终包含文本和可访问名称。

### `StocktakeDifferencePreview`

盘点导入专用组件，负责有效行、失败行、差异金额和确认影响，不负责解析外部文件或直接写数据库。

### `MetricVersionDrawer`

展示月度指标的公式版本、来源批次、冻结信息和历史版本；只读，不在抽屉内修改业务事实。

## 页面状态

- 加载：保留页面标题和筛选区域；指标使用固定高度骨架，表格显示行骨架。
- 空数据：按缺少库存、盘点、销售成本或账期分别给出下一步，不使用统一“暂无数据”。
- 错误：说明失败来源、影响范围、最近成功时间和是否可重试；保留可用旧数据。
- 无权限：说明当前角色可查看范围和负责部门，不展示被裁剪金额的占位数字。
- 禁用：冻结、确认或同步按钮同时显示禁用原因。
- 成功：导入、确认、冻结和重算后显示批次、影响记录数和后续状态。
- 过期：页面顶部及受影响指标同时标记，不能只在同步页提示。
- 部分成功：展示成功数、失败数和进入异常队列的记录数。

## 权限与不可逆动作

- 所有写权限在服务端校验，客户端角色只用于交互提示。
- 仓库只能维护所属仓库盘点实存；供应链确认数量差异；财务维护账期并确认金额。
- CCC 冻结、盘点金额确认和历史更正需要乐观版本；冲突返回最新版本供用户重新核对。
- 不提供物理删除盘点、事件或冻结指标的操作。

## 外部系统与环境边界

- Cloudflare Pages Functions 承载认证 API 和 provider adapter，D1 使用现有 `PRODUCT_FLOW_DB` 绑定。
- 新表在实现阶段加入环境能力清单、迁移和生产就绪检查；设计文档不把尚未创建的表标记为当前已就绪。
- 钉钉继续通过服务端审批适配器同步；浏览器不保存 process payload、凭据或原始敏感字段。
- 快麦订单/销售能力可作为 Phase 0 成本来源；库存接口只有在独立验证后才能登记为已连接能力。
- ERP 文件导入当前状态为集成中，必须保留批次、来源和人工校准标识。
- 本地读取生产数据必须复用受控生产数据网关；生产数据库写入权不包含真实钉钉或快麦动作权。

## 响应式与钉钉 WebView

- 1440px 及常见笔记本宽度保持左侧 Tab、三个顶部指标和主表格可读。
- 窄于 1180px 时指标卡可以换行，表格进入水平滚动，不能压缩成竖排中文。
- 移动或窄 WebView 中左侧导航收起为抽屉，保持当前页面和筛选条件。
- 使用动态视口高度和安全区，弹窗不超出 WebView；底部操作可在键盘弹出时滚动到达。

## 交互文案

- 过期提示：`ERP 库存同步失败，当前展示上次成功数据（{time}）。`
- 未盘点：`该库存尚未经过线下盘点校准。`
- 部分成功：`已写入 {success} 条，{failed} 条进入数据异常队列。`
- 盘点确认：`确认后将新增盘盈/盘亏事件，不会覆盖 ERP 原始快照。`
- CCC 覆盖不足：`当前 CCC 覆盖 {coverage}%，缺少 {sources}。`
- 冻结确认：`冻结后本版本保持只读；后续补录将生成新版本。`

## 无障碍

- 焦点顺序按页面标题、日期范围、顶部指标、例外、图表摘要、表格和操作执行。
- 左侧 Tab 使用导航语义和当前页标记；抽屉打开后约束焦点，关闭后返回触发按钮。
- 表格表头、排序、筛选、错误摘要和行操作提供可访问名称。
- 状态不只依赖红黄绿；文字和图标共同表达。
- 文本和背景满足 WCAG AA；遵循减少动效设置。

## 视觉验收

- 1440px：供应链管理左侧 Tab、顶部三个指标、例外区、趋势和库存表。
- 1180px：指标换行、固定产品列、金额与数量可读、原料和成品横向显示。
- 768px 钉钉 WebView：导航抽屉、筛选、盘点差异预览和确认按钮。
- 状态截图：正常、加载、无库存、未盘点、同步过期、部分成功、无权限、版本冲突、冻结完成。
