# 商品主数据销量视图 PRD

## 文档状态

- 状态：验证完成，待合并
- 负责人：数据中心 / 运营部
- 最近更新：2026-07-20

## 背景与问题

现有商品主数据已经汇总快麦商品、SKU、主/规格商家编码和 69 码，但页面仍以 ERP 状态和档案质量为中心。老板、运营和产品在查看商品时，不能直接回答“这个商品在选定平台和日期卖了多少”，需要切换到销售页面后再按条码人工对应。

快麦订单已经按 `69 码 × 日期 × 平台` 进入 D1 的 `product_sales_daily`，商品档案进入 `product_catalog_items` 与 `product_catalog_skus`。本功能扩展现有共享商品目录，不复制销售事实，也不建立第二套商品或销量表。

## 目标

1. 商品主数据表去掉“ERP 状态”，直接显示所选范围内的销量。
2. 支持按平台、预设日期和自定义日期筛选，销量与筛选范围同步变化。
3. 用 SKU 条码将快麦销售事实聚合到商品，展示商品销量、销售额和平台构成。
4. 快麦自动订单同步使用订单创建时间，与公司销售时间口径一致。
5. 商品档案和销售事实均保存于 D1，前端只读取受控聚合结果。

## 非目标

- 不把商品主数据改造成完整销售分析驾驶舱。
- 不把快麦原始响应或订单整行保存到商品目录 payload。
- 不从本系统反向修改快麦商品、订单或平台配置。
- 不删除商品启停信息；状态仍作为数据库字段和数据质量信息保留，只从主表移除。
- 不在本期接入快麦库存接口。

## 用户与权限

- 商品目录现有已登录可读权限保持不变。
- 销量聚合只返回非敏感经营字段：销量、销售额、平台和日期覆盖；采购成本仍按既有部门权限裁剪。
- 总经办和运营部继续拥有商品同步与文件导入权限；其他用户只读。
- 所有权限在服务端执行，前端隐藏操作不作为授权边界。

## 核心业务规则

1. 销售事实通过 `product_catalog_skus.barcode` 或规格商家编码与 `product_sales_daily.code` 确定性关联；不能匹配的事实不猜测归属。
2. 一个商品多个 SKU 的销售行按已落库销售编码聚合；目录中重复条码只建立一次归属，避免重复计算。
3. 默认日期为最近 30 个自然日，包含今天；支持最近 7 天、最近 30 天、本月、上月和自定义范围。
4. 自定义范围包含起止日期，最长 370 天；非法范围返回 400，不执行无界查询。
5. 平台选项来自筛选日期内已落库的实际销售事实；默认“全部平台”排除 `其它`、`其他`、`未知`、`未知平台`。
6. 销量为 `SUM(qty)`，销售额为 `SUM(net_sales)`；页面主列突出销量，销售额为辅助信息。
7. 快麦自动拉单以订单 `created` 和子订单 `created` 归属日期；子订单缺失时可回退订单创建时间，不使用支付时间替代统计日。
8. 商品与销量同步时间分别展示，并返回筛选范围内的 `latestDataDate`；查询结束日晚于实际数据日时必须标记覆盖滞后，不能写成“截止昨天”。
9. D1 中已有 `product_sales_daily` 继续作为唯一销售事实表，商品目录 API 只做按条件聚合读取，GET 不执行建表或改表。
10. 商品表默认按销量降序、净销售额降序、商品名排序，让有经营贡献的商品优先进入首屏。

## API 契约

`GET /api/platform/v1/product-catalog?from=YYYY-MM-DD&to=YYYY-MM-DD&platform=<平台>`

- `from`、`to` 必填且最多 370 天；前端始终提交默认范围。
- `platform` 可省略；省略时聚合所有正常平台。
- 每个商品增加 `sales`：`quantity`、`netSales`、`platforms`、`matchedCodeCount`。
- `meta.sales` 返回查询范围、可用平台、总销量、总销售额、覆盖商品数、未匹配销售行数、`latestDataDate`、时间口径和最近销售入库时间。
- 不带日期参数的旧客户端继续获得原商品目录，不增加 `sales` 且不触发销售表扫描。

## 验收标准

1. 主表不出现“ERP 状态”，出现“销量”列；数量右对齐并使用稳定数字宽度。
2. 用户可按平台与日期预设筛选，并可输入自定义起止日期。
3. 筛选变化触发服务端查询；加载期间保留上次成功表格并标记销量更新中。
4. 同一商品多个 SKU 的销售被正确合计，平台过滤与日期边界正确。
5. 无销量商品显示 `0` 和“该范围暂无销售”，不伪装成数据缺失。
6. 匹配覆盖不足时显示未匹配提示；“其它”不混入默认经营统计。
7. 加载、空目录、无结果、错误、只读、同步中和无销售事实均有明确状态。
8. 1440、1180、900、640 和 390px 下表头不逐字换行，页面无横向溢出，表格容器可横向滚动。
9. 首屏优先展示高销量商品；销售事实晚于查询结束日时显示实际更新日期。

## 上线与回滚

- 不新增环境变量、绑定或 D1 表；复用 `PRODUCT_FLOW_DB`、`product_catalog_*` 和 `product_sales_daily`。
- 回滚前端/API 聚合后，现有目录和销售数据不受影响。
- 快麦创建时间口径变更只影响后续自动同步；历史月度回填仍可通过既有导入/回填链路校准。
- 未经单独授权不执行生产部署或真实快麦同步。
