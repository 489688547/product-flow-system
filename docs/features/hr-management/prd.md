# 人事管理 App PRD

## 文档状态

- 状态：待书面复核
- 负责人：行政人事部 / 财务部
- 最近更新：2026-07-18

## 背景与问题

公司的人事信息、入转调离、绩效、考勤假期和工资核算目前分散在钉钉、表格、聊天和各业务 App 中。主管能够分配任务，却缺少一条从正式绩效目标、员工自评、业务证据到主管评价的可追溯链路；行政人事和财务还需要重复整理考勤、假期、绩效结果与工资调整，容易出现版本不一致、漏项和越权查看。

人事管理 App 应成为独立业务 App，负责员工关系、绩效周期、考勤假期和工资核算辅助。产品全周期、供应链管理、电商店铺运营、品牌内容协同和公司经营继续维护各自任务与结果，人事 App 只读取授权证据并形成冻结的人事记录，不复制或回写业务执行数据。

## 目标

1. 新增与现有业务 App 同级的“人事管理”App。
2. 建立以钉钉组织身份为基准的员工、部门、岗位和任职关系主档。
3. 管理入职、转岗、调动、离职的任务、材料、权限交接和生效记录。
4. 完成“模板与周期 → 主管定项 → 员工执行和自评 → 系统建议分 → 主管终评 → 员工确认/复核 → 人事冻结”的绩效闭环。
5. 从其他业务 App 引用任务、结果和复盘证据，避免重复填报，不以单一 GMV、ROI 或任务数量自动评价员工。
6. 管理工作日历、考勤异常、请假、加班和假期余额，并把已确认影响转换为可解释的工资调整项。
7. 根据有效期规则、考勤假期、冻结绩效和人工调整生成工资草稿，由行政人事复核、财务确认后发布工资条。
8. 为员工提供自己的档案、绩效、考勤假期和工资条；为主管、行政人事、财务提供最小必要范围。

## 分期范围

### 一期：人事运营底座

- 人事总览。
- 员工与岗位。
- 入转调离。
- 绩效管理。
- 考勤与假期。
- 工资核算辅助。
- 人事任务。
- 人事设置与规则版本。

### 二期：人才获取与发展

- 招聘需求、候选人流程、面试协同和入职衔接。
- 培训计划、课程、学习记录和岗位能力提升。

### 三期：组织与人才治理

- 人才盘点、继任计划、合同到期、员工关系和组织分析。

## 非目标

- 一期不做招聘、培训、人才盘点、继任计划或完整劳动合同档案。
- 不保存身份证号、银行卡号、个人手机号、医疗材料原件等高敏感原始资料。
- 不自动申报或缴纳个税、社保、公积金，不自动发放工资。
- 不内置不可修改的劳动法规或假期扣薪公式；公司规则必须有生效日期、版本、审批人与依据说明。
- 不把生成式 AI 的主观判断作为员工分数；AI 最多辅助归纳证据和提示缺口。
- 不允许系统建议分覆盖主管终评分，也不允许主管终评分自动回写业务 App。
- 不在本 App 直接抓取电商、广告、ERP 或浏览器后台数据。
- 不把员工考勤、请假、工资或个人绩效明细写入数据中心经营事实。
- 未经单独确认，不部署生产、不发送真实钉钉待办或审批、不导入真实工资数据。

## 用户与权限

### 员工

查看自己的员工信息、绩效项目、证据、自评、考勤异常、假期余额和已发布工资条；提交自评、异常说明和一次绩效复核申请。不能查看其他员工、工资规则或未发布工资批次。

### 直属主管

查看当前管理范围内员工的任职摘要、绩效项目、执行证据、自评和团队进度；制定绩效项目、目标、权重和评分标准，完成终评和反馈。不能查看员工完整工资、他人薪酬规则或人事隐私材料。

### 行政人事专员

维护被授权员工主档、异动流程、绩效周期、考勤假期、工资规则和工资草稿；处理复核和异常。工资写入权限与普通人事资料权限分开授权。

### 行政人事负责人

维护人事角色、绩效模板、规则版本和冻结动作；查看公司范围人事运营和敏感审计；不能绕过财务确认直接发布工资条。

### 财务

查看工资批次、员工级工资计算明细、调整来源和复核状态；确认或退回工资批次。默认只读员工非财务人事资料，不能修改绩效终评。

### 总经办

查看组织、人员、绩效和人力成本汇总；个人工资明细默认不可见，只有被显式授予工资敏感权限后才能查看。可处理最高级别的绩效复核和规则审批。

### 业务 App 负责人

维护本 App 中的任务、结果和复盘，只提供经过授权的证据引用；不能读取人事评分、工资或考勤明细。

## 当前流程

1. 钉钉保存组织和人员，其他任职信息分散在表格。
2. 主管通过聊天或表格制定绩效，员工月底补充说明，业务任务需要重复抄写。
3. 考勤、请假和加班分别查询，异常通过聊天确认。
4. 行政人事把假期、考勤和绩效结果手工换算成工资调整。
5. 财务通过表格复核，历史版本、退回原因和员工工资条难以统一追溯。

## 目标流程

### 员工与异动

1. 行政人事从钉钉组织读取稳定人员身份，补充员工编号、岗位、直属主管、用工状态和生效日期。
2. 入职、转岗、调动或离职创建人事事件和跨部门任务清单。
3. 目标部门、IT、财务、行政和业务负责人完成各自交付；人事事件只在必需任务完成并经有权限人员确认后生效。
4. 历史任职记录保留，不覆盖旧岗位和旧主管关系。

### 绩效

1. 行政人事创建绩效周期并选择已批准模板。
2. 直属主管为员工制定项目、目标、权重、评分标准和业务任务引用；员工确认已知悉。
3. 周期内员工推进业务任务，系统定期读取可访问的任务状态、结果和复盘摘要。
4. 周期结束后员工提交自评、自评分和证据说明。
5. 系统根据已配置的确定性规则生成建议分，并解释每个项目的计算和缺失证据；AI 可归纳事实或提示矛盾，但不直接决定分数。
6. 主管提交终评分、评价和改进要求；员工确认或发起一次复核。
7. 复核完成后行政人事冻结绩效快照；后续更正只能形成新版本和审计记录。

### 考勤假期与工资

1. 系统按工资周期导入或同步排班、出勤、请假、加班和假期余额。
2. 员工处理异常，主管或行政人事按既有制度确认；未确认异常不能静默进入扣款。
3. 行政人事冻结考勤期，系统依据当期有效规则生成工资调整项。
4. 工资引擎读取员工有效薪资项、考勤调整、冻结绩效结果和人工调整，生成逐项可解释的工资草稿。
5. 行政人事复核后提交财务；财务确认或带原因退回。
6. 确认批次发布员工工资条；后续纠错使用补差批次，不覆盖已发布记录。

## 业务规则

1. App ID 为 `hr-management`，左侧分组名称为“人事管理”，位于各业务协同 App 之后、平台之前。
2. 一期页面为人事总览、员工与岗位、入转调离、绩效管理、考勤与假期、工资核算、人事任务和设置。
3. 新员工必须绑定钉钉 `userId` 或 `unionId`；姓名仅用于展示和历史兼容，不作为唯一身份。
4. 员工、岗位、直属主管、部门和薪资项均使用生效区间；历史记录不被新值覆盖。
5. 业务 App 是任务和结果事实源；人事 App 保存稳定引用与冻结快照，不修改源任务。
6. 绩效项目权重合计必须为 100%；每项包含目标、验收标准、评分规则和至少一个证据来源或人工说明要求。
7. 员工自评分、系统建议分和主管终评分分开保存并同时展示，不允许互相覆盖。
8. 系统建议分只能来自已版本化的确定性规则；缺失证据时返回“无法计算”或待补充，不以 0 替代。
9. AI 只做证据摘要、矛盾检查和改进提示；AI 输出记录输入版本、限制和生成时间，不写入终评分字段。
10. 员工最多对一个终评版本发起一次复核；复核必须填写理由，可补充证据但不能修改源业务事实。
11. 绩效冻结后，工资批次和公司激励只能引用冻结版本；不因源任务后续变化自动改写历史。
12. 公司激励可引用冻结绩效作为一个依据，但不自动计算或发放奖金。
13. 考勤、请假和加班原始记录与确认结果分开；工资只读取已确认且属于当期的调整项。
14. 假期是否带薪、每日/每小时折算、病事假扣减、加班换休或计薪均由有效期规则决定，不硬编码单一公司或法规公式。
15. 工资草稿必须逐项展示基础薪资、固定项、绩效浮动、考勤假期调整、人工调整、税社保手工项和应发/实发估算。
16. 人工工资调整必须填写类型、金额、原因、依据、操作者和审批记录；敏感变更写入审计。
17. 财务确认前员工不可见工资草稿；发布后员工只查看自己的工资条。
18. 已确认工资批次不可直接编辑；更正使用补差批次并关联原批次。
19. 钉钉现有能力只视为身份、组织通讯录、待办、日历、文档和内嵌工作台已连接；考勤、请假、加班或审批同步需独立验证后才能标记为自动接入。
20. 自动同步不可用时，允许使用受控 Excel/CSV 导入；导入记录来源、期间、文件哈希、行数、操作者和校验结果。
21. 所有查看和写入均由服务端按人员范围、角色和数据敏感级别校验；隐藏导航不是权限边界。

## 数据定义

- `Employee`：员工编号、钉钉身份引用、姓名、在职状态、入职/离职日期和资料完整度；不含一期排除的高敏感字段。
- `EmploymentAssignment`：部门、岗位、直属主管、工作地点、用工类型、生效起止时间和变更来源。
- `HrRoleAssignment`：人事、主管、财务、工资敏感等系统角色及适用组织范围和生效区间。
- `LifecycleEvent`：入职、转岗、调动、离职事件，包含申请、原因、目标任职、生效日、状态和任务清单。
- `PerformanceTemplate`：适用岗位/部门、项目结构、默认权重、评分区间、规则版本和审批状态。
- `PerformanceCycle`：周期、适用人员、阶段、截止时间、冻结版本和负责人。
- `PerformanceItem`：员工、目标、权重、评分标准、任务证据引用、自评、建议分、终评分和反馈。
- `WorkEvidenceRef`：来源 App、实体类型、实体 ID、版本、负责人、验收状态、摘要、数据截止时间和取得时间。
- `PerformanceSnapshot`：冻结时的绩效项目、证据、三类分数、复核和审批快照。
- `WorkCalendar`：工作日、休息日、法定/公司假期和适用范围。
- `AttendanceRecord`：日期、班次、出勤、迟到早退、缺卡、来源和确认状态。
- `LeaveRecord`：假期类型、起止时间、时长、余额影响、审批状态和来源引用。
- `OvertimeRecord`：加班时长、补偿方式、确认状态和来源引用。
- `LeavePayRule`：假期类型、带薪比例、折算单位、适用范围、生效区间、版本和批准人。
- `CompensationProfile`：员工薪资项和生效区间；按敏感权限单独读取。
- `PayrollAdjustment`：来源、类型、金额、正负方向、原因、期间、确认状态和审计。
- `PayrollRun`：工资期间、规则版本、考勤冻结版本、绩效冻结版本、状态、汇总和财务确认。
- `Payslip`：员工级工资明细、应发/扣减/实发估算、发布状态和原批次引用。
- `ImportBatch`：来源、期间、文件哈希、字段映射、行数、错误数和操作者；不保存原始敏感文件内容。
- `HrAuditLog`：对象、动作、变更字段名、操作者、时间和原因；不复制敏感字段值。

## 跨 App 协同

- **钉钉**：复用登录和组织通讯录；待办仅在另行授权后同步。考勤、请假、加班和审批属于待验证的新能力，未验证前使用导入通道。
- **电商店铺运营**：提供重点产品方案、执行任务、数据结果和复盘证据引用；人事 App 读取冻结快照，不回写方案或团队管理数据。
- **产品全周期、供应链管理、品牌内容协同和公司经营**：按同一证据引用模型提供任务、里程碑、交付和复盘摘要；各 App 保留事实所有权。
- **数据中心**：提供经营指标、指标版本、截止时间和质量状态，帮助解释业务结果；不保存工资、个人考勤或绩效明细，也不直接决定个人分数。
- **财务**：确认工资批次、退回原因和调整项；已确认结果可向公司经营提供脱敏人力成本汇总，不暴露个人工资。
- **公司激励**：只引用冻结绩效版本和汇总结果，不自动产生奖金。

## 异常与边界

- 钉钉组织未同步：保留最近组织缓存并标记过期；不能新建无法绑定稳定身份的正式员工。
- 考勤或请假自动接入不可用：允许导入，页面显示来源和截止时间，不伪造“已同步”。
- 导入重复：按文件哈希、期间和外部记录 ID 检测；重复批次需人工确认，不静默覆盖。
- 数据冲突：员工主档、绩效、规则和工资批次使用版本号；版本落后返回 409 并保留用户草稿。
- 业务证据不可访问：保留已取得快照和源链接，标记权限或新鲜度限制；不得把缺失证据计为失败。
- AI 不可用：绩效、自评、主管评分和工资流程继续运行，只隐藏证据摘要建议。
- 考勤异常未确认：工资草稿显示待处理，不自动扣款；提交财务前必须解决或由授权人员带原因覆盖。
- 绩效未冻结：可生成工资预览，但不能提交包含绩效浮动项的正式工资批次。
- 规则跨期变化：按记录发生日和工资期间匹配有效版本，不追溯改写已发布工资。
- 员工离职：历史绩效、工资条和审计保留；账号失效后只能由授权人事和财务查看。
- 权限变化：立即按新组织和角色范围计算访问，已下载文件无法远程撤回，因此导出需要单独权限和水印/审计。
- D1 或 API 不可用：只读展示上次安全缓存；工资、绩效冻结和批量导入等高影响写入不可离线确认。

## 验收标准

1. 人事管理作为独立左侧 App 分组出现，包含八个入口，并位于业务协同 App 之后、平台之前。
2. 行政人事可从钉钉组织选择员工并创建有效期任职记录；姓名重复时仍以稳定身份区分。
3. 入职、转岗、调动和离职事件保留目标任职、生效日、任务、完成状态和历史版本。
4. 主管可创建权重合计 100% 的绩效项目并关联至少一个业务任务或人工证据要求；员工只能编辑自己的自评。
5. 页面同时展示员工自评分、可解释的系统建议分和主管终评分；AI 不写终评分。
6. 绩效终评后员工能确认或发起一次复核；冻结后修改会形成新版本而不是覆盖。
7. 业务 App 证据以来源、实体、版本、验收状态和截止时间引用；人事 App 不修改源任务。
8. 考勤、请假、加班和假期余额可通过已验证同步或导入进入；来源和同步状态清楚可见。
9. 未确认考勤异常不会自动产生扣款；确认后按当期有效规则生成可解释调整项。
10. 工资草稿逐项列出计算来源；行政人事复核后由财务确认，确认前员工不可见。
11. 已发布工资条仅员工本人和具有工资敏感权限者可见；直属主管不能通过前端或 API 读取完整工资。
12. 已确认工资更正通过补差批次完成，原工资条和审计保留。
13. 数据中心只提供经营证据和脱敏汇总，不接收个人考勤、工资或绩效明细。
14. 钉钉考勤未验证时页面明确显示“导入数据”，不得显示自动同步成功。
15. 权限、并发、导入重复、数据过期、空数据、AI 失败、D1 缺失和外部接入失败均有可恢复状态。
16. 键盘、焦点、空/错/禁用/只读/过期状态、390–1440px 和钉钉 WebView 均通过验收。

## 上线与回滚

- App 注册、导航和 API 使用独立功能开关，默认只在本地和预览环境启用。
- 一期新增独立人事表和审计表，不把工资与绩效放进现有共享大状态。
- 初始员工主档只从钉钉组织建立引用，不导入真实工资数据；工资功能先用脱敏测试人员和样例规则验收。
- 钉钉考勤/审批接入必须先更新集成注册表、环境能力清单、适配器和契约测试，再从“导入”切换为“自动同步”。
- 监控 API 错误、权限拒绝、版本冲突、导入失败、规则缺失、工资批次状态和敏感导出。
- 回滚时关闭人事 App 功能开关并停止写入；保留独立表和审计供恢复，不删除员工历史或已发布工资记录。
- 未经单独确认不部署、不配置生产凭证、不发送钉钉动作、不导入或导出真实工资数据。
