# 销售数据服务查询设计

## 页面结构

数据服务页依次展示 AI 服务配置、销售数据服务、应用订阅。销售数据服务包含覆盖状态、月份快捷选择、共用日期范围、查询按钮和汇总结果。

## 交互流程

页面加载时读取可用范围。系统用最新可用月份填充待查询条件，但不自动执行汇总查询。用户可选择月份或修改起止日期；只有点击“查询数据”才发出汇总请求。

## 组件复用

将数据总览现有日期范围控件提取为业务中立的 `DateRangeControls`，由数据总览和数据服务共同使用。月份选择和查询状态留在数据服务特性内。

## 页面状态

- 加载：仅覆盖范围区域显示加载状态。
- 待查询：显示已选范围和查询按钮，不展示虚构结果。
- 查询中：按钮禁用并显示查询中。
- 空数据：说明该范围没有销售事实。
- 错误：显示服务返回的稳定错误信息并保留待查询条件。
- 成功：显示记录数、净销量、净销售额、平台数及实际覆盖日期。

## 无障碍与响应式

所有输入有可见标签，错误使用 `role=alert`，状态使用 `role=status`；键盘可完成月份、日期和查询操作。窄屏下控件纵向排列，结果卡片自动换行。
