# 销售数据服务查询

## 文档状态

- 状态：开发中
- 负责人：数据中心
- 最近更新：2026-07-22

## 背景与问题

销售历史已写入线上 D1，但数据服务页面只展示应用订阅，无法确认可用月份或按日期读取数据。日期输入若直接复用数据中心全局范围，还会在只选完开始日期时立即请求数据库。

## 目标

- 在数据服务页面展示销售数据的最早日期、最新日期和可用月份。
- 支持月份快捷选择和任意起止日期，只有点击“查询数据”才读取 D1。
- 返回面向经营判断的汇总，不罗列全部销售明细。
- 形成版本化平台接口，供其他业务 App 读取同一数据库口径。

## 非目标

- 不新增 D1 表、绑定或销售采集逻辑。
- 不开放浏览器直连 D1。
- 不替代现有销售明细内部接口。

## 业务规则

- 日期口径固定为订单创建时间，时区为 `Asia/Shanghai`。
- 日常经营数据排除空平台、“其它”“其他”“未知”“未知平台”。
- 月份和日期控件只修改待查询条件；查询按钮是唯一触发读取的动作。
- 查询结果包含记录数、净销量、净销售额、平台数和实际覆盖日期。

## 验收标准

- 页面打开后仅加载轻量可用范围，月份选项来自 D1 实际数据。
- 选择月份、开始日期或截止日期均不触发汇总查询。
- 点击查询后按完整日期范围返回汇总，跨月范围可用。
- 无权限、无 D1、无数据、非法日期和服务错误均有明确状态。

## 上线与回滚

本次无迁移。回滚页面和新路由即可，现有 D1 数据及 `/api/data-center/sales` 不受影响。
