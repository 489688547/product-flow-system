# 前端运行时防白屏设计书

## 用户任务

用户正常打开并使用经营执行平台；即使浏览器缓存不可用或某个 React 子树异常，也能看到当前页面或明确的恢复入口，而不是空白屏幕。

## 信息层级

- 主信息：页面无法继续显示及“重新加载”主操作。
- 辅助信息：线上数据未被删除，清理本机缓存可能丢失未同步草稿。
- 低频信息：“清理本机缓存后重试”次操作。

## 页面结构

正常状态不新增页面或常驻提示。故障状态覆盖应用内容，使用居中的窄内容区，依次显示平台标识、标题、影响说明、主次操作和数据安全说明。

## 交互流程

1. 普通缓存失败被安全存储边界吸收，用户继续使用页面，不弹阻断提示。
2. React 故障边界捕获渲染或生命周期错误后显示故障页。
3. 点击“重新加载”直接刷新当前正式地址。
4. 点击“清理本机缓存后重试”先显示浏览器确认框，说明可能丢失本机未同步草稿。
5. 用户确认后只删除白名单业务缓存键，再刷新；取消则留在故障页。

## 组件复用

- 复用现有全局 CSS 变量、按钮视觉和焦点规范。
- 现有 `deploymentRecovery` 继续专门处理分包加载失败，不承担 React 运行时故障。
- 现有 `resilientLocalStorage` 扩展为所有 Provider 共用的纯函数边界。

## 新增组件

### `ApplicationErrorBoundary`

- 职责：捕获其子树的 React 渲染和生命周期错误，切换到故障页。
- 属性：`children`；测试时允许注入 `windowRef` 或恢复函数只通过独立纯函数测试，不暴露业务配置。
- 状态：`hasError`。
- 键盘行为：两个原生按钮进入正常 Tab 顺序，焦点样式清晰。
- 复用边界：只在根入口挂载一次，不放进各业务 App。

### 应用缓存恢复函数

- 职责：按白名单尽力删除本应用业务缓存。
- 输入：可注入的 `localStorage`、`sessionStorage`。
- 输出：每类缓存是否全部成功；失败不抛出。
- 不负责：清理 Cookie、IndexedDB、认证或服务端数据。

## 页面状态

- 加载：沿用现有启动壳和各 Provider 加载状态。
- 空数据：不受影响。
- 错误：故障页展示“页面遇到问题”，不展示原始堆栈。
- 无权限：沿用认证与权限页面，不进入故障页。
- 禁用：清理按钮在执行期间无需异步等待；确认取消保持可操作。
- 成功：重载后恢复正常应用，不额外显示成功提示。

## 响应式与钉钉 WebView

故障页使用 `min-height: 100dvh`、安全区内边距和最大宽度；窄屏按钮纵向排列，桌面横向排列。无需依赖固定视口、弹窗门户或浏览器扩展。

## 交互文案

- 标题：`页面遇到问题`
- 说明：`应用没有删除线上数据。你可以先重新加载；如果问题持续，再清理本机缓存后重试。`
- 主按钮：`重新加载`
- 次按钮：`清理本机缓存后重试`
- 确认：`这只会清理当前浏览器中的本机业务缓存，可能丢失尚未同步的本机草稿；不会删除线上数据库。确认继续吗？`

## 无障碍

- 故障容器使用 `role="alert"` 和可聚焦标题。
- 标题在故障状态挂载后获得焦点，读屏器立即获得上下文。
- 原生按钮保留可见焦点、禁用和键盘语义。
- 文案与按钮满足 WCAG AA 对比度，不用颜色单独表达严重程度。
- 不使用自动动画。

## 视觉验收

- 1440×900 正常应用状态。
- 1440×900 故障页与焦点状态。
- 390×844 窄屏故障页。
- 钉钉 WebView 动态视口与安全区。
