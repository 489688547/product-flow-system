# 前端运行时防白屏 PRD

## 文档状态

- 状态：已评审
- 负责人：平台研发
- 最近更新：2026-07-23

## 背景与问题

2026-07-23 生产环境中，产品全周期共享状态写入 `localStorage` 时超过浏览器配额，未捕获的 `QuotaExceededError` 使 React 根节点卸载，用户看到整页白屏。首轮修复只保护了产品全周期缓存；品牌内容、数据中心、战略执行、供应链和 AI 会话仍直接写浏览器存储，存在同类复发风险。应用入口也缺少通用故障边界，其他渲染或生命周期异常仍可能表现为空白页面。

## 目标

- 浏览器本地或会话存储容量不足、被禁用、序列化失败时，应用继续渲染并继续使用服务端权威数据。
- 所有现有业务 Provider 通过同一个安全存储边界读写缓存，不再直接执行可能抛错的存储写入或删除。
- 未预见的 React 渲染或生命周期错误显示中文可恢复页面，不再卸载为无信息白屏。
- 故障页允许重新加载；用户明确确认后可只清理本应用的本机业务缓存再加载。
- 线上 D1、登录态、凭据、Cookie 和其他站点数据不受缓存清理影响。

## 非目标

- 不新增错误上报服务、监控供应商或服务端表。
- 不把浏览器缓存升级为业务事实源，也不恢复离线优先写入。
- 不修改 D1 表、API 数据契约、权限或外部平台连接。
- 不承诺捕获浏览器崩溃、网络断开、扩展程序崩溃等页面进程之外的故障。

## 用户与权限

- 所有已登录和未登录用户都受运行时故障边界保护。
- “重新加载”无需额外权限。
- “清理本机缓存后重试”只影响当前浏览器、当前域名下登记的非敏感业务缓存，不需要服务端权限；执行前必须明确确认本机未同步草稿可能丢失。

## 当前流程

1. Provider 在状态变化时直接调用 `localStorage.setItem` 或 `sessionStorage.setItem`。
2. 存储容量不足时异常逃逸。
3. React 根节点被卸载，页面无恢复入口。
4. 用户只能手工清理站点数据或等待研发重新部署。

## 目标流程

1. Provider 通过安全存储函数序列化、写入、读取或删除缓存。
2. 存储失败返回失败结果但不抛出，服务端 D1 和内存状态继续工作。
3. 若其他 React 渲染或生命周期异常发生，应用显示故障页。
4. 用户可先重新加载；若故障持续，可确认后只清理登记的本机业务缓存并重新加载。
5. 页面重新认证或读取现有会话，并从服务端恢复权威数据。

## 业务规则

- D1 和现有服务端 API 始终是共享业务数据权威源；浏览器缓存只用于首屏、本机草稿和人工恢复。
- 安全存储函数必须同时吞掉存储访问、序列化、写入和删除异常，并返回布尔结果。
- Provider 不得因缓存失败阻止内存状态更新、服务端读取或服务端保存。
- 清理范围采用显式键白名单，不调用 `localStorage.clear()` 或 `sessionStorage.clear()`。
- 清理范围不得包括认证、凭据、Cookie、Token 或第三方数据。
- 故障页不得展示原始异常、堆栈、业务数据或敏感上下文。

## 数据定义

- 本机业务缓存键：`productFlowState`、`productFlowStateDirty`、`productFlowStateRecoveryBackup`、`brandContentState:v1`、`dataCenterMetadata`、`platformExecutionState`、`supplyChainState`。
- 会话缓存键：AI 助手当前会话缓存键及部署刷新冷却标记。
- 缓存写入结果：`true` 表示当前写入成功，`false` 表示缓存不可用；失败不改变服务端数据状态。
- 故障页不持久化原始错误，仅在控制台保留 React 默认开发诊断。

## 异常与边界

- `QuotaExceededError`、`SecurityError`、受限 WebView、隐私模式和模拟存储异常都按缓存不可用处理。
- `JSON.stringify` 因循环引用或不支持值抛错时按缓存不可用处理。
- 清理某个键失败时继续处理其他白名单键，最终仍允许重新加载。
- 故障发生在 React 挂载之前时继续由现有静态启动壳和 Pages 分包恢复负责。
- 动态分包失效继续由 `vite:preloadError` 自动刷新逻辑负责，不与本功能重复。

## 验收标准

- 给任一 Provider 注入会抛配额错误的存储写入时，状态更新和页面渲染不抛错。
- 代码检查确认业务 Provider 不再直接调用存储写入或删除。
- 人为触发子组件渲染异常时，页面显示“页面遇到问题”及两个恢复动作。
- 清理动作只删除白名单键；未登记键、认证态和线上数据保持不变。
- 故障页支持键盘聚焦、Enter/Space 操作、窄屏和钉钉 WebView。
- 发布后在保留原有超额缓存的同源浏览器中打开正式地址，首屏、刷新和业务页面切换均正常，控制台无新增配额异常。

## 上线与回滚

- 无数据迁移、环境变量、D1 绑定或服务端开关。
- 先合入 `main` 触发 Cloudflare Pages 生产部署，再核对部署提交和入口资源哈希。
- 生产验证失败时回滚到上一已知良好 Pages 部署；回滚不删除 D1、浏览器缓存或会话。
- 回滚后若原容量问题复发，保留现有产品全周期安全存储修复，并停止扩大缓存写入。
