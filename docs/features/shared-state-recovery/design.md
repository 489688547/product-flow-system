# 产品全周期共享状态恢复设计书

## 用户任务

员工继续正常查看和修改产品进度；系统在后台保证旧页面、旧分支和本地缓存不能覆盖更新的公司数据。发生冲突时，用户看到明确提示并刷新取得最新数据。

## 信息层级

- 主信息：共享数据是否加载成功、当前修改是否保存成功。
- 辅助信息：版本冲突时的刷新建议、本机未同步副本已保留。
- 低频信息：快照、审计、恢复书签仅出现在运维和平台资料中。

## 页面结构

不新增页面。继续使用现有产品全周期界面和 `sharedError` 提示区域。平台资料中的 API 契约和架构文档承载长期规则。

## 交互流程

1. 首屏可以显示缓存内容，但保存按钮和自动同步逻辑在共享基线就绪前保持锁定。
2. GET 成功后以服务器状态替换首屏缓存；若发现旧脏标记，先保存本机恢复副本。
3. 用户修改后延迟合并保存，并携带服务器基线。
4. 保存成功后更新基线并清除对应脏标记。
5. 409 时保留本地修改，提示“线上数据已更新，请刷新后重新操作”，不自动覆盖或重试。

## 组件复用

- 复用 `ProductFlowProvider` 的 `sharedError`、本地缓存和延迟保存机制。
- 复用 `/api/platform/v1/production-data/state` 已有快照、审计和恢复能力。
- 复用现有公司会话与只读角色校验。

## 新增组件

不新增视觉组件。新增业务中立的状态同步会话模块，职责仅为记录服务器基线、决定何时允许写入、构造带版本的请求和接受成功响应；不包含 React、业务字段或外部平台逻辑。

## 页面状态

- 加载：读取服务器基线期间禁止共享写入。
- 空数据：显示不能自动初始化共享数据，避免演示数据落库。
- 错误：加载失败保持写锁；版本冲突保留本地修改并提示刷新。
- 无权限：沿用只读账号服务端 403。
- 禁用：没有有效 `baseUpdatedAt` 时同步模块不生成写请求。
- 成功：保存成功后清除对应本地脏标记并更新服务器基线。

## 响应式与钉钉 WebView

没有新增布局。需在笔记本、窄屏和钉钉 WebView 验证错误提示不会遮挡产品进度操作；旧 WebView 客户端 POST 因缺少基线返回 409。

## 交互文案

- 加载失败：`共享数据加载失败，已暂停自动保存，请刷新重试。`
- 本机副本：`发现本机未同步修改，已保留恢复副本并加载线上最新数据。`
- 版本冲突：`线上数据已被其他页面更新，本次修改未覆盖线上数据，请刷新后重新操作。`
- 未初始化：`线上共享数据尚未初始化，已阻止默认数据自动写入。`

## 无障碍

沿用现有错误提示语义和焦点顺序；不得只用颜色表达冲突。刷新后焦点保持在原页面主区域。

## 视觉验收

- 1440×900 产品进度正常态。
- 1280×800 版本冲突提示。
- 390px 窄屏加载失败提示。
- 钉钉 WebView 中旧客户端被拒绝后页面不崩溃。
