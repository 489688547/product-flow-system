# 共享商品主数据功能设计

## 设计结论

商品主数据属于平台共享能力，不放进供应链独立状态，也不把 ERP 商品等同于产品全周期项目。数据中心负责写入和治理；产品全周期、供应链、供应商管理通过 `ProductCatalogProvider` 读取同一 API。

## 信息架构

数据中心侧边栏在“数据分析”后增加“商品主数据”。页面不增加二级侧栏：

1. 状态带：商品数、SKU 数、标准 69 码、异常条码、最近同步。
2. 操作与筛选：搜索、状态、分类、关联状态、同步快麦、导入文件。
3. 单一商品表：商品、编码、SKU/69 码、分类品牌、价格状态、业务关联和操作；真实目录按 50 条分页。
4. SKU/69 码在主表单元格内最多展示 3 条，更多数量明确提示；不以嵌套卡片重复主表。

产品全周期的编辑弹窗增加“ERP 商品关联”。选择后展示只读主商家编码和 SKU/69 码，人工售价仍可在旧字段中覆盖；解除关联不会删除旧 69 码。

产品供应链直接展示目录商品表。关联供应商从当前行发起，默认带入目录商品和可选 SKU，不再先创建一条看不见商品档案的供应关系。

供应商管理在供应商名称/编码下显示已关联目录商品数，点击数量跳转产品供应链并带筛选条件后续扩展；本期先提供清晰覆盖数。

## 视觉与交互

- 延续现有浅色、克制、高密度工作台；标题 24px、表格 13px、4px 间距体系。
- 状态带是一条连续容器，不使用五张装饰卡片。
- 表头和编码保持 `white-space: nowrap`；商品名称允许两行，条码使用等宽数字。
- 主操作只有“同步快麦”，文件导入为次操作。
- 同步中主按钮禁用并显示进度；读取失败保留现有表格和明确的“重新加载”入口，重试期间按钮禁用。
- 文件导入解析后先显示文件名、有效商品/SKU、异常行和确认/取消操作。
- 筛选无结果与完全无目录使用不同空状态。

## 页面状态

- 加载：状态带和表格行骨架，保留页面标题。
- 空目录：HTTP 200 且 `synced: false`、`items: []` 表示尚未首次同步，是正常空状态；解释可从快麦同步或导入 ERP 文件，不显示加载失败。
- 无搜索结果：保留筛选栏，提供清除筛选。
- 错误：网络异常、非成功 HTTP 或无效读取响应才进入错误状态；保留缓存数据，显示安全消息和“重新加载”操作。
- 无写权限：仍可搜索查看，不显示导入控件，同步按钮显示只读原因。
- API 未配置：同步请求返回明确配置错误，文件导入仍可用。
- 同步成功：显示本批商品/SKU 数和完成时间；文件补齐项不因 API 未返回而失效。

## 响应式与无障碍

- 1280px 以上完整列；900px 隐藏次要价格列；640px 操作栏竖排；390px 表格内部滚动。
- 所有按钮具备文本标签、可见焦点、禁用原因和忙碌状态。
- 搜索输入有明确 label；筛选菜单保留键盘操作；展开行使用 `aria-expanded`。
- 状态不只依赖颜色，条码类型显示文字。

## 共享组件结论

- `ProductCatalogProvider`、领域标准化和 API 客户端成为共享平台能力。
- `ProductCatalogSelect` 有产品档案和供应链两个真实消费者，放在 `src/features/product-catalog`。
- 商品主数据表仅数据中心使用，保留在数据中心功能目录。
- 复用现有 `Button`、`DataTable`、`HeaderFilter`、`Modal` 和上传控件样式。
