# 共享商品主数据 PRD

## 文档状态

- 状态：已完成本地验收，待独立生产发布确认
- 负责人：数据中心 / 产品部 / 供应链
- 最近更新：2026-07-19

## 背景与问题

产品全周期通过产品记录里的 `skuCodes` 维护 69 码，供应链又通过产品 ID 维护供应商关系，快麦 ERP 的商品档案没有进入统一数据层。相同商品因此需要重复录入，ERP 的主商家编码、规格商家编码、69 码、品牌、分类、成本和启停状态无法稳定关联销售、库存、采购和质量事实。

2026-07-19 实测商品档案 V2 默认视图有 298 条规格记录、274 个商品；开放平台 `item.list.query` 返回 912 条、按系统商品 ID 合并为 855 个主商品。API 返回主/规格商家编码、分类、成本、售价、重量和状态，但当前授权响应没有单独的 `barcode` 字段；商品档案导出可补齐 298 条规格明细，69 码主要保存在规格商家编码中。因此 API 与文件是互补来源，不能把其中一方的缺失当作另一方已删除。

## 目标

1. 建立平台级商品主数据，数据中心是导入、同步和质量治理入口。
2. 产品全周期、供应链和供应商管理读取同一份商品及 SKU/69 码数据。
3. 保留产品全周期“业务产品”和 ERP“商品主数据”的边界，通过稳定 ID 显式关联。
4. 支持快麦 API 全量同步和 ERP XLSX/CSV 文件导入，重复执行保持幂等。
5. 用统一条码映射连接销量、库存、采购成本和质量问题。

## 非目标

- 不把全部 ERP 历史商品自动创建为产品全周期项目。
- 不从本系统反向新增或修改快麦商品。
- 不在本期声称快麦实时库存已经接通。
- 不把快麦返回的成本价写入产品全周期共享状态。
- 不删除旧 `skuCodes`；旧数据继续兼容并可在关联目录商品后自动校准。

## 用户与权限

- 所有已登录员工可读取商品名称、编码、分类、品牌和启用状态。
- 总经办与运营部可在数据中心执行商品文件导入和快麦同步。
- 产品部可在产品档案关联已有目录商品，不能改写 ERP 主数据。
- 供应链、采购和财务可读取成本价；其他部门的 API 响应不返回采购成本。
- `readonly` 会话不能导入、同步或修改关联。

## 核心业务规则

1. 商品款优先以来源 + 主商家编码生成稳定身份，保证 API 与不带系统 ID 的导出文件落到同一记录；系统商品 ID作为来源属性保留。缺少主商家编码时才回退来源系统 ID。
2. SKU 优先以商品身份 + 规格商家编码生成稳定身份，系统 SKU ID 作为来源属性保留；缺少规格商家编码时才回退系统 SKU ID 或条码。
3. 一个 ERP 商品包含一个或多个 SKU；SKU 69 码不属于产品全周期记录的主事实。
4. 产品全周期记录保存 `catalogProductId`，展示和销售查询优先使用目录 SKU；未关联时继续读取旧 `skuCodes`。
5. 产品供应关系保存 `catalogProductId` 和可选 `catalogSkuId`；旧 `productId` 继续兼容。
6. 69 开头、12 至 14 位数字按现有销售条码规则标记为标准 69 码；其他非空条码显示为“非标准条码”并进入质量统计。
7. API 全量同步只有在全部分页成功后才提交本批次；失败保留上次成功目录。
8. 快麦商品列表 API 与商品档案导出的覆盖范围不同；自动同步只做幂等更新，不把文件中存在、API 未返回的商品标记为缺失。
9. 文件导入先解析、预览有效/异常行，用户确认后写入；文件名、行数和批次结果进入同步记录，不保存原始整行。

## 数据定义

- 商品：`id`、来源、来源商品 ID、主商家编码、名称、简称、图片、类型、分类、品牌、启用状态、创建/修改/同步时间。
- SKU：`id`、商品 ID、来源 SKU ID、规格商家编码、69 码/条码类型、规格、别名、图片、单位、重量、采购成本、销售价、启用状态、创建/修改/同步时间。
- 同步批次：来源、方式、状态、开始/完成时间、读取页数、商品数、SKU 数、异常数、错误码和安全消息。
- 产品关联：产品全周期记录上的 `catalogProductId`；旧 `skuCodes` 由目录 SKU 派生并保留人工定价兼容。

## 验收标准

1. 数据中心新增“商品主数据”入口，能搜索名称、69 码、主/规格商家编码并按状态、分类和关联状态筛选。
2. 页面显示商品总数、SKU 数、标准 69 码数、异常条码数和最后成功同步时间。
3. 快麦同步读取全部分页，当前 912 条记录在 5 页完成；任一分页失败时不覆盖旧数据。
4. XLSX/CSV 导入支持快麦商品档案 V2 中文表头与 GB18030 CSV，并在确认前显示有效和异常行。
5. 产品档案可以选择目录商品，保存后销售查询使用该商品全部标准 69 码。
6. 产品供应链以目录商品为主表，直接展示编码、品牌分类、SKU、关联供应商和成本覆盖。
7. 供应商管理显示每家供应商已关联的目录商品数量。
8. 无目录、加载、错误、无权限、同步中和导入失败状态均有可执行中文反馈。
9. 桌面、900px、640px 和 390px 下无页面级横向溢出；表格文字不竖排，水平滚动仅在表格容器。

## 上线与回滚

- 新增 D1 表和 API，不替换 `/api/state`、`/api/supply-chain` 或 `/api/sales`。
- 回滚前端时停止挂载 Provider 和入口；新增表保留，不删除数据。
- 产品记录和供应关系保留旧字段，旧版本仍可读取。
- 未经单独确认不执行 Cloudflare 生产部署或真实快麦同步。
