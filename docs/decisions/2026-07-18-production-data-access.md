# 架构决策：测试环境通过受控网关读写生产数据

## 状态

已接受；本地日常开发部分由 `2026-07-20-local-online-account.md` 取代，本文继续约束生产数据运维修复网关。

## 背景

测试环境需要使用真实业务数据，否则钉钉审批、共享状态和其他平台联动只能在不完整的假数据上验证。用户确认当前最高权限测试账号可以直接读写线上数据库，但数据库修改权不应自动扩展为钉钉消息、待办、审批或其他外部副作用权限。

## 决策

- 本地测试通过服务端生产数据网关实时读取生产 D1，不由浏览器直接持有凭据。
- 初始只为当前指定的最高权限钉钉身份签发个人访问令牌；稳定身份使用 `userId` 与 `unionId`，不使用姓名。
- 原始个人令牌只保存在被忽略的本地 `.env`；D1 只保存 SHA-256 哈希。
- 跨环境生产写入必须具备 `executive` 身份、个人令牌 `write` 能力和 15 分钟有效解锁。
- 写入前执行版本冲突检查和快照，写入后记录审计；允许从写前快照回滚。
- 测试环境的数据库写入能力与外部平台真实操作能力分离，本期外部副作用保持禁止。
- 环境变量名、绑定名和表结构要求由公开环境能力清单驱动；运行时只展示存在性，不展示值。

## 备选方案

1. 测试直接绑定生产 D1：实现最简单，但 D1 绑定没有数据库级只读权限，新增路由容易绕过保护，未采用。
2. 定时复制生产数据到测试库：隔离最好，但数据延迟且不满足实时修正线上数据的要求，未采用。
3. 浏览器跨域调用生产 API：会暴露个人令牌或依赖跨站 Cookie，未采用。

## 后果

收益是测试能使用实时真实数据，并且线上修正具备明确身份、短时授权、冲突保护、快照和审计。成本是需要维护个人令牌、生产网关和快照容量；生产数据令牌丢失时必须立即撤销。外部平台真实写操作仍需单独设计与授权。

## 兼容与迁移

生产应用继续使用现有公司会话和 `/api/state`，不要求普通员工解锁。新增网关和表是旁路能力；回滚时撤销个人令牌、移除本地配置并回退应用即可，现有业务数据和登录不受影响。
